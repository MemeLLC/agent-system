```markdown
### 命名規則

**キャンペーン名:** `{目的}_{ファネル}_{ターゲット}_{地域}_{補足}`

| 要素 | 値 |
|------|-----|
| 目的 | `Sales` `Traffic` `Reach` `VideoViews` `LeadGen` `AppInstall` `Shop` |
| ファネル | `TOF`（Top of Funnel）`MOF`（Middle）`BOF`（Bottom）`RT`（Retargeting） |
| ターゲット | `Broad` `Lookalike` `Custom` `Interest` `Retarget` |
| 地域 | `JP` `Tokyo` `Kansai` 等（全国は省略可） |

例: `Sales_BOF_Retarget_JP` / `Traffic_TOF_Broad` / `Shop_GMVMax`

**広告グループ名:** `{オーディエンス}_{補足}`

例: `Broad_18plus` / `LAL_Purchasers_Narrow` / `RT_CartAbandoners_7d` / `Interest_Beauty`

**広告名:** `{クリエイティブタイプ}_{バリエーション}`

例: `UGC_HookA_v1` / `ProductDemo_Review_v2` / `SparkAd_Creator01`

**ルール:**
- 区切りは `_`（アンダースコア）で統一
- 英語ベース推奨。キャメルケース（`CartAbandoners`, `ProductDemo`）で単語を区切る
- 日付はキャンペーン名に含めない（テスト施策のみ `_Test_YYMM` を末尾に付与）
```

```
| 許容される変更 | 不承認クリエイティブの差し替え、明らかな設定ミスの修正 |

### クリエイティブリフレッシュスケジュール

| アクション | 頻度 | トリガー |
|-----------|------|---------|
| マイクロリフレッシュ（フック差し替え等） | 3〜5日ごと | フリクエンシー > 2.5 |
| 中規模リフレッシュ | 7〜10日ごと | CTR/CVR持続的低下 |
| フルリフレッシュ（新規3〜5本追加） | 7日ごと | 定期 |
| 完全刷新（新コンセプト） | 2〜4週間ごと | 全体パフォーマンス低下 |

### 予算調整スケジュール

| アクション | ルール |
|-----------|-------|
| 増額（パフォーマンス良好時） | 24時間ごとに最大20%まで |
| 減額 | 50%以上の削減は学習フェーズリセットのリスク。段階的に |
| CBO時の調整 | ラーニングフェーズ中: 40%以上の増額を避ける。完了後: 30%以内 |
| シーズナリティ対応 | ピーク2-3週間前から段階的増加（1日30%ずつ） |

## 9. リスクと対策

| リスク | 対策 |
|-------|------|
| ラーニングフェーズ中のCPA高騰 | 7日間は想定内。予算は目標CPAの20倍以上を維持 |
| クリエイティブ疲労（TikTokはMetaの4倍速い） | 7日ごとに3〜5本の新規クリエイティブ投入。フリクエンシー2.5超でリフレッシュ |
| 予算消化不足（Cost Cap時） | CPA入札を20%以上引き上げ、またはMaximum Deliveryに切り替え |
| 急激な予算変更による学習リセット | 20%ルールを厳守。50%以上の増減を避ける |
| シーズナリティによるCPM高騰 | ピーク期間前に勝ちクリエイティブを確定。Q5期間での積極投資 |
| 審査落ち（景表法・薬機法） | 出稿前に法規制チェックリスト確認。誇張表現・効果保証を排除 |
| ステマ規制違反 | インフルエンサー起用時・Spark Ads利用時はPR表記必須 |
| 内部競合（同一ターゲット重複） | 広告グループ間で差別化ポイントを明確に設定 |
```
