# In-Feed Ads（インフィード広告）

## 概要

ユーザーのFor You（おすすめ）フィードに自然に表示される動画広告。通常の投稿動画の間に挿入されるため、ユーザー体験を妨げにくいネイティブな広告フォーマット。いいね・コメント・シェアなどのエンゲージメントが可能。

- **購入方式**: オークション型（運用型）。リアルタイム入札で配信。予算・ターゲティングを自由に調整可能。日予算2万〜10万円程度からスモールスタート可能。Reach & Frequency（予約型）にも対応
- **課金方式**: CPM, CPC, oCPM, CPV
- **対応するキャンペーン目的**: リーチ、トラフィック、動画視聴、コミュニティインタラクション、アプリプロモーション、リード獲得、ウェブコンバージョン、販売（全目的に対応する最も汎用性の高いフォーマット）
- **主な差別化ポイント**: 最も基本的かつ一般的な広告フォーマット。全てのキャンペーン目的に対応し、柔軟な予算設定が可能。ユーザーの43%がTikTok広告をオーガニックコンテンツと区別しにくいと回答

## 仕様

### 動画仕様

| 項目 | 仕様 |
|------|------|
| アスペクト比 | 9:16（推奨）、1:1、16:9 |
| 推奨解像度 | 1080 x 1920 px（9:16）、640 x 640 px（1:1）、1280 x 720 px（16:9） |
| 最小解像度 | 540 x 960 px（9:16）、640 x 640 px（1:1）、960 x 540 px（16:9） |
| 動画の長さ | 5〜60秒（推奨: 21〜34秒） |
| 最適尺 | 9〜15秒（高エンゲージメント）/ 15〜30秒（詳細訴求） |
| ファイル形式 | .mp4, .mov, .mpeg, .3gp, .avi |
| ファイルサイズ | 最大500 MB |
| ビットレート | 516 kbps以上推奨 |

### テキスト仕様

| 項目 | 仕様 |
|------|------|
| 広告説明文（キャプション） | 1〜100文字（最初の80〜100文字が表示、最大4行） |
| フォント | 白色統一フォント（カスタマイズ不可） |
| 絵文字 | 使用可 |
| ブランド/アプリ名 | 2〜20文字（日本語）/ 4〜40文字（英語） |

### セーフゾーン

- 動画の上部・下部にはTikTokのUI要素（ユーザー名、キャプション、いいね/コメント/シェアボタン等）がオーバーレイ表示される
- 重要なテキスト・ロゴは画面中央のセーフゾーン内に配置すること
- 1080x1920px動画の場合、上部120px・下部120pxはUI被りの可能性あり

## クリエイティブ制作ガイド

### フックの設計（最初の3秒が全てを決める）

- **最初の3秒で視聴継続率の71%が決まる**。スクロールを止めるフックが最重要
- 最初の3秒を視聴したユーザーの**45%が30秒以上視聴を継続する**
- **広告想起の90%は最初の6秒以内**に形成される

#### フックの具体的テクニック

| フック手法 | 具体例 |
|-----------|--------|
| 質問型 | 「まだ〇〇で悩んでる？」「知ってた？〇〇の事実」 |
| 驚き・衝撃型 | 衝撃的なビフォーアフター、予想外の展開 |
| 問題提起型 | 「〇〇が失敗する3つの理由」 |
| カウントダウン型 | 「3つの方法を教えます」「ランキングTOP5」 |
| ビジュアルインパクト型 | 動きのあるシーン、色のコントラスト |

### 映像制作のポイント

- **UGCスタイルの動画がプロ制作動画を22%上回る**パフォーマンスを記録
- スマートフォンで撮影した自然なスタイルが信頼感を生む
- **縦型（9:16）で撮影**し、フルスクリーン体験を最大化
- 720p以上の解像度を確保（推奨は1080p）
- UIセーフゾーンを意識し、重要な要素が隠れないようにする
- 1080x1920pxの動画では、上部120pxと下部120pxにテキストやロゴを配置しない

### コンテンツスタイル

- **TikTokネイティブなトーン**: カジュアル、ユーモア、親しみやすさを重視
- **人物を中心に据える**: クリエイター、社員、顧客など「人」が映るとエンゲージメント向上
- **ストーリーテリング構造**: 問題→解決、ビフォーアフター、驚きの展開
- 過度にブランド色を出さず、オーガニック投稿に見える仕上がりを目指す

### 音楽・サウンド

- **ナレーション付き広告はCVRが12%向上**する
- TikTok Commercial Music Library（CML）から選曲すること（商用利用はCMLの楽曲のみ許可）
- トレンドサウンドは一般アカウント用であり、**ビジネスアカウントではCML以外の楽曲使用は著作権侵害のリスク**あり
- 30-40%のユーザーが音声オフで視聴するため、テキストオーバーレイとの併用が必須

### テキストオーバーレイ

- テキストオーバーレイを含む広告は**リコール+58%、ブランド好感度+95%**
- **推奨ペース: 5-10語/秒**（+2.1xの認知リフト）
- 冒頭フレームにタイトルを表示すると**ブランド想起+48%、購買意向+11%**
- 全広告の**86%がテキストボックスを使用**、71%がテキストオーバーレイを含む
- CTAをオープニングフレームに配置すると**CVR+44%**
- **センター左寄りに配置**するとデバイス問わず視認性が高い

### クリエイティブの更新頻度

- **TikTokではMeta比4倍の速さでクリエイティブ疲労が発生**する
- 高予算キャンペーンでは**3日程度で疲労が始まる**ケースも
- **週2回のクリエイティブリフレッシュ**を推奨（大規模運用時）
- 通常の運用でも**1-2週間ごとに更新**
- フリークエンシーが2.5を超えたらリミックス版をアップロード
- CTRの低下やCPMの上昇が疲労のサイン

### クリエイティブテスト

- 1キャンペーンあたり**10-20のクリエイティブバリエーション**をテスト
- 1広告グループあたり**3-5本のクリエイティブ**を設定
- 1キャンペーンあたり**3-5の広告グループ**で多様化
- フック→ボディ→CTAの各パートでA/Bテスト実施
- Smart Creativeを活用し、自動的にクリエイティブ組み合わせを最適化

### Do's（推奨事項）

- UGCスタイル・ネイティブな見た目で制作する
- 最初の3秒に強力なフックを配置する
- 人物（クリエイター、社員、顧客）を映す
- テキストオーバーレイを含め、音声オフでも伝わるようにする
- CMLから音楽を選曲する
- 1-2週間ごとにクリエイティブを更新する

### Don'ts（禁止・非推奨事項）

- プロ制作の過度にポリッシュされた動画（UGCに劣後する）
- 重要な要素をセーフゾーン外に配置する
- CML以外のトレンドサウンドをビジネスアカウントで使用する
- 同じクリエイティブを長期間使い回す
- TikTokのUIを模倣するテキストデザイン
- 誇張表現や絶対的な表現（「世界一」「100%」など）

## 広告設定ガイド

### キャンペーン構造

| レベル | 設定項目 | 推奨設定 |
|--------|---------|---------|
| キャンペーン | 目的 | 認知度（Awareness）/ トラフィック / アプリインストール / コンバージョン |
| キャンペーン | 予算 | 日予算最低$50（キャンペーンレベル） |
| 広告グループ | 予算 | 日予算最低$20（広告グループレベル） |
| 広告グループ | ターゲティング・配信 | 詳細は下記参照 |
| 広告 | クリエイティブ・CTA | 3-5本/広告グループ |

### 予算設定

- **推奨日予算 = ターゲットCPA x 10**（学習フェーズ完了に必要）
- 学習フェーズ完了には約**50コンバージョン**（インストール目標）または**20コンバージョン**（深いファネル目標）が必要
- 30日のテスト期間で**$1,500-$3,000**の予算を目安に
- 学習フェーズ中の予算調整は**40%以下の増額**に抑える
- 安定後のスケーリングは**2-3日ごとに15-20%ずつ**増額

### 入札戦略

| 入札戦略 | 特徴 | 推奨シーン |
|---------|------|-----------|
| **Maximum Delivery（最大配信）** | 予算内で最大数のコンバージョンを獲得。CPA目標なし | コンバージョン数の最大化が目標の場合 |
| **Cost Cap（コストキャップ）** | 設定したターゲットCPAに近づくよう最適化 | 特定のCPA目標がある場合 |

- **初期段階は高めの入札から開始**し、低い入札から徐々に上げるよりもパフォーマンスが良い
- CPA/ROASが目標未達でも**すぐに入札を下げない**。システムの調整を待つ
- Hybrid Biddingで入札戦略を切り替え可能（新規広告グループ不要）

### 学習フェーズの管理

- 学習フェーズは**約25結果または7日間**で安定化が始まる
- 学習フェーズ中は**ターゲティング・入札・予算の変更を避ける**（最低2日間は維持）
- 変更を加えると学習がリセットされる可能性がある
- 学習完了後にスケーリングを開始する

### ターゲティング設定

| ターゲティング手法 | 説明 | 推奨事項 |
|-----------------|------|---------|
| **インタレスト** | コンテンツカテゴリへの長期的な関心 | 3-5カテゴリを選択し、過度に絞りすぎない |
| **行動ターゲティング** | 動画の視聴・いいね・コメント・シェア行動 | 直近のアクションに基づく高精度ターゲティング |
| **購買意向** | 特定の商品・サービスカテゴリへの購買意向 | ECやアプリ広告に効果的 |
| **カスタムオーディエンス** | CRMリスト、ウェブ訪問者、アプリユーザーなど | 最低1,000ユーザーが必要。最もCPA/CPLが低い |
| **類似オーディエンス** | カスタムオーディエンスに類似したユーザー | ソース10,000人以上推奨。類似度で精度 vs リーチを調整 |
| **Smart+キャンペーン** | AI自動最適化。手動ターゲティング不要 | 2025年導入。KPI・クリエイティブ・地域・言語のみ設定 |

### 最適化のヒント

- 初期は**広めのターゲティング**で開始し、データ収集後に絞り込む
- リターゲティング用のカスタムオーディエンスは**フリークエンシー5-8**が許容範囲
- 広告パフォーマンスを定期的にチェックし、新規ユーザーが減少傾向の場合はクリエイティブを更新
- Campaign Budget Optimization（CBO）を活用して自動的に予算を最適配分

## コピーライティングガイド

### キャプション（広告テキスト）

- **文字数制限: 1-100文字**（絵文字使用可）
- 推奨文字数: **40-60文字**でパンチのあるメッセージ
- 最大4行まで表示可能
- フォントは白色の統一フォント（カスタマイズ不可）
- 最も重要な情報を最初に配置する（折り返し前に伝える）

### キャプションの書き方

| ポイント | 説明 |
|---------|------|
| **簡潔さ** | 短く、インパクトのある表現。ダラダラ書かない |
| **行動喚起** | パワーワードと行動動詞を使う |
| **好奇心の喚起** | 質問形式や「知ってた？」系で興味を引く |
| **絵文字の活用** | テキスト単体より情報伝達が速い。適度に使用 |
| **会話調のトーン** | 企業的・営業的な言い回しを避け、友達に話すように |
| **ハッシュタグ** | 関連ハッシュタグを2-3個。ブランドタグも検討 |

### CTA（コールトゥアクション）ボタン

| CTA | 推奨用途 |
|-----|---------|
| **Shop Now** | EC・商品購入ページへの誘導 |
| **Learn More** | 情報提供・LP誘導 |
| **Download** | アプリインストール |
| **Sign Up** | 会員登録・メルマガ登録 |
| **Order Now** | 注文・予約 |
| **Play Game** | ゲームアプリ誘導 |
| **Visit Store** | 店舗・TikTok Shop誘導 |
| **Contact Us** | お問い合わせ・リード獲得・B2B |
| **Apply Now** | 求人・サービス申込 |
| **Book Now** | 予約・来店促進 |
| **Watch More** | 動画視聴促進 |

- **Dynamic CTA**: 複数のCTAテキストをプールから選択し、システムが最適化して配信
- **AIGC CTA**: ランディングページの内容からAIがCTAテキストを自動生成
- CTAディスプレイカードを使用すると**リコール+45%、好感度+19%**
- アクション指向のオーバーレイ（「今すぐ購入」「詳しく見る」など）で**CVR+18%以上**

### テキストオーバーレイのルール

- **5-10語/秒のペース**で表示（読みやすさの最適ペース）
- テキスト量とタイミングのバランスが重要。過剰なテキストは逆効果
- **センター左寄りに配置**するとデバイス問わず視認性が高い
- スペルミスや文法エラーは審査不通過の原因になる
- TikTokのUIを模倣するテキストデザインは禁止

### コピーの品質基準（TikTokポリシー）

- スペル・文法ミスがないこと
- 記号の不正使用がないこと
- 誇張表現や絶対的な表現（「世界一」「100%」など）を避ける
- 誤解を招く主張や過度な効果保証は禁止
- AI生成コンテンツの場合は明確なラベリングが必要
