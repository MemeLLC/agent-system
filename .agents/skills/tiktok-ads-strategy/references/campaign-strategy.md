## 1. キャンペーン構造

### 3階層構造

```
広告アカウント（Ad Account）
└── キャンペーン（Campaign）           ← 目的・予算タイプを設定
    └── 広告グループ（Ad Group）       ← ターゲティング・配信先・入札を設定
        └── 広告（Ad）                 ← クリエイティブ・テキスト・CTAを設定
```

### 各階層の役割

| 階層 | 役割 | 上限数 |
|------|------|--------|
| **キャンペーン** | 広告の目的（Objective）と予算タイプを決定する「戦略層」 | アカウントあたり複数作成可能 |
| **広告グループ** | 誰に・どこで・いくらで配信するかを制御する「戦術層」 | 1キャンペーンあたり最大999 |
| **広告** | ユーザーが実際に目にするクリエイティブの「実行層」 | 1広告グループあたり最大50 |

### 推奨構成（2025年ベストプラクティス）

- 1キャンペーンあたり **3〜5広告グループ**
- 1広告グループあたり **3〜5クリエイティブ**（テスト時は4〜6種類）
- 2025年のトレンド: 細かいニッチターゲティングよりも**ブロードターゲティング**を設定し、アルゴリズムにクリエイティブベースで最適なユーザーを見つけさせるアプローチが主流

### 広告フォーマット

| フォーマット | 説明 |
|------------|------|
| **インフィード広告（In-Feed Ads）** | For Youフィードに自然に表示される最も一般的な形式 |
| **TopView** | アプリ起動時に最初に表示されるプレミアム動画広告 |
| **ブランドテイクオーバー（Brand Takeover）** | アプリ起動時の3-5秒フルスクリーン広告。1カテゴリ1日1広告主のみ（独占枠） |
| **ブランドハッシュタグチャレンジ** | ハッシュタグを使ったUGCキャンペーン |
| **ブランドエフェクト** | AR/VRエフェクトを使ったインタラクティブ広告 |
| **コレクション広告** | 動画とプロダクトカードの組み合わせ（Video Shopping Adsに移行中） |
| **ダイナミックショーケース広告（DSA）** | 商品カタログに基づく動的パーソナライズ広告 |
| **カルーセル広告** | 複数画像のスワイプ形式広告 |

### Spark Ads

実在するTikTokアカウントのオーガニック投稿を広告として配信するフォーマット。

- 通常広告と比較して**完全視聴率が30%向上**
- **エンゲージメント率が142%向上**
- 全エンゲージメント（いいね、コメント、シェア、フォロー）がオーガニック投稿に帰属

### 配信先（Placement）

| 配信先タイプ | 説明 |
|------------|------|
| **自動配信（Automatic Placement）** | TikTok、Pangle、Global App Bundleなど複数配信面に自動最適化 |
| **手動選択（Select Placement）** | 配信面を手動で選択 |

配信面の種類:
- **TikTok**: For Youフィード等（課金: oCPM）
- **Pangle**: TikTok for Businessのアドネットワーク（課金: CPC / CPM / oCPM）
- **Global App Bundle**: ByteDanceの新興アプリ群（CapCut、Fizzo等）（課金: CPC / oCPM）

> **注意**: 配信先は広告グループ作成後に変更不可。

### 遷移先（Destination）

| 遷移先タイプ | 説明 |
|------------|------|
| **外部URL** | ウェブサイトやランディングページへ遷移 |
| **Instant Page** | TikTokアプリ内で開く高速ネイティブLP。通常のモバイルページの**最大11倍高速** |
| **アプリストア** | App Store / Google Playへ遷移 |
| **TikTok Shop** | TikTokショップの商品ページへ遷移 |

## 2. キャンペーン目的

3カテゴリ・7目的で構成される。

### 認知（Awareness）

| 目的 | 説明 |
|------|------|
| **リーチ（Reach）** | 広告を記憶する可能性が高いユーザーにリーチし、ブランド認知を拡大 |
| **動画再生数（Video Views）** | 動画広告の再生数とエンゲージメントを最大化 |
| **コミュニティインタラクション** | フォロワー獲得、プロフィール訪問促進、ライブ配信のプロモーション |

### 検討（Consideration）

| 目的 | 説明 |
|------|------|
| **トラフィック（Traffic）** | ウェブサイトやアプリへのアクセスを増加 |
| **リード生成（Lead Generation）** | フォームを通じてリード情報を収集 |
| **アプリプロモーション（App Promotion）** | アプリのインストールとアプリ内アクションを促進 |

### コンバージョン（Conversion）

| 目的 | 説明 |
|------|------|
| **セールス（Sales）** | ウェブサイトコンバージョンと商品販売を統合した目的 |

> **2025年の変更点**: Website ConversionsとProduct Salesが**Sales**に一本化。2025年7月以降、TikTok Shop広告ではGMV Maxがデフォルトかつ唯一のキャンペーンタイプとなる予定。

## 3. CBO vs ABO

### CBO（Campaign Budget Optimization）

| 項目 | 詳細 |
|------|------|
| **仕組み** | キャンペーン全体の予算をTikTokが広告グループ間で自動配分 |
| **最低日予算** | $50（約7,500円）/日 |
| **必要条件** | 2個以上のアクティブな広告グループ（推奨3-5個） |
| **再配分頻度** | 15〜30分ごとにリアルタイム |
| **対応入札戦略** | Maximum Delivery / Cost Cap |
| **適したケース** | スケーリング、予算が限定的、検証済みのオーディエンス |

**CBO予算調整のガイドライン**:
- ラーニングフェーズ中: 1回の調整で**40%以上の増額を避ける**
- ラーニングフェーズ終了後: 1回の調整で**30%以上の増額を避ける**

### ABO（Ad Group Budget Optimization）

| 項目 | 詳細 |
|------|------|
| **仕組み** | 各広告グループに個別の予算を設定し、手動で管理 |
| **最低日予算** | $20（約3,000円）/日 |
| **予算タイプ** | 日予算（Daily）/ 通算予算（Lifetime） |
| **適したケース** | テスト、大規模予算、個別制御が必要な場合 |

### 使い分けガイド

| シナリオ | 推奨 |
|---------|------|
| 新規アカウント・テスト段階 | ABO |
| クリエイティブA/Bテスト | ABO |
| オーディエンステスト | ABO |
| 検証済みキャンペーンのスケーリング | CBO |
| 少額予算での運用 | CBO |
| 複数オーディエンスの自動最適化 | CBO |

### CBO運用のベストプラクティス

- 最適化前に**最低3日間または50コンバージョン**を待つ
- 予算調整は現在の日予算の**30%以内**に抑える
- キャンペーン内に**3-5個のアクティブな広告グループ**を維持
- パフォーマンスが低い広告グループは一時停止して予算を再配分

## 4. ターゲティング設計

### 4.1 デモグラフィックターゲティング

| 項目 | 設定内容 |
|------|----------|
| **地域（Location）** | 国、地域、都市、郵便番号レベルで指定可能 |
| **年齢（Age）** | 13-17, 18-24, 25-34, 35-44, 45-54, 55+ |
| **性別（Gender）** | 男性、女性、無制限 |
| **言語（Language）** | ユーザーのアプリ設定言語で指定 |

### 4.2 興味関心・行動ターゲティング（4カテゴリ）

| カテゴリ | 特性 | 用途 |
|---------|------|------|
| **興味関心（Interests）** | 長期的・受動的な嗜好を反映 | ブロードなリーチ |
| **購入意向（Purchase Intention）** | 特定商品を積極的にリサーチ・購入中 | 高意向ユーザーへのアプローチ |
| **動画インタラクション** | 過去7日間/15日間の動画アクション（完全視聴、いいね、コメント、シェア） | 直近の高インテントユーザー |
| **クリエイターインタラクション** | 特定カテゴリのクリエイターのフォロー・プロフィール閲覧 | カテゴリ特化のリーチ |

> **興味関心 vs 行動の違い**: 興味関心は受動的・長期的・ブロード。行動は能動的・直近（7-15日）・高インテント。

### 4.3 カスタムオーディエンス

| タイプ | 説明 |
|--------|------|
| **顧客ファイル** | メール、電話番号、広告IDリストをアップロードしてマッチング |
| **ウェブサイトトラフィック** | TikTok Pixelで取得した訪問者のリターゲティング |
| **アプリアクティビティ** | アプリ内イベントに基づくセグメント |
| **エンゲージメント** | TikTok上のインタラクションに基づくセグメント |
| **リード生成** | リード広告のフォーム閲覧・未送信ユーザーのリターゲティング |
| **ビジネスアカウントオーディエンス** | ビジネスアカウントとのインタラクション |
| **ショップアクティビティ** | TikTok Shopでのユーザー行動 |

> **最小要件**: カスタムオーディエンスを使用するには、マッチしたユーザーが**最低1,000人**必要。

### 4.4 類似オーディエンス（Lookalike Audience）

| 類似度レベル | 類似度 | リーチ | 推奨用途 |
|------------|--------|--------|---------|
| **Narrow（狭い）** | 最も高い | 最も小さい | 精度重視の獲得キャンペーン、テスト初期 |
| **Balanced（バランス）** | 中程度 | 中程度 | 汎用的、多くのケースで推奨 |
| **Broad（広い）** | 低め | 最も大きい | 認知拡大、スケーリング段階 |

- ソースの最小サイズ: **1,000人**（推奨2,000-5,000人以上、最適10,000人以上）
- 作成所要時間: **24〜48時間**
- 推奨戦略: Narrow → Balanced → Broadの順に拡張してスケーリング

#### 効果的なソースオーディエンス（優先度順）

1. **購入者リスト** - 最も高品質なシグナル
2. **カート追加者** - 購買意欲の高いユーザー
3. **リード獲得者** - ビジネスへの関心が高いユーザー
4. **高エンゲージメント動画視聴者** - ブランドへの関心あり
5. **全サイト訪問者** - 最も広いが質は低め

### 4.5 ブロードターゲティング（2025年推奨）

ブロードターゲティング（国内の潜在ユーザーの80%以上をカバー）が最も高い成果を出す。

| 指標 | ブロード vs 詳細ターゲティング |
|------|------|
| CPA | **-15%**（ブロードの方が低い） |
| CVR | **+20%**（ブロードの方が高い） |

**設定方法:**
- インタレスト・行動ターゲティングを空白にする
- 地域と年齢（18歳以上）のみを設定
- Smart Targetingを有効化

**詳細ターゲティングが有効なケース:**
- 地域限定ビジネス
- 高額商品（「高い購買力」ディメンション使用）
- ニッチ市場
- テスト初期の特定セグメント検証

### 4.6 ハイブリッドアプローチ（推奨）

```
広告グループ1: ブロードターゲティング（メイン配信用）
広告グループ2: インタレストベース（特定カテゴリ）
広告グループ3: カスタムオーディエンス（リターゲティング）
広告グループ4: 類似オーディエンス（拡張用）
```

### 4.7 Smart Targeting / Auto Targeting

| 機能 | 説明 |
|------|------|
| **Smart Targeting** | 設定した範囲を超えて、目標達成に貢献する可能性の高いユーザーに自動拡張 |
| **Targeting Recommendation** | キャンペーン目標に貢献しやすいオーディエンスの興味関心・行動カテゴリを推奨 |

### 4.8 デバイスターゲティング

| 項目 | 設定内容 |
|------|----------|
| **OS** | iOS / Android |
| **OSバージョン** | 特定バージョン以上を指定可能 |
| **デバイスモデル** | 特定の端末モデルを指定可能 |
| **接続タイプ** | Wi-Fi / モバイルデータ |
| **キャリア** | 特定の通信キャリアを指定可能 |
| **デバイス価格** | 端末の価格帯で指定可能 |

## 5. 入札戦略

### 5.1 入札タイプ

| 入札タイプ | 説明 |
|-----------|------|
| **CPM** | 1,000インプレッションあたりのコスト |
| **oCPM** | 最適化されたCPM（TikTokのメイン課金モデル） |
| **CPC** | クリックあたりのコスト |
| **CPV** | 動画再生あたりのコスト |

### 5.2 入札戦略（Bid Strategy）

#### Maximum Delivery（最大配信 / Lowest Cost）

| 項目 | 詳細 |
|------|------|
| **タイプ** | 支出ベース戦略 |
| **目標** | 予算内で最大数のコンバージョンを獲得 |
| **CPA指定** | 不要（TikTokが自動最適化） |
| **適した場面** | 新規キャンペーン、学習フェーズ、コンバージョン数最大化が最優先 |

#### Cost Cap（コスト上限）

| 項目 | 詳細 |
|------|------|
| **タイプ** | 目標ベース戦略 |
| **目標** | 指定したCPA目標にできるだけ近い単価でコンバージョン獲得 |
| **CPA指定** | 必要（目標CPAを入力） |
| **推奨予算** | 日予算 = Cost Cap x 50（理想）/ x 5（最低） |
| **適した場面** | CPA目標が明確、スケーリング段階 |

> **低予算消化時の対策**: Cost Capで予算消化が進まない場合、CPA入札を**20%以上引き上げる**か、Maximum Deliveryに切り替え。

#### Hybrid Bidding（ハイブリッド入札）

Maximum DeliveryとCost Capをアクティブ配信中に切り替え可能。初期テスト（Maximum Delivery） → 安定運用（Cost Cap）への移行に有効。

### 5.3 入札戦略の進化パターン（推奨）

```
フェーズ1（初期）: Maximum Delivery → データ収集
      ↓
フェーズ2（安定化）: Cost Cap → ベースラインCPAに基づく目標設定
      ↓
フェーズ3（最適化）: Target ROAS → ROAS目標に基づく価値最大化
```

### 5.4 入札設定のポイント

- **Cost Cap設定時**: 許容できる最大CPAを設定
- **ROAS設定時**: 許容できる最低ROASを設定
- **コールドスタート時**: 目標CPAの20-30%高めに設定し、段階的に下げる

### 5.5 最適化イベント（Optimization Goal）

| 最適化目標 | 説明 | 対応する広告目的 |
|-----------|------|----------------|
| **インプレッション** | 広告の表示回数を最大化 | Reach |
| **リーチ** | ユニークユーザーへのリーチを最大化 | Reach |
| **動画再生数** | 再生数（2秒/6秒/完全視聴）を最適化 | Video Views |
| **クリック** | クリック数を最大化 | Traffic |
| **ランディングページビュー** | LP読み込み完了数を最適化 | Traffic |
| **コンバージョン** | コンバージョンイベント数を最適化 | Sales, Lead Generation |
| **バリュー** | コンバージョンの金銭的価値を最適化 | Sales |
| **アプリインストール** | インストール数を最適化 | App Promotion |
| **アプリ内イベント** | アプリ内イベント数を最適化 | App Promotion |
| **リード** | リードフォーム送信数を最適化 | Lead Generation |

> **注意**: 最適化目標とコンバージョンイベントは広告グループ作成後に変更不可。

### 5.6 予算の最低要件

| 予算タイプ | CBO | ABO |
|-----------|-----|-----|
| 最低日予算 | $50（約7,500円） | $20（約3,000円） |

### 5.7 初期予算の計算式

```
最低日予算 = 目標CPA x 20
推奨日予算 = 目標CPA x 50（より早い学習完了のため）
```

| 目標CPA | 最低日予算 | 推奨日予算 | 週間予算 |
|---------|----------|----------|---------|
| ¥500 | ¥10,000/日 | ¥25,000/日 | ¥70,000-175,000 |
| ¥1,000 | ¥20,000/日 | ¥50,000/日 | ¥140,000-350,000 |
| ¥3,000 | ¥60,000/日 | ¥150,000/日 | ¥420,000-1,050,000 |
| ¥5,000 | ¥100,000/日 | ¥250,000/日 | ¥700,000-1,750,000 |

## 6. バリューベース最適化（VBO）

VBOは高価値ユーザー（より大きな購入やアプリ内アクションを行う可能性が高いユーザー）にリーチするための入札戦略。

| 入札戦略 | 説明 | 対応キャンペーン |
|---------|------|----------------|
| **Highest Value（最高バリュー）** | ROAS指定なしで最大のバリューを追求。VBOのデフォルト | 手動 / Smart+ |
| **Target ROAS（目標ROAS）** | 目標ROASを指定。Day 0 ROASまたはDay 7 ROASから選択 | Smart+のみ |
| **Minimum ROAS（最低ROAS）** | 最低限のROASを設定（Web VBO向け） | 手動 |

### VBO運用のベストプラクティス

- 過去7日間の実績ROASの**80%程度**をMinimum ROAS目標に設定
- 最低**7日間**はキャンペーンを稼働させて信頼性のあるデータを収集
- 最初の**3日間は調整を避ける**（ラーニングフェーズの安定化）

## 7. Smart+ キャンペーン

### 概要

Smart+キャンペーンはTikTokのAIが設定、ターゲティング、最適化、クリエイティブを自動化する統合ソリューション。標準キャンペーンと比較して**最大80%のケースで優れたパフォーマンス**を発揮。

### 対応キャンペーンタイプ

| タイプ | 説明 | パフォーマンス |
|--------|------|-------------|
| **Smart+ Web Campaigns** | ウェブコンバージョンの自動最適化 | CPA約36%低下（カタログ広告の場合） |
| **Smart+ App Campaigns** | アプリインストール・イベントの自動最適化 | - |
| **Smart+ Catalog Campaigns** | 商品カタログベースの自動配信（最低4商品、最大2,000万商品） | - |
| **Smart+ Lead Generation** | リード獲得の最大化 | 実績例: リード82%増加、CPL 75%低下 |

### 自動最適化の対象モジュール

| モジュール | 自動化内容 |
|-----------|-----------|
| **ターゲティング** | AIが最適なオーディエンスを自動特定 |
| **予算** | CBOによる自動予算配分 |
| **クリエイティブ** | Symphony Automationによる自動最適化・生成・リフレッシュ |
| **入札** | Cost Cap / Maximum Delivery / Target ROASの自動入札 |

### 2025年アップグレード: モジュール別コントロール

2025年のアップデートにより、「ブラックボックス」型から**モジュール単位で自動化レベルを調整可能**な統合セットアップに進化。

- ターゲティング: 自動 or 手動
- 予算: 自動 or 手動
- クリエイティブ: 自動 or 手動
- 各モジュールの自動化レベルを広告主が個別にカスタマイズ可能

### Smart+ vs マニュアルの使い分け

**Smart+を使うべきケース:**
- 月額$1,500以上の広告予算がある
- 運用リソースが限られている
- ブロードターゲティングが適する商材
- カタログベースのEC広告

**マニュアルを使うべきケース:**
- ニッチなターゲティングが必要
- 詳細なクリエイティブ制御が必要
- テスト段階で変数を個別に検証したい
- 予算が限られている（月額$1,500未満）

### Smart+ 導入ステップ（推奨）

1. 月額$1,500以上のキャンペーン1つでSmart+をテスト開始
2. 既存のマニュアルキャンペーンと**14日間並行運用**
3. 結果を比較（CPA、ROAS、コンバージョン数）
4. Smart+が**15-20%以上**の改善を示した場合、追加キャンペーンを移行
5. 学習フェーズ: 7-10日（50コンバージョン以上）、14日目まで改善、15-30日で安定

## 8. ラーニングフェーズ

### 完了条件

- **50コンバージョン**を達成する、または
- **7日間**が経過する（いずれか早い方）

### 予算要件

- App Install目的: 広告グループ予算は**目標CPAの50倍**以上
- AEO / Conversions目的: 広告グループ予算は**目標CPAの20倍**以上

### 早期脱出のポイント

1. **十分な予算を設定**: 目標CPAの20倍以上の日予算
2. **ブロードターゲティングを使用**: 狭いターゲティングはデータ収集を遅らせる
3. **複数クリエイティブを用意**: アルゴリズムに多くのバリエーションを提供
4. **最適化イベントの選択**: コンバージョンが少ない場合は上位ファネルのイベントで最適化
5. **変更を控える**: 7日間または50コンバージョンまで設定変更しない

### 停滞する原因と対策

| 原因 | 対策 |
|------|------|
| 予算が少なすぎる | 目標CPAの20倍以上に増額 |
| 頻繁な設定変更 | 1回の変更後7日間待つ |
| ターゲットが狭すぎる | ブロードターゲティングに変更 |
| 最適化イベントが稀 | より頻度の高いイベントに変更 |
| クリエイティブが少ない | 3-5本のバリエーションを追加 |

### 注意事項

- 小幅な予算変更（例: $100 → $110）は学習フェーズをリセットしない
- 大幅な変更（例: $100 → $300）は学習フェーズを再開させる
- ステータスが「Learning」から「Active」に変わったら学習完了

## 9. テスト・スケーリング設計

### テスト用 vs スケール用キャンペーンの分離

#### テスト用キャンペーン

- **目的**: 新しいクリエイティブ、ターゲティング、入札戦略の検証
- **予算**: 全体予算の20-30%
- **構成**: ABO（変数を個別に制御）
- **期間**: 最低7日間、50コンバージョンを目安に評価
- **広告グループ数**: 3-5個（各グループに2-3本のクリエイティブ）

#### スケール用キャンペーン

- **目的**: 検証済みの勝ちパターンを拡大
- **予算**: 全体予算の70-80%
- **構成**: CBO（自動最適化）
- **スケーリング**: 20%ルールに基づく段階的な予算増加

#### テスト → スケールのサイクル

```
Week 1-2: テスト
  - ABO設定で複数のオーディエンス/クリエイティブをテスト
  - 各広告グループに均等予算を配分
  - データを蓄積（50コンバージョン目標）

Week 2-3: 検証
  - CPA、ROAS、CTR、コンバージョン率を比較
  - 統計的有意差を確認
  - 勝者パターンを特定

Week 3以降: スケール
  - 勝者パターンをCBOキャンペーンに移行
  - 20%ルールで段階的に予算拡大
  - 新たなテストを並行して開始
```

### クリエイティブテスト

#### Hook-Body-CTAフレームワーク

| 要素 | 役割 | テストポイント |
|------|------|-------------|
| **Hook** | 最初の2-3秒で注意を引く | 問いかけ、衝撃的な事実、ビフォーアフター、ストーリー導入 |
| **Body** | メッセージ・価値提案を伝える | 商品デモ、ベネフィット説明、テスティモニアル |
| **CTA** | 次のアクションを促す | テキストCTA、音声CTA、限定オファー |

**モジュラーテスト方式:**
```
例: フック3種 x ボディ2種 x CTA2種 = 12パターン
```

#### 勝者の判定基準

- CPA **25%低下** または CTR **40%向上**

### A/Bテスト（Split Test）

- 2つの広告グループを直接比較
- テスト可能な変数: ターゲティング、クリエイティブ、入札・最適化
- **1回のテストで変更する変数は1つだけ**
- 最低**7日間**のテスト期間

### スケーリング方法

#### 垂直スケーリング（20%ルール）

**最重要ルール: 日予算の増加は24時間ごとに最大20%まで。**

```
Day 1: ¥10,000/日
Day 2: ¥12,000/日（+20%）
Day 3: ¥14,400/日（+20%）
Day 4: ¥17,280/日（+20%）
Day 5: ¥20,736/日（+20%）
```

- 急激な増加はアルゴリズムが低品質トラフィックに入札する原因に
- 50%以上の削減は学習フェーズをリセット

#### 水平スケーリング

1. **新しいオーディエンスセグメントの追加** - 類似オーディエンスの拡張、広いインタレストカテゴリのテスト
2. **新しいクリエイティブの追加** - 2025年はクリエイティブ駆動型スケーリングが最も効果的
3. **新しい地域・配置の追加** - 対象地域の拡大、Pangle等の追加配置

### Events API（サーバーサイド計測）

サーバーからTikTokに直接コンバージョンデータを送信する仕組み。

**Pixel + Events API併用の効果:**
- イベント捕捉数が**13%向上**
- CPAが**15%低下**
- トラッキングされるコンバージョンが**最大60%増加**

**推奨構成**: PixelとEvents APIの**併用**。ブラウザ側のリアルタイムインタラクションをPixelで捕捉しつつ、サーバーサイドからの確実なイベント送信をEvents APIで担保。

|-----------|------|

| 14日 | 可 |
| 28日 | 可 |

| ウィンドウ | 設定 |
|-----------|------|

| 7日 | 可 |

#### エンゲージドビュースルーアトリビューション（EVTA）

広告を**6秒以上視聴**（6秒未満の動画は全編視聴）したがクリックしなかった場合のコンバージョン計測。

> **EVTAの意義**: 動画プラットフォームであるTikTokでは、「十分な視聴」がコンバージョンに寄与するケースが多い。EVTAは「エンゲージされた視聴」による間接的なコンバージョン貢献を可視化する。

## 14. シーズナリティ

### ピーク期間の影響

- ブラックフライデー・年末商戦期はCPMが**30-50%上昇**
- クリスマス前7-10日が最も広告費用が高い期間
- TikTokは他プラットフォームと比較して祝日期間でもCPM効率が比較的良好

### シーズナリティ対応の戦略

| 時期 | 戦略 |
|------|------|
| **ピーク2-3週間前** | テストキャンペーンで勝ちクリエイティブを確定。予算を段階的に増加開始（1日30%ずつ） |
| **ピーク1週間前** | スケールキャンペーンの予算を最大化。新規クリエイティブの投入は控える |
| **ピーク期間中** | 勝ちクリエイティブに集中投資。日予算のキャップを設定してROAS管理 |
| **ピーク後（Q5期間）** | CPMが下がるタイミングで積極投資。ギフトカード・新年関連の訴求 |

### コスト効率の良い時期

- **10月中旬 - 11月中旬**: 購買意欲が上昇中だが、本格的な価格上昇前
- **Q5（1月初旬）**: ホリデー後のCPM低下期間で高効率な配信が可能
- **11月中旬（Jump Start Week）**: TikTokのCPMが**13%低下**しつつ購買意欲が高い
