# 楽天市場 戦略・設計スキル

楽天市場への出品設計と広告運用の戦略立案から仕様書作成までを支援する。出品設計（商品ページ最適化・楽天SEO・クリエイティブ）と広告設計（RPP/クーポンアドバンス/TDA/CPA/純広告）の両方をカバーする。

## ワークフロー

```
Step 1: 情報収集        → ビジネス概要・商品情報・目的・予算をヒアリング
Step 2: 戦略選択        → 出品設計 / 広告設計 / 統合設計を判定し、最適な手法を選定
Step 3: 設計・構成      → 商品ページ構成・広告キャンペーン構造・クリエイティブを設計
Step 4: 仕様書作成      → 出品仕様書・広告運用指示書を出力
```

---

## Step 1: 情報収集

`contexts/` ディレクトリに既存の情報があれば最初に確認する。

### 必須情報

| 項目 | 内容 |
|------|------|
| ビジネス概要 | 業種・事業モデル・自社の強み |
| 商品情報 | 商品カテゴリ・価格帯・差別化ポイント |
| 目的 | 新規出店 / リスティング改善 / 広告開始 / 広告改善 / 統合設計 |
| 予算 | 月間広告予算・初期投資可能額 |
| ターゲット | 想定顧客層・購買動機 |
| KPI | 目標ROAS / CVR / 月商目標 |

### 状況把握（あれば確認）

| 項目 | 内容 |
|------|------|
| 出店状況 | 出店済み/未出店・プラン・契約年数 |
| 既存実績 | 現在の月商・CVR・レビュー数・検索順位 |
| 広告運用状況 | 現在利用中の広告メニュー・ROAS |
| SKU対応 | SKUプロジェクト移行済み/未対応 |
| 配送体制 | 自社配送/RSL利用・最強翌日配送対応有無 |

## Step 2: 戦略選択

ユーザーの目的に応じて、以下の3つの戦略パスから選択する。

### A. 出品設計（商品ページ最適化）

商品ページの構成・最適化・楽天SEO対策を行うパス。

| 設計領域 | 詳細リファレンス | 読むタイミング |
|---------|----------------|--------------|
| アカウント・出店設定 | [account-setup.md](references/account-setup.md) | 出店プラン選定・審査・RMS設定・店舗デザインを設計する時 |
| 商品ページ設計 | [listing-design.md](references/listing-design.md) | 商品名・キャッチコピー・説明文・画像・SKU・楽天SEOを設計する時 |

**出品設計の活用シーン：**
- 新規商品の楽天市場出品を設計する
- 既存リスティングのCVR改善
- 楽天SEO（検索アルゴリズム）対策
- 商品画像ガイドライン対応

### B. 広告設計（楽天広告）

広告メニューの選定・キャンペーン構造・運用設計を行うパス。

| 広告タイプ | 特徴 | 最低予算 | 詳細リファレンス |
|-----------|------|---------|----------------|
| **RPP広告** | 検索連動型。売上直結の主力広告 | 月5,000円 | [ad-types.md](references/ad-types.md) |
| **クーポンアドバンス** | AI自動マッチング+クーポン付与 | 月5,000円 | [ad-types.md](references/ad-types.md) |
| **TDA広告** | ターゲティングディスプレイ。認知〜リターゲティング | 月50,000円 | [ad-types.md](references/ad-types.md) |
| **楽天CPA広告** | 成果報酬型（売上の20%）。リスクゼロ | なし | [ad-types.md](references/ad-types.md) |
| **楽天市場広告（純広告）** | 枠購入型。大量インプレッション獲得 | 約40,000円 | [ad-types.md](references/ad-types.md) |

**広告戦略・運用最適化：**

| 設計領域 | 詳細リファレンス | 読むタイミング |
|---------|----------------|--------------|
| 広告運用戦略 | [ad-strategy.md](references/ad-strategy.md) | フルファネル設計・予算配分・イベント連動・ROAS改善のPDCAを設計する時 |

**ビジネス目標 → 推奨広告タイプ：**

| 目標 | 第1選択 | 第2選択 | 第3選択 |
|------|---------|---------|---------|
| 売上直結（既存商品） | RPP（キーワードCPC） | クーポンアドバンス | CPA広告 |
| 新規顧客獲得 | RPP（カテゴリKW） | クーポンアドバンス | TDA（セグメント） |
| ブランド認知向上 | TDA（会員属性） | 楽天市場広告（純広告） | 楽天ROOM |
| イベント時の爆発的集客 | RPP（CPC増額） | 純広告（イベント枠） | クーポンアドバンス |
| リピート促進 | TDA（自店舗購入者） | R-Mail | クーポンアドバンス |
| 新商品ローンチ | RPP（ロングテールKW） | CPA広告 | TDA（カテゴリ購入者） |

**予算別推奨構成：**

| 月予算 | 推奨構成 | 配分 |
|--------|---------|------|
| 〜5万円 | RPP + CPA | RPP 80% / R-Mail 20% / CPA ON |
| 5〜10万円 | RPP + CA + CPA | RPP 60% / CA 20% / R-Mail 10% / 予備 10% |
| 10〜30万円 | RPP + CA + TDA | RPP 50% / CA 15% / TDA 17% / R-Mail 5% / 予備 13% |
| 30万円〜 | フルファネル | RPP 40% / CA 12% / TDA 16% / 純広告 16% / R-Mail 4% / 予備 12% |

### C. 統合設計

出品設計と広告設計を同時に行う。新規楽天出店や大幅リニューアル時に推奨。
Step 2A → Step 2B の順で設計し、出品基盤を整えてから広告を構築する。

## Step 3: 設計・構成

### 出品設計の場合

1. **アカウント・基盤設計**: 出店プラン・配送方法・SKU対応の方針決定
2. **商品ページ設計**: 商品名（127文字）・キャッチコピー（PC/モバイル各87文字）・説明文・画像の設計
3. **楽天SEO対策**: キーワード配置・商品属性登録・セマンティックサーチ対応
4. **レビュー獲得計画**: フォローメール・レビュー特典・同梱カードの設計

### 広告設計の場合

1. **広告メニュー選定**: 目的・予算に応じた広告タイプの組み合わせ
2. **RPP広告設計**: キーワード3層構造（ビッグ/ミドル/ロングテール）・CPC設定・除外設定
3. **イベント連動設計**: スーパーSALE・マラソン・5と0のつく日の広告強化計画
4. **運用PDCA設計**: 週次/月次チェック項目・ROAS改善アクション

### 命名規則

広告キャンペーン・レポートの命名規則を統一する。

```
[商品/ブランド]_[広告タイプ]_[ターゲティング]_[日付]

例:
有機抹茶_RPP_KeywordExact_BrandDefense_202602
有機抹茶_RPP_Auto_Discovery_202602
有機抹茶_CA_AutoTarget_202602
有機抹茶_TDA_Retarget_Purchase30d_202602
```

## Step 4: 仕様書作成

以下の成果物をMarkdown形式で出力する。対象パスに応じて必要なものを選択する。

### 出品仕様書（出品設計の場合）

1. **商品ページ設計書**
   - 商品名（127文字以内、冒頭19文字にメインKW）
   - キャッチコピー（PC用・モバイル用各87文字、重要KWは両方に）
   - 商品説明文（PC用・スマホ用を別途設計）
   - 商品属性情報（必須属性100%登録）
   - SKU設計（シングル/マルチSKU、バリエーション統合方針）

2. **クリエイティブ仕様書**
   - サムネイル画像（白背景、テキスト20%以内、枠線なし）
   - 商品説明画像構成（10枚程度：キャッチ→悩み共感→解決策→特徴→使用シーン→比較→レビュー→スペック→FAQ→CTA）
   - スマホ最適化チェックリスト

3. **運用設計書**
   - 配送方針（自社/RSL、最強翌日配送対応）
   - レビュー獲得計画
   - 楽天SEOチェックリスト

### 広告運用仕様書（広告設計の場合）

1. **戦略概要**
   - 目的・KPI・予算
   - ファネル設計と広告タイプの役割分担
   - 予算配分

2. **広告設計書**
   - RPP広告: キーワードリスト・CPC設定・除外商品/キーワード
   - クーポンアドバンス: 対象商品・値引き率設定
   - TDA広告: セグメント設計・バナー仕様（4サイズ必須）
   - CPA広告: ON/OFF判断基準
   - 純広告: 枠選定・予約スケジュール

3. **イベント連動計画**
   - スーパーSALE対応タイムライン（2ヶ月前から）
   - お買い物マラソン対応
   - 5と0のつく日の施策
   - 月別シーズナルカレンダー

4. **運用・計測設計**
   - 週次チェック項目（ROAS/CTR/CVR/CPC消化額）
   - ROAS改善アクション基準
   - A/Bテスト計画
