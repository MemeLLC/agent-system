## 1. 商品登録の必須項目と推奨項目

### 必須項目

| 項目 | 説明 | 備考 |
|------|------|------|
| **商品コード** | JAN/EAN/UPC/ISBN | ブランド登録済みの場合は免除申請可能 |
| **商品名（タイトル）** | 商品の正式名称 | 全角200文字以内（カテゴリにより異なる） |
| **ブランド名** | メーカー・ブランドの名称 | ブランド登録推奨 |
| **メーカー名** | 製造元の名称 | - |
| **商品画像** | メイン画像1枚（必須）+ サブ画像（最大8枚） | メイン画像は白背景必須 |
| **価格** | 販売価格 | 税込価格で設定 |
| **在庫数** | 現在の在庫数量 | FBAの場合は納品数量 |
| **コンディション** | 新品/中古（各グレード） | - |
| **カテゴリ** | 商品が属するカテゴリ | ブラウズノードで指定 |

### 推奨項目（SEO・CVRに大きく影響）

| 項目 | 説明 | 備考 |
|------|------|------|
| **商品仕様（箇条書き）** | 商品の特徴を最大5項目 | 1項目あたり500文字以内推奨 |
| **商品説明文** | 商品の詳細説明 | 2,000文字以内 |
| **検索キーワード** | バックエンドキーワード | 一般: 500バイト / ファッション: 250バイト |
| **寸法・重量** | 商品のサイズと重さ | FBA手数料計算にも影響 |
| **素材・成分** | 使用素材や成分情報 | カテゴリによっては必須 |

## 2. JAN/EANコード

- **JAN（Japanese Article Number）**: 日本国内で使用される商品識別バーコード。GTINの一種
- **新規商品**: JANコードの取得が必要（GS1 Japanで申請、初期登録料は事業規模により約1万円〜）
- **ブランド登録済みの場合**: JAN免除の申請が可能
- **既存商品への相乗り出品**: 既存のJAN/ASINを使用

## 3. タイトル最適化

### 基本ルール（2025年1月21日〜新要件適用）

| 項目 | 内容 |
|------|------|
| **最大文字数** | 全角200文字以内（スペース含む） |
| **推奨文字数** | **60〜80文字**（モバイル表示を考慮） |
| **最優先配置** | 重要キーワードは**冒頭30〜40文字以内**に |
| **スペース** | 必ず**半角スペース**で区切る（全角スペース禁止） |
| **同一単語の繰り返し** | 禁止（前置詞・冠詞・接続詞を除く） |

### カテゴリ別文字数制限

| カテゴリ | 文字数制限 |
|---------|----------|
| 一般カテゴリ | 全角200文字以内 |
| カメラ | 全角50文字以内 |
| おもちゃ | 全角50文字以内 |
| ジュエリー | 全角50文字以内 |

> カテゴリごとに独自のルールがある場合がある。セラーセントラルのスタイルガイドで必ず確認すること。

### 推奨タイトル構成式

```
[ブランド名] [商品名/表示名] [仕様（色、サイズ、タイプ等）] [型番] [内容量/重量]
```

**カテゴリ別の構成例:**

| カテゴリ | 構成 |
|---------|------|
| 一般 | ブランド名 + 商品名 + 仕様 + 型番 |
| カメラ | メーカー名（英語）+ 一般名 + 商品名 + 特徴 + カラー + 型番 |
| アパレル | ブランド名 + 対象（メンズ/レディース）+ 商品種別 + 素材 + カラー |
| 食品 | ブランド名 + 商品名 + フレーバー + 内容量 + 数量 |

### 禁止事項

| カテゴリ | 禁止内容 |
|---------|---------|
| **プロモーション表現** | 送料無料、ベストセラー、限定、セール、新商品、No.1 |
| **特殊文字** | ★ ♪ 【 】 { } ^ _ ? ! $ ¬ ¦（ブランド名に含まれる場合は例外） |
| **誤解を招く表現** | 関連のない商標名、誇大表現（100%効果がある等） |
| **商品コード** | JAN/ASIN/SKUをタイトルに含めない |
| **主観的表現** | 最高、最安、すごい等 |

### タイトル最適化チェックリスト

- [ ] 最重要キーワードが冒頭30文字以内に配置されているか
- [ ] モバイル表示（約60文字）で意味が通じるか
- [ ] 半角スペースで区切っているか（全角スペースは不可）
- [ ] 同一キーワードの重複がないか
- [ ] 禁止表現・特殊文字を使用していないか
- [ ] 200文字以内に収まっているか（カテゴリ制限も確認）
- [ ] 自然な日本語として読みやすいか

## 4. 箇条書き（Bullet Points）最適化

### 基本ルール

| 項目 | 内容 |
|------|------|
| **項目数** | 最大**5項目** |
| **1項目あたりの文字数** | 最大500文字（超過でエラー発生） |
| **推奨文字数** | 5項目合計で**1,000文字未満** |
| **表示位置（PC）** | 商品画像の右側 |
| **表示位置（モバイル）** | ページ中間あたり（**最初の3項目のみ表示**、残りはタップで展開） |

> モバイルでは最初の3項目のみが初期表示される。最も重要な情報を上位3項目に配置すること。

### A10アルゴリズムとの関係

- A10は箇条書きの最初の**1,000文字**のみをインデックス・ランキングに使用
- モバイルでは**400文字**で切り捨てられる
- 検索キーワードのすべてが検索アルゴリズムにインデックスされるわけではない

### 構成テンプレート

```
【見出しキーワード】具体的な特徴の説明 → 購入者にとってのメリット
```

### 5要素の推奨順

| 順番 | 要素 | 内容 |
|------|------|------|
| 1 | **商品の最大の特徴・差別化ポイント** | 最も訴求力のある特徴 |
| 2 | **使用方法・対象シーン** | どんな場面で役立つか |
| 3 | **素材・品質** | 素材、製法、品質へのこだわり |
| 4 | **サイズ・仕様** | 具体的な数値情報 |
| 5 | **保証・付属品・注意事項** | アフターサポートや注意点 |

### 禁止事項

- 価格やプロモーション情報の記載
- 配送・返品に関する情報
- 他社商品との直接比較（「○○社より優れている」等）
- HTMLタグの使用
- 外部リンク・連絡先情報

### 箇条書き最適化チェックリスト

- [ ] 最重要情報が上位3項目に配置されているか
- [ ] 各項目が100〜250文字で簡潔にまとまっているか
- [ ] 「特徴 → メリット」の構成になっているか
- [ ] 関連キーワードが自然に含まれているか
- [ ] 数値データ（サイズ、容量、時間等）が具体的か
- [ ] 句点（。）を使わず簡潔な表現になっているか
- [ ] モバイル表示で最初の3項目だけでも魅力が伝わるか

## 5. 商品説明文

### 基本ルール

| 項目 | 内容 |
|------|------|
| **最大文字数** | **2,000文字** |
| **HTML使用** | 基本的なHTMLタグ使用可能（`<br>`, `<b>`, `<ul>`, `<li>`等） |
| **表示位置** | 商品詳細ページの下部 |

> A+コンテンツを設定した場合、通常の商品説明文は表示されなくなる。A+コンテンツが優先表示される。

### 効果的な構成

| 順番 | セクション | 内容 |
|------|-----------|------|
| 1 | **フック** | 購入者の悩みや欲求に訴えかける導入文 |
| 2 | **商品の詳細説明** | 箇条書きでは伝えきれない詳細な特徴や背景 |
| 3 | **使用シーン** | 具体的な利用場面やターゲットユーザーの描写 |
| 4 | **素材・製法** | 品質へのこだわりや製造プロセス |
| 5 | **セット内容** | 同梱物の詳細 |
| 6 | **注意事項** | サイズ選びのポイントや使用上の注意 |

### SEO対策

- タイトルや箇条書きに含められなかったキーワードを自然に織り込む
- ロングテールキーワード（3語以上の組み合わせ）を意識する
- キーワードの詰め込み（スタッフィング）は避ける
- 2次キーワード・関連キーワードを散りばめる

## 6. バックエンドキーワード

### 基本仕様

| 項目 | 内容 |
|------|------|
| **設定場所** | セラーセントラル > 在庫 > 商品編集 > 「キーワード」タブ |
| **最大バイト数（一般）** | **500バイト**（約166全角文字） |
| **最大バイト数（ファッション）** | **250バイト**（約83全角文字） |
| **区切り方** | **半角スペース**で区切る（カンマ・全角スペースは不可） |
| **表示** | 商品ページには表示されない（検索アルゴリズム用） |

> 文字数制限を超過すると、キーワードフィールド**全体が無効化**される。必ず制限内に収めること。

### 入力すべきキーワード

- タイトル・箇条書き・説明文に含めなかったキーワード
- 同義語・類義語（例: スマホケース → スマートフォンケース、携帯カバー）
- 表記ゆれ（ひらがな・カタカナ・漢字の異なる表記）
- 略語・俗称
- 英語表記（日本語商品でも英語で検索されるキーワード）

### 入力してはいけないキーワード

| 禁止カテゴリ | 具体例 |
|-------------|--------|
| **他社ブランド名** | Apple、Nike、Sony等（商標侵害のリスク） |
| **ASIN・商品コード** | B0xxxxxxxxx、4900xxxxxxxxx |
| **プロモーション用語** | 新商品、セール中、限定品、送料無料 |
| **主観的表現** | 最高、最安、すごい、人気 |
| **暴力的・不適切な表現** | - |

### 最適化テクニック

1. **Amazonサジェスト活用**: 検索窓に関連ワードを入力し、候補キーワードを収集
2. **Brand Analytics**: ブランド登録済みなら検索頻度ランキングを活用
3. **競合分析**: 上位商品のタイトル・箇条書きからキーワードを抽出
4. **Google Trends/キーワードプランナー**: 外部ツールでの需要調査
5. **重複排除**: タイトルや箇条書きに既に含まれているキーワードは不要（Amazonは全フィールドを統合してインデックスする）

### プラチナキーワード

- Amazonが大口出品者に個別に案内するフィールド
- 特別なキャンペーン時にのみ使用される
- **Amazonから明示的な指示がない限り、入力しない**のが安全

## 7. A+コンテンツ（商品紹介コンテンツ）

### 3つのコンテンツタイプ

| タイプ | 利用条件 | モジュール数 | 特徴 |
|--------|---------|-------------|------|
| **ベーシック** | 大口出品アカウント | 最大7個（17種類から選択） | テキスト+画像、比較表 |
| **プレミアムA+** | ブランド登録 + 5件以上の承認済みA+ + 全ASINにブランドストーリー設定 | 最大7個（19種類から選択） | 動画、カルーセル、Q&Aモジュール |
| **ブランドストーリー** | ブランド登録 | カルーセル形式 | ブランドの物語、商品/ストアページへのリンク |

### モジュール別画像サイズ一覧

| モジュール | 推奨画像サイズ（px） |
|-----------|---------------------|
| テキスト付き標準画像ヘッダー | 970 x 600 |
| 標準マルチ画像モジュールA | 300 x 300 |
| 企業ロゴ | 600 x 180 |
| 商品比較表 | 150 x 300 |
| 4つの画像+テキスト（クワドラント） | 135 x 135 |
| ライトテキストオーバーレイ | 970 x 300 |
| 3つの画像+テキスト | 300 x 300 |
| 4つの画像+テキスト | 220 x 220 |
| ダークテキストオーバーレイ | 970 x 300 |
| シングル画像+ハイライト | 300 x 300 |
| シングル画像+スペック | 300 x 300 |
| シングル右画像 | 300 x 300 |
| シングル左画像 | 300 x 300 |
| シングル+サイドバー | 300 x 400 / 350 x 175 |
| **プレミアムA+コンテンツ** | **1464 x 600** |

### 画像の技術要件

| 項目 | 仕様 |
|------|------|
| ファイル形式 | JPG / PNG / BMP |
| ファイルサイズ | **2MB未満** |
| 解像度 | **72dpi以上** |
| カラーモード | sRGB |

### A+コンテンツ作成手順

1. セラーセントラルの「広告」>「A+コンテンツマネージャー」にアクセス
2. 「コンテンツの作成を開始する」をクリック
3. コンテンツタイプを選択（ベーシック/プレミアム/ブランドストーリー）
4. コンテンツ名と言語を設定
5. モジュールを選択し、画像とテキストを追加（最大7モジュール）
6. 対象商品のASINを入力
7. デスクトップ・モバイルでプレビュー確認
8. 「承認用に送信」で提出
9. Amazon審査（通常7営業日以内）

### ベストプラクティス

- **CVRへの効果**: 適切に作成されたA+コンテンツでCVRが最大**1.5倍**向上する事例あり
- **モバイル最適化**: スマートフォンでの表示を必ずプレビューで確認
- **ブランドの一貫性**: カラー、フォント、トーンを統一する
- **比較表の活用**: 自社商品間の比較表は購入検討を後押しする

### A+コンテンツの禁止事項

- 外部リンク・連絡先情報
- 保証に関する主張
- 価格・プロモーション情報
- 著作権侵害のコンテンツ

## 8. バリエーション出品（親子ASIN）

### バリエーションタイプ

| タイプ | 例 |
|--------|-----|
| **サイズ** | S / M / L / XL |
| **カラー** | レッド / ブルー / ブラック |
| **サイズ x カラー** | S-レッド / M-ブルー（2軸組み合わせ） |
| **フレーバー** | バニラ / チョコレート / ストロベリー |
| **パターン** | ストライプ / ドット / ソリッド |

### 親子ASIN構造

```
親ASIN（架空の商品 = 販売対象外）
├── 子ASIN 1（レッド / Sサイズ）← 実際に販売される商品
├── 子ASIN 2（レッド / Mサイズ）
├── 子ASIN 3（ブルー / Sサイズ）
└── 子ASIN 4（ブルー / Mサイズ）
```

> 親ASINは実商品ではない。子商品をまとめるための「入れ物」であり、直接購入されることはない。

### 新規商品でのバリエーション登録手順

1. セラーセントラルの「カタログ」>「商品登録」を選択
2. 「Amazonで販売されていない商品を追加します」を選択
3. 適切なカテゴリを選択
4. 「バリエーション」タブで該当するバリエーションタイプを選択
5. 各子商品のJANコード・SKU・価格・在庫数を入力
6. 各子商品の画像をアップロード
7. 登録を完了

### 既存商品のバリエーション統合

1. 新しい親商品ページを作成
2. バリエーション設定で「ASIN」を商品コードタイプとして選択
3. 統合したい既存商品のASINを入力
4. システムが自動的に既存商品情報を新バリエーション構造に統合

> 大量バリエーションの場合は、カテゴリ別の在庫ファイル（テンプレート）をダウンロードし、Excelで一括入力してアップロードする方法が効率的。

### バリエーション出品のメリット

- レビューが親ASINに集約され、各子商品のページにも反映される
- 1つのページ内で選択できるため、購入者の離脱を防げる
- 検索結果での表示力が向上する

## 9. A10アルゴリズム最適化

### A10アルゴリズムとは

Amazonの商品検索ランキングアルゴリズム（2021年にA9からアップデート）。「購入につながる商品の表示」に完全特化している。

### 主要ランキング要素

#### パフォーマンス指標（最重要）

| 要素 | 影響度 | 説明 |
|------|--------|------|
| 販売履歴・販売速度 | 非常に高い | 直近の販売実績が継続的に高いほど有利 |
| コンバージョン率（CVR） | 非常に高い | ページ訪問者のうち購入に至った割合 |
| クリック率（CTR） | 高い | 検索結果からのクリック率 |
| レビュー数・評価 | 高い | レビューの量と質（星の数） |

#### 関連性（Relevance）

| 要素 | 説明 |
|------|------|
| タイトルのキーワード | 検索クエリとの一致度（前方一致が重視される） |
| 箇条書きのキーワード | 最初の1,000文字がインデックス対象 |
| バックエンドキーワード | タイトル・箇条書きの補完 |
| カテゴリノードの正確性 | 適切なカテゴリ設定 |

#### 顧客満足度

| 要素 | 基準 |
|------|------|
| 注文不良率（ODR） | 1%未満 |
| 返品率 | 低いほど有利 |
| 出品者フィードバック | 高評価を維持 |
| レスポンス時間 | 24時間以内の回答 |

#### 外部トラフィック（A10で重要度が大幅上昇）

- Google検索からの流入
- SNS（Instagram、TikTok、YouTube等）からのトラフィック
- ブログ・メディアからの紹介リンク
- インフルエンサーマーケティングからの流入

> A9と比較して外部トラフィックの重要性が大幅に上昇。SNSやGoogle経由のトラフィックを獲得している商品は、ランキングで最大20%の改善が見られる。

#### その他の要素

- FBA利用の有無（FBA利用者が有利）
- 在庫の安定性（在庫切れペナルティ）
- 価格競争力（Amazonは1日に250万回以上価格を調整）
- 商品画像の品質

### A10最適化の10ステップ

1. **キーワードリサーチ**: 短尾・長尾キーワードの両方を調査
2. **タイトル最適化**: 主要キーワードを前方に配置
3. **箇条書き最適化**: 特徴→メリット構成でキーワードを自然に含む
4. **説明文最適化**: 2次キーワードを自然に織り込む
5. **バックエンドキーワード**: 同義語・略語・表記ゆれを網羅
6. **画像最適化**: 高品質な画像でCTR・CVRを向上
7. **価格最適化**: 競争力のある価格設定と動的価格調整
