## 概要

Amazon DSP（Demand-Side Platform）はAmazonの独自データを活用したプログラマティック広告プラットフォーム。Amazon内外のプレミアムサプライに広告を配信でき、Amazon出品者でなくても利用可能。

| 項目 | 内容 |
|------|------|
| 課金方式 | **CPM（インプレッション課金）** |
| Amazon出品 | **不要**（Amazon以外のブランドも利用可能） |
| ブランド登録 | 不要（ただしAmazon出品者の場合は推奨） |
| 配信面 | Amazon.co.jp、Twitch、IMDb、Freevee、Fire TV、サードパーティサイト |
| データ活用 | Amazonの購買・閲覧・検索データに基づくターゲティング |
| クリエイティブ | 静止画、動的広告、OLV動画、OTT（ストリーミングTV）、音声広告 |

**DSPの特徴:**

- Amazon独自のファーストパーティデータ（購買・閲覧・検索行動）を活用
- Amazon内外のプレミアム配信面にプログラマティック配信
- フルファネル対応（認知〜検討〜購入〜リピート）
- Amazon出品なしでもブランド認知広告として利用可能
- SP/SB/SDでリーチできないユーザー層にアプローチ

## 利用条件と費用（日本）

### 3つの利用方法

| 利用方法 | 最低月額費用 | 特徴 |
|----------|------------|------|
| **Amazon直接（マネージドサービス）** | **月額400万円〜** | Amazon担当者がフル運用。大規模広告主向け |
| **代理店経由** | **月額数十万円〜** | Amazon認定代理店が運用。中規模広告主向け |
| **セルフサービス** | **月額150万円〜** | 自社で運用。DSP運用経験のあるチーム向け |

### 費用の内訳

| 費用項目 | 説明 |
|----------|------|
| メディア費 | CPM x インプレッション数。実際の広告配信コスト |
| プラットフォーム費 | DSP利用料（メディア費の10-15%程度） |
| 代理店手数料（代理店経由の場合） | メディア費の15-20%程度 |
| クリエイティブ制作費 | 動画・バナー制作費（別途） |

### DSP導入の判断基準

| 基準 | 推奨 | 非推奨 |
|------|------|--------|
| 月間広告予算 | 150万円以上 | 50万円未満 |
| SP/SB/SDの運用状況 | 十分に最適化済み | まだ最適化の余地あり |
| 目的 | ブランド認知、ファネル上部拡大 | 即効性のある売上増加のみ |
| 商品カテゴリ | 検討期間の長い商品 | 衝動買い系の低単価商品 |

## 広告フォーマット5種

### 1. 静止画ディスプレイ広告

最も基本的なフォーマット。複数サイズで配信され、Amazon内外のウェブサイトに表示。

#### 全サイズ一覧

| サイズ（px） | 名称 | 重要度 | 配信面 |
|-------------|------|--------|--------|
| **300 x 250** | ミディアムレクタングル | **★★★ 最重要** | 商品詳細、サードパーティ |
| **728 x 90** | リーダーボード | ★★☆ | デスクトップ上部 |
| **160 x 600** | ワイドスカイスクレーパー | ★★☆ | サイドバー |
| **970 x 250** | ビルボード | ★★☆ | デスクトップトップページ |
| **300 x 600** | ハーフページ | ★☆☆ | サードパーティサイト |
| **320 x 50** | モバイルバナー | ★★☆ | モバイルサイト |
| **300 x 50** | モバイルバナー（小） | ★☆☆ | モバイルサイト |
| **414 x 125** | モバイルバナー（大） | ★☆☆ | モバイルアプリ |
| **320 x 480** | モバイルインタースティシャル | ★☆☆ | モバイルアプリ |

> **最優先:** 300x250は全配信面で使用されるため、必ず作成すること。次に728x90、160x600を準備。

#### ファイル仕様

| 項目 | 仕様 |
|------|------|
| ファイルサイズ | **日本: 50KB以下** |
| ファイル形式 | JPG、PNG、GIF、HTML5 |
| アニメーションGIF | 最大15秒、3ループまで |
| HTML5 | IAB準拠、150KB以下 |

#### 静止画デザインのポイント

- **300x250が最重要:** この1サイズでインプレッションの50%以上を占める
- 商品画像は鮮明に、背景はシンプルに
- CTAボタンを明確に配置
- ロゴは左上 or 右上に配置
- テキストは最小限（見出し+1行説明）
- ブランドカラーを活用して統一感を出す

### 2. eコマース広告（動的広告）

Amazonの商品カタログ情報を自動取得して動的に生成されるディスプレイ広告。

| 項目 | 詳細 |
|------|------|
| データソース | 商品カタログ（タイトル、画像、価格、レビュー評価） |
| 自動更新 | 価格変更、在庫切れ、セール時に自動反映 |
| パーソナライズ | ユーザーの閲覧履歴に基づき表示商品をカスタマイズ |
| 推奨場面 | リターゲティング、クロスセル |

**eコマース広告の利点:**

- クリエイティブ制作コスト不要
- リアルタイムで商品情報が反映される
- ユーザーごとに最適な商品を自動選択
- 在庫切れ商品は自動で非表示

### 3. OLV動画（Online Video）

ウェブサイトやアプリ内の動画プレーヤーで再生される動画広告。

#### 動画仕様

| 項目 | 仕様 |
|------|------|
| 動画の長さ | 6〜120秒（**推奨: 15〜30秒**） |
| アスペクト比 | **16:9**（横型）、**1:1**（正方形）、**9:16**（縦型） |
| 解像度 | 1920x1080（16:9）、1080x1080（1:1）、1080x1920（9:16） |
| ファイルサイズ | **500MB以下** |
| 映像コーデック | H.264、H.265、VP8、VP9 |
| 音声コーデック | AAC、MP3、**128kbps以上** |
| フレームレート | 24fps以上（推奨30fps） |
| ビットレート | 2Mbps以上推奨 |

#### OLV配信タイプ

| タイプ | 説明 | 課金 |
|--------|------|------|
| インストリーム | 動画コンテンツの前後/途中に再生 | CPM（完全視聴） |
| アウトストリーム | 記事やフィード内で自動再生 | vCPM（視認可能時のみ） |
| インターステイシャル | 全画面表示 | CPM |

### 4. OTT（ストリーミングTV）広告

Fire TV、Amazon Freevee、Prime Video等のストリーミングサービスで配信されるTV広告。

#### 動画仕様

| 項目 | 仕様 |
|------|------|
| 動画の長さ | **15秒 または 30秒のみ** |
| 解像度 | **1920 x 1080px 必須** |
| アスペクト比 | 16:9 |
| ファイルサイズ | 500MB以下 |
| 映像コーデック | H.264 |
| 音声コーデック | AAC、**192kbps以上** |
| 音声ラウドネス | **-24 LKFS**（TV放送基準準拠） |
| フレームレート | 24fps or 30fps |

#### OTTの特徴

| 項目 | 詳細 |
|------|------|
| スキップ | **不可**（完全視聴が保証される） |
| 視聴完了率 | **95%以上** |
| 配信先 | Fire TV、Freevee、Prime Video、Twitch |
| Netflix配信 | **2025年Q4〜日本対応予定** |
| 計測 | ブランドリフト、ストア訪問、Amazon内購買 |

#### OTTクリエイティブの注意点

- TV品質の映像が必須（低解像度やウェブ用動画の流用は不可）
- 音声は必須（ミュート前提のSB動画とは異なる）
- 音声ラウドネス基準 -24 LKFS の遵守
- セーフゾーン（画面端10%）にテキストを配置しない
- 最初の5秒でブランドを認識させる
- CTAは「Amazonで検索」「Alexaで注文」等

### 5. 音声広告

Amazon Music（無料プラン）やAlexaデバイスで配信される音声広告。

#### 音声仕様

| 項目 | 仕様 |
|------|------|
| 音声の長さ | **10〜30秒** |
| ファイル形式 | MP3、WAV |
| サンプルレート | 44.1kHz以上 |
| ビットレート | 192kbps以上 |
| ラウドネス | -14 LUFS（ストリーミング基準） |

#### 音声広告のポイント

- 最初の3秒でブランド名を伝える
- シンプルなメッセージ（1つのベネフィットに集中）
- 明確なCTA（「Amazonで検索」「Alexaに話しかけて」）
- BGMは控えめに（ナレーションが聞き取れること）
- コンパニオンバナー（300x250）を同時配信可能

## オーディエンスターゲティング

### 7つのターゲティング手法

| # | ターゲティング | 説明 | データソース | 推奨用途 |
|---|----------------|------|-------------|----------|
| 1 | **インマーケット** | 特定カテゴリで購買を検討中のユーザー | Amazon検索・閲覧データ | 検討フェーズのユーザー獲得 |
| 2 | **ライフスタイル** | 購買パターンから推定されるライフスタイル | Amazon購買データ | ブランドフィット度の高い層 |
| 3 | **デモグラフィック** | 年齢、性別、世帯収入、居住地域 | Amazon登録情報 | ターゲット層の絞り込み |
| 4 | **ルックアライク** | 既存顧客と類似した行動パターンのユーザー | 自社顧客データ + Amazon ML | 新規顧客獲得の拡大 |
| 5 | **広告主オーディエンス** | 自社CRMデータやDMPデータとのマッチング | 広告主提供データ | 既存顧客のAmazon内リーチ |
| 6 | **AMCカスタム** | Amazon Marketing Cloud分析に基づくカスタムセグメント | AMC分析結果 | 高度なセグメント作成 |
| 7 | **コンテキスト** | コンテンツの文脈に基づく配信（記事内容、動画カテゴリ等） | ページコンテンツ解析 | ブランドセーフティ重視 |

### インマーケットオーディエンスの詳細

Amazonの購買意向データに基づく最も効果的なターゲティング。

| カテゴリ例 | 説明 | 活用シーン |
|-----------|------|-----------|
| 家電・カメラ | カメラ、レンズ、三脚等を検索・閲覧中 | 高単価家電の検討者にリーチ |
| ヘルス&ビューティー | スキンケア、サプリメント等を検索・閲覧中 | 新商品の訴求 |
| スポーツ&アウトドア | フィットネス機器、キャンプ用品等を検討中 | シーズン商品の訴求 |
| ベビー&マタニティ | ベビー用品、マタニティグッズを閲覧中 | ライフイベント連動 |
| ペット用品 | ペットフード、ケア用品等を検討中 | リピート促進 |

### ルックアライクオーディエンスの活用

| ステップ | 内容 |
|----------|------|
| 1. シード作成 | 過去の購入者、高CVRユーザー等をシードとして指定 |
| 2. 類似度設定 | 1-10の範囲で類似度を設定（1=最も類似、10=最大リーチ） |
| 3. 配信開始 | Amazonの機械学習がシードに類似したユーザーを特定 |
| 4. 最適化 | パフォーマンスを見ながら類似度を調整 |

**推奨:** まず類似度3-5で開始し、パフォーマンスを見ながら拡大/縮小。

## レポーティング・計測

### 利用可能な計測手法

| 計測手法 | 説明 | 対象 |
|----------|------|------|
| **Amazon Attribution** | 広告のAmazon内売上への貢献を計測 | 全DSP広告 |
| **Amazon Marketing Cloud（AMC）** | SQLベースのカスタム分析。クロスチャネル分析可能 | DSP + SP/SB/SD横断 |
| **ブランドリフト調査** | 広告接触者/非接触者の認知度・購買意向の差を測定 | OTT、OLV |
| **14日ルックバックウィンドウ** | 広告接触から14日以内の購買をアトリビュート | 全DSP広告 |

### Amazon Attribution詳細

| 指標 | 説明 |
|------|------|
| 商品詳細ページビュー | 広告経由で商品詳細ページを閲覧した回数 |
| カートに追加 | 広告接触後にカートに追加した回数 |
| 購入 | 広告接触後に購入に至った回数 |
| 売上 | 広告接触後の総売上金額 |
| ROAS | 広告費に対するリターン（売上/広告費） |
| 新規顧客購入 | 過去12ヶ月の新規購入者数 |

### AMCの活用例

| 分析ユースケース | 説明 |
|-----------------|------|
| クロスチャネルアトリビューション | SP + SB + SD + DSPの広告接触パスを分析 |
| 新規 vs リピートの購買行動 | 新規顧客と既存顧客の行動差を分析 |
| フリークエンシーキャップ最適化 | 最適な接触頻度を分析 |
| カスタムオーディエンス作成 | 高LTVユーザーの特徴を分析し、ルックアライクのシードに活用 |
| 購買ジャーニー分析 | 初回接触から購買までのタッチポイント分析 |

### 14日ルックバックウィンドウ

| 項目 | 詳細 |
|------|------|
| 計測期間 | 広告のクリックまたはビューから**14日間** |
| ビュースルー | 広告を見た（クリックなし）後の購買も計測 |
| クリックスルー | 広告をクリックした後の購買を計測 |
| 重複排除 | 複数広告に接触した場合、最後のタッチポイントにアトリビュート |

## AI駆動ターゲティングの活用

### 概要

Amazon DSPでは2024年後半からAI/ML駆動のターゲティング機能が強化され、従来の手動セグメント選択よりも高い効果が報告されている。

### 顧客獲得コスト40%低減の事例

| 項目 | 詳細 |
|------|------|
| 業種 | 消費財（FMCG） |
| 施策 | AI駆動のオーディエンスターゲティング導入 |
| 比較対象 | 従来の手動インマーケットセグメント |
| 結果 | 顧客獲得コスト（CPA）**40%低減** |
| 要因 | AIが購買シグナルをリアルタイムで分析し、最適なタイミングで配信 |

### AI機能の活用ポイント

| 機能 | 説明 | 推奨活用 |
|------|------|----------|
| **予測オーディエンス** | 購買確率の高いユーザーをAIが予測 | 新規獲得キャンペーン |
| **自動入札最適化** | 目標KPIに基づきCPMをリアルタイム調整 | 全キャンペーン |
| **クリエイティブ最適化** | ユーザーに最適なクリエイティブを自動選択 | 複数クリエイティブ運用時 |
| **フリークエンシー最適化** | 最適な接触回数をAIが判定 | 認知キャンペーン |
| **ペーシング最適化** | 予算消化ペースを目標に合わせて自動調整 | 期間限定キャンペーン |

### DSP運用のベストプラクティス

1. **十分なデータ蓄積:** 最低2週間のデータ蓄積期間を設ける
2. **複数オーディエンスの並行テスト:** インマーケット vs ルックアライク vs AI予測
3. **クリエイティブのローテーション:** 最低3種類のクリエイティブを用意
4. **フリークエンシーキャップ:** 1日3-5回、1週間15-20回を目安
5. **Amazon内 + 外のバランス:** Amazon外はリーチ拡大、Amazon内はCVR重視
6. **AMCとの連携:** 定期的にAMC分析を実施し、ターゲティング精度を向上
