# 競合分析（Competitor Analysis）

直接・間接の競合を特定し、各社のサービス内容・強み・弱み・ポジショニング・広告戦略を調査してレポートにまとめる。

## 前提条件

以下のスキルが完了していることが望ましい:
- `industry-landscape` — 対象業界の前提知識調査
- `market-analysis` — 市場規模・市場構造・価格帯・顧客像の調査

## 調査項目

以下の3領域を調査する。各領域の具体的な調査項目・クエリ例は [references/research-guide.md](references/research-guide.md) を参照すること。

1. **競合の特定** — 直接競合・間接競合のリストアップ
2. **競合企業の基本情報** — 売上規模、従業員数、事業内容、代表者、URL
3. **Google検索での競合比較** — 主要キーワードでの検索順位と自社との比較
4. **広告運用の調査** — 広告ライブラリで確認できる広告・LP、スクリーンショット撮影

## 検索ルール

### 複数ソースの原則

1つのトピックにつき**最低3〜5回**の異なるクエリで検索し、**2〜3件以上**のソースをWebFetchで本文まで読むこと。

### クエリ設計

- **広→狭のファネル戦略**: まず全体像を掴み、次に絞り込む
- **毎回クエリを変える**: 類義語・視点を変える
- **年を入れる**: 鮮度が重要な情報には必ず年を明記
- **言語はビジネスに合わせる**: ターゲット市場・展開地域に応じた言語で検索する（日本ローカルなら日本語、海外展開なら英語や現地語など）
- **比較・対立軸を使う**: 比較クエリは質の高い分析記事がヒットしやすい

### 情報収集ツール

Web検索（WebSearch）とWebFetchに加え、`playwright-browser` スキルのscrapeコマンドを活用して競合サイトの詳細情報を取得すること。

```
Step 1: WebSearchで検索 → URLリストを取得
Step 2: WebFetchまたはplaywright-browserのscrapeで本文を取得
```

## 広告ライブラリの活用

競合の広告・マーケティング戦略を調べるために、`playwright-browser` スキルを使って以下のツールにアクセスすること。

| ツール | URL | 用途 |
|--------|-----|------|
| Meta広告ライブラリ | `https://www.facebook.com/ads/library/` | Facebook/Instagram広告の検索・閲覧 |
| Google広告の透明性センター | `https://adstransparency.google.com/?region=JP` | Google広告の検索・閲覧 |
| TikTok Top Ads | `https://ads.tiktok.com/business/creativecenter/inspiration/topads/pc/ja?period=30&region=JP` | TikTok広告のパフォーマンス上位クリエイティブ閲覧 |
| TikTokキーワードインサイト | `https://ads.tiktok.com/business/creativecenter/keyword-insights/pc/ja` | TikTok広告で使われているキーワードのトレンド |

### 調査手順

1. `playwright-browser` スキルで対象ツールにアクセスする
2. 競合の企業名・ブランド名で検索する
3. 以下の観点で情報を収集する：
   - **広告ライブラリ**: どのようなLPを出しているか、広告クリエイティブの内容、訴求ポイント
   - **TikTok Top Ads**: パフォーマンス上位の広告の訴求パターン、フォーマット
   - **TikTokキーワードインサイト**: 注目キーワード
4. 複数の競合を比較し、傾向をまとめる

### スクリーンショットの撮影（重要）

広告クリエイティブと競合のLPは、`playwright-browser` スキルの screenshot コマンドでスクリーンショットを撮影し、画像ファイルとして保存すること。

**撮影対象:**
- 広告ライブラリで見つけた主要な広告クリエイティブ
- 広告から遷移するランディングページ（`--full-page` オプションでページ全体を撮影）
- 競合のホームページのトップ

**保存先:** `research/screenshots/competitor-analysis/` ディレクトリに保存する。ファイル名は `[競合名]-[種別]-[連番].png` の形式とする（例: `company-a-ad-01.png`、`company-a-lp-01.png`）。

マークダウンレポート内では、スクリーンショットのファイルパスを記載し、どの画像が何を撮影したものかを明記する。

### URLの記録

広告ライブラリで見つけた情報は、分析内容だけでなく**該当ページへのリンクやURL**も必ず記録すること。

記録すべきもの：
- 各広告の個別リンク（広告ライブラリの検索結果URL、個別広告のURL）
- 広告内に含まれるランディングページURL
- TikTokキーワードインサイトの検索結果URL
- 広告クリエイティブ内で言及されているWebサイトやサービスのURL

## アウトプットルール

### 網羅的な記述

- 各競合のサービス内容・料金・強み・弱みは具体的に記載する
- 複数ソースから得た情報はすべて記載し、差異があればそれも明記する
- 「など」「その他多数」で省略せず、把握した項目はすべて列挙する
- 広告ライブラリで確認した内容も具体的に記述する

### ソースURLの記載

すべての発見事項・データには、情報源のURLを必ず記載すること。URLのないデータは出力に含めない。

### ファイル出力

リサーチ結果は必ずマークダウンファイルとして出力する。出力先はユーザーの指示に従う（デフォルト: `research/` ディレクトリ）。

### ファイル構成

```
research/
├── competitor-analysis.md                    # 競合分析レポート
└── screenshots/competitor-analysis/          # スクリーンショット
    ├── serp-[keyword-slug].png              # Google検索結果
    ├── [競合名]-ad-01.png                    # 広告クリエイティブ
    ├── [競合名]-lp-01.png                    # ランディングページ
    └── [競合名]-hp-01.png                    # ホームページ
```
