# 業界全体像調査（Industry Landscape）

業界・市場・分野の「前提知識」を体系的に収集し、レポートにまとめる。前提知識が揃っていないリサーチは、的外れな質問を立て、表面的な情報しか得られない。このスキルで業界の言語・構造を掴むことで、後続の調査（市場調査、競合分析、法規制、価格調査等）の精度と効率が格段に上がる。

## 調査項目

以下の4領域を調査する。各領域の具体的な調査項目・クエリ例は [references/research-guide.md](references/research-guide.md) を参照すること。

1. **業界概要と前提知識** — 業界の定義・範囲・歴史・成熟度を把握する
2. **専門用語・業界用語** — 業界固有の用語集（最低20〜30語）を作成する
3. **ビジネスモデルとマネタイズ** — 収益構造、お金の流れ、価格帯・利益率を理解する
4. **バリューチェーンとキープレイヤー** — 川上から川下の構造、主要企業とシェア、力関係を把握する

## 検索ルール

### 複数ソースの原則

1つのトピックにつき**最低3〜5回**の異なるクエリで検索し、**2〜3件以上**のソースをWebFetchで本文まで読むこと。1つのサイトの情報だけでアウトプットしてはならない。

### クエリ設計

- **広→狭のファネル戦略**: まず全体像を掴み、次に絞り込む
- **毎回クエリを変える**: 類義語・視点を変える
- **年を入れる**: 鮮度が重要な情報には必ず年を明記
- **言語はビジネスに合わせる**: ターゲット市場・展開地域に応じた言語で検索する（日本ローカルなら日本語、海外展開なら英語や現地語など）

### 検索演算子の活用

| 演算子 | 用途 | 例 |
|--------|------|-----|
| `"完全一致"` | フレーズ完全一致検索 | `"市場規模 2025" SaaS` |
| `site:` | 特定サイト内検索 | `site:meti.go.jp 電子商取引` |
| `filetype:` | ファイル形式指定 | `filetype:pdf [業種] 業界レポート` |
| `intitle:` | タイトル内検索 | `intitle:業界地図 [業種]` |

**一次情報を狙うsite:指定:**
- 業界団体: `site:or.jp`
- 調査会社: `site:statista.com` `site:grandviewresearch.com`

### 検索→取得の2ステップ

```
Step 1: WebSearchで検索 → URLリストを取得
Step 2: WebFetchで有望なURLの中身を読み込む → 詳細情報を取得
```

スニペットだけで判断せず、重要な情報はWebFetchで本文を読むこと。

## アウトプットルール

### 網羅的な記述

- 複数ソースから得た情報はすべて記載し、差異があればそれも明記する
- 「など」「その他多数」で省略せず、把握した項目はすべて列挙する

### ソースURLの記載

すべての発見事項・データには、情報源のURLを必ず記載すること。URLのないデータは出力に含めない。

### ファイル出力

リサーチ結果は必ずマークダウンファイルとして出力する。出力先はユーザーの指示に従う（デフォルト: `research/` ディレクトリ）。

```markdown

調査日: YYYY-MM-DD

---

- 業界の成熟度

|------|------|------|
（最低20〜30語）
```
