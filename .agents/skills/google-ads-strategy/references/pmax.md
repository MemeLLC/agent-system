# P-MAX（Performance Max）実践マニュアル

---

## 用語定義

本マニュアルで使用する略語・用語の定義。

| 略語 | 正式名称 | 定義 |
|------|---------|------|
| P-MAX | Performance Max | Google全チャネル横断でAIが自動最適化するキャンペーンタイプ |
| AG | アセットグループ | P-MAX内の広告素材（テキスト・画像・動画）と配信シグナルのセット |
| LG | リスティンググループ | 商品フィードのどの商品をAGに含めるかを制御するフィルタ |
| CL | カスタムラベル | Merchant Centerの商品フィードに付与する分類ラベル（0〜4の5種類） |
| CV | コンバージョン | 広告の目標アクション（購入、問い合わせ等）の完了 |
| CV値 | コンバージョン値 | 1件のCVに紐づく金額（売上、リードスコア等） |
| CPA | コンバージョン単価 | 1件のCV獲得にかかった広告費（費用 ÷ CV数） |
| ROAS | 広告費用対効果 | 広告費に対する売上の倍率（CV値合計 ÷ 費用） |
| VTC | ビュースルーコンバージョン | 広告表示後、クリックせずにCVした件数 |
| tCPA | 目標コンバージョン単価 | 入札戦略の1つ。目標とするCPA |
| tROAS | 目標広告費用対効果 | 入札戦略の1つ。目標とするROAS |
| GMC | Google Merchant Center | 商品フィードを管理するプラットフォーム |
| URL拡張 | Final URL Expansion | Googleが指定URLとは異なるLPに自動誘導する機能 |
| フィードオンリー | Feed-Only Campaign | テキスト・画像・動画アセットを入稿せず、商品フィードだけで配信する方式 |

## 意思決定早見表

P-MAXを導入すべきか、どう設計するかの判断基準。

### 導入判断

| 条件 | 判断 | 理由 |
|------|------|------|
| 月間CV 30件以上（アカウント全体） | 導入を検討 | AIの学習に最低限必要なデータ量 |
| 月間CV 50件以上 | 積極的に導入 | 安定した最適化が期待できる |
| 月間CV 10件未満 | 時期尚早 | 学習データ不足で最適化が困難。まず検索広告でCVデータを蓄積 |
| 検索キャンペーンが未実施 | 検索を先に開始 | P-MAXは既存キャンペーンのCVデータを活用して最適化する |

### キャンペーン数の目安

| 月間CV件数（アカウント全体） | 推奨P-MAXキャンペーン数 | 備考 |
|---------------------------|----------------------|------|
| 30〜50件 | 1キャンペーン | データを集約して学習効率を最大化 |
| 50〜150件 | 1〜2キャンペーン | ブランド/ノンブランド分離を検討 |
| 150件以上 | 2〜4キャンペーン | 商品カテゴリ・利益率・顧客タイプで分割 |

### ビジネスタイプ別の設計方針

| ビジネスタイプ | キャンペーン構造 | アセット方針 | 特記事項 |
|--------------|----------------|------------|---------|
| EC（単品〜少品目） | 1キャンペーン・2〜3 AG | フルアセット推奨 | 商品フィード最適化が最重要 |
| EC（多品目） | 2〜4キャンペーン（カテゴリ・利益率別） | フィードオンリー or フルアセット | カスタムラベルで商品セグメント |
| リード獲得（BtoB） | 1〜2キャンペーン | フルアセット必須 | URL拡張OFF推奨。ブログ等への流出防止 |
| リード獲得（BtoC） | 1〜2キャンペーン | フルアセット必須 | デモグラフィック除外の活用 |
| 店舗集客 | P-MAX for Store Goals | フルアセット推奨 | Googleビジネスプロフィール連携必須 |

## クイックスタート（立ち上げ手順）

P-MAXを初めて立ち上げる際の最短ステップ。詳細は各章を参照。

### 事前条件チェック

| チェック項目 | 必須/推奨 | 未達の場合 |
|------------|---------|----------|
| CV計測が正確に動作している | 必須 | まずCV計測を整備（[9章](#9-コンバージョン計測の最適化)参照） |
| Enhanced Conversions（拡張CV）が実装済み | 強く推奨 | 導入すると計測精度が向上 |
| 月間CV 30件以上（アカウント全体） | 必須 | 検索キャンペーンでCVデータを蓄積してから導入 |
| 検索キャンペーンが稼働中 | 強く推奨 | ブランドKW制御・カニバリ監視のため先に構築 |
| 商品フィードが最適化済み（EC） | 必須（EC） | [shopping-ads.md](shopping-ads.md) 参照 |
| オフラインCVインポート体制（リード獲得） | 推奨 | CRM連携でGCLIDを保存する体制を構築 |

### 立ち上げステップ

| ステップ | アクション | 参照章 |
|---------|----------|-------|
| 1. 構造決定 | キャンペーン数・AG数を決定（初期は1キャンペーン・1〜3 AG推奨） | [2章](#2-キャンペーン構造の設計) |
| 2. アセット制作 | テキスト（見出し11個+説明文4個以上）・画像（横長+スクエア各5枚以上）・動画（横長+縦長各1本以上） | [4章](#4-アセット仕様と入稿要件) |
| 3. シグナル設定 | カスタマーマッチリスト + 購買意向セグメントをオーディエンスシグナルに設定 | [5章](#5-オーディエンスシグナル) |
| 4. 検索テーマ | 8〜12テーマを設定 | [6章](#6-検索テーマとurl拡張) |
| 5. 除外設定 | ブランド除外 + 無関係KW除外 + URL除外ルール（リード獲得はURL拡張OFF） | [7章](#7-除外キーワードとブランド除外) |
| 6. 入札設定 | 初期は「コンバージョン数の最大化」or「CV値の最大化」（制約なし）で開始 | [8章](#8-入札戦略) |
| 7. 予算設定 | 目標CPA × 3倍以上の日予算を設定 | [8章](#8-入札戦略) |
| 8. 公開 | キャンペーンを有効化 | — |

### 最初の2週間（学習期間）

| やること | やらないこと |
|---------|------------|
| 毎日の予算消化率チェック | 入札戦略の変更 |
| 異常な支出パターンの監視 | 予算の増減 |
| 検索語句レポートの確認（明らかに無関係な語句の除外のみ可） | アセットの大量差替え |
| チャネルレポートの確認（傾向把握のみ） | tCPA/tROAS制約の追加 |

### 3〜6週間目（安定化→最適化移行）

| 判断 | 条件 | アクション |
|------|------|----------|
| tCPA/tROASに移行 | 月間CV 30件以上が安定 | 過去30日の実績値ベースで目標を設定（[8-3](#8-入札戦略)参照） |
| AGの追加 | 既存AGが月間20CV以上 | 新しいカテゴリ/ターゲットのAGを追加 |
| アセット改善 | 「Low」評価アセットが確認できる | 「Low」アセットを差替え |
| 除外KW追加 | 検索語句レポートで無関係語句を確認 | 戦略的に除外（個別語句ではなくカテゴリで） |

## 2. キャンペーン構造の設計

### 2-1. 構造設計の原則

P-MAXのキャンペーン構造は、以下の3層で考える。

```
アカウント
 └── キャンペーン（予算・入札戦略・地域・除外キーワードの単位）
      └── アセットグループ（クリエイティブ・オーディエンスシグナル・検索テーマの単位）
           └── リスティンググループ（商品フィードのフィルタ。EC向け）
```

**キャンペーンを分ける基準:**
- 異なる予算配分が必要な場合
- 異なる入札戦略（tCPA vs tROAS）が必要な場合
- 異なる地域ターゲティングが必要な場合
- 異なる除外キーワード戦略が必要な場合

**アセットグループで分ける基準:**
- 異なる商品カテゴリ・サービスライン
- 異なるオーディエンスセグメント
- 異なるクリエイティブテーマ・訴求軸

> **原則: 迷ったらキャンペーン数を少なく、アセットグループで分ける。** データが十分になってからキャンペーンを分割する。各キャンペーンに月間30件以上のCVが確保できることが最低条件。

### 2-2. EC向けキャンペーン構造パターン

| パターン | 構造 | 適した状況 |
|---------|------|----------|
| シンプル構造 | 1キャンペーン / 1〜3 AG | 月間CV 30〜100件。少品目EC |
| カテゴリ分割 | 2〜3キャンペーン / 各2〜3 AG | 月間CV 100件以上。カテゴリ間で利益率が大きく異なる |
| 利益率分割 | 高利益率 / 中利益率 / 低利益率の3キャンペーン | tROAS運用。商品ごとの利益率差が大きい |
| ブランド/ノンブランド分割 | ブランド指名用 / 一般用の2キャンペーン | ブランド指名の予算・ROASを独立管理したい |

### 2-3. リード獲得向けキャンペーン構造パターン

| パターン | 構造 | 適した状況 |
|---------|------|----------|
| シンプル構造 | 1キャンペーン / 2〜3 AG（サービス別） | 月間CV 30〜50件。サービスラインが少ない |
| サービス分割 | サービスA用 / サービスB用の2キャンペーン | サービスごとにCPA目標が異なる |
| ファネル分割 | 顕在層向け（CV目標=問い合わせ） / 潜在層向け（CV目標=資料DL） | CV種別で入札戦略を変えたい |

### 2-4. キャンペーン設定チェックリスト

| 設定項目 | 推奨設定 | 備考 |
|---------|---------|------|
| キャンペーン目標 | 販売促進 / リード獲得 | ビジネス目標に合わせる |
| 入札戦略 | CV値重視→tROAS / CV数重視→tCPA | 詳細は[8章](#8-入札戦略)参照 |
| 予算 | 目標CPAの3倍以上/日 | 最低でもCPA×1倍/日 |
| 地域 | 対象地域を明示指定 | 「所在地」のみに限定推奨 |
| 言語 | 対象言語を明示指定 | |
| 広告スケジュール | 必要に応じて設定 | デフォルトは24時間配信 |
| URL拡張 | 目的に応じてON/OFF | 詳細は[6章](#6-検索テーマとurl拡張)参照 |
| ブランド除外 | 必要に応じて設定 | 詳細は[7章](#7-除外キーワードとブランド除外)参照 |
| 新規顧客獲得 | 必要に応じて有効化 | 詳細は[12章](#12-新規顧客獲得モード)参照 |

## 3. アセットグループの設計

### 3-1. アセットグループの基本

1つのアセットグループには以下の要素が含まれる。

| 要素 | 説明 |
|------|------|
| テキストアセット | 広告見出し・長い広告見出し・説明文・ビジネス名・表示URLパス |
| 画像アセット | 横長・スクエア・縦長の画像、ロゴ |
| 動画アセット | YouTube動画（横長・縦長・スクエア） |
| オーディエンスシグナル | ターゲットユーザーのヒント情報 |
| 検索テーマ | 検索意図のテーマ（最大25個/AG） |
| 最終URL | 広告クリック後の遷移先 |
| リスティンググループ | 対象商品のフィルタ（EC向け） |

### 3-2. アセットグループの分割戦略

| 分割方式 | 説明 | 適した状況 |
|---------|------|----------|
| カテゴリ別 | メインのWebサイトカテゴリに対応 | 管理が容易。汎用的 |
| ターゲット別 | ペルソナ・セグメント単位 | アパレル・美容など属性差が大きい業種 |
| 訴求別 | 価格訴求 / 品質訴求 / 利便性訴求 | テスト目的でクリエイティブを分けたい場合 |
| プロモーション別 | セール・季節キャンペーン用 | 既存AGを乱さず期間限定訴求を出したい場合 |

### 3-3. アセットグループ設計ルール

| ルール | 理由 |
|-------|------|
| 1 AG あたり月間20件以上のCVを確保 | 学習に必要な最低データ量 |
| CV 5件未満の AG は統合を検討 | データ不足でAIが最適化できない |
| 最初は1〜3 AGで開始 | 過剰な分割はデータ分散を招く |
| AG間で商品・ターゲットの重複を避ける | AIの学習が混乱する |
| 全アセットタイプ（テキスト・画像・動画）を登録 | 全チャネルに配信するために必要 |
| 「広告の強度」を「優良」にする | Googleの推奨。配信量に影響 |

## 4. アセット仕様と入稿要件

### 4-1. テキストアセット

| アセット種類 | 最小数 | 最大数 | 文字数上限 | 備考 |
|------------|-------|-------|----------|------|
| 広告見出し | 3 | 15 | 半角30文字 | 15文字以下の見出しを1つ以上含める |
| 長い広告見出し | 1 | 5 | 半角90文字 | |
| 説明文 | 2 | 5 | 半角90文字 | 60文字以下の説明文を1つ以上含める |
| ビジネス名 | 1 | 1 | 半角25文字 | |
| 表示URLパス | 0 | 2 | 半角15文字/各 | |
| CTA | 1 | 1 | — | Googleが自動選択 or 手動指定 |

> **日本語・中国語・韓国語**: 全角1文字 = 半角2文字としてカウント。

**テキストアセットのベストプラクティス:**
- 見出しは最低10個以上登録してAIの学習精度を向上
- 各見出しが単独で意味が通じるように書く（組み合わせて表示されるため）
- ブランド名、USP、CTA、価格/割引、社会的証明（実績数など）をバランスよく含める
- 各チャネルの広告コピーのベストプラクティスは [search-ads.md](search-ads.md) を参照

### 4-2. 画像アセット

| アスペクト比 | 用途 | 最小数 | 最大数 | 推奨サイズ | 最小サイズ |
|------------|------|-------|-------|----------|----------|
| 横長（1.91:1） | GDN・Discover・Gmail | 1 | 20 | 1200×628 | 600×314 |
| スクエア（1:1） | GDN・Discover・Gmail | 1 | 20 | 1200×1200 | 300×300 |
| 縦長（4:5） | モバイル配信面 | 0 | 20 | 960×1200 | 480×600 |
| ロゴ（スクエア 1:1） | 全配信面 | 1 | 5 | 1200×1200 | 128×128 |
| ロゴ（横長 4:1） | 一部配信面 | 0 | 5 | 1200×300 | 512×128 |

| 仕様 | 要件 |
|------|------|
| ファイル形式 | JPEG または PNG |
| 最大ファイルサイズ | 5MB |
| 被写体配置 | フレームの中央80%に配置 |

**画像のベストプラクティス:**
- 横長（1.91:1）とスクエア（1:1）は必須。縦長（4:5）も登録推奨
- 画像内にテキストを過度に入れない（画像面積の20%以下が目安）
- プロフェッショナルな品質の画像を使用
- 商品画像、利用シーン、人物、ライフスタイル画像をバランスよく混ぜる
- 各チャネルの画像ベストプラクティスは [display-ads.md](display-ads.md) を参照

### 4-3. 動画アセット

| 項目 | 要件 |
|------|------|
| ホスティング | YouTube（事前にアップロード必須） |
| 最小尺 | 10秒 |
| 推奨尺 | 15秒以上 |
| 最大ファイルサイズ | 256GB |
| 推奨解像度 | 1080p Full HD |
| 形式 | MPEG-2 / MPEG-4 (.MPG) |
| AG あたり上限 | 各アスペクト比5本ずつ |

| アスペクト比 | 用途 | 推奨数 |
|------------|------|-------|
| 横長（16:9） | YouTube インストリーム | 1本以上 |
| 縦長（9:16） | YouTube Shorts | 1本以上（10〜60秒。Shorts配信に必須） |
| スクエア（1:1） | モバイル配信面 | 推奨 |

**動画のベストプラクティス:**
- 動画を登録しない場合、Googleが画像から自動生成動画を作成する。品質面から**自社制作動画の登録を強く推奨**
- 最低セット: 横長（16:9）1本 + 縦長（9:16）1本
- 各チャネルの動画ベストプラクティスは [video-campaigns.md](video-campaigns.md) を参照

### 4-4. 推奨アセット登録数

「広告の強度」を「優良」にするための目安。

| アセット種類 | 推奨登録数 |
|------------|----------|
| 広告見出し | 11〜15個 |
| 長い広告見出し | 3〜5個 |
| 説明文 | 4〜5個 |
| 横長画像 | 5〜15枚 |
| スクエア画像 | 5〜15枚 |
| 縦長画像 | 3〜5枚 |
| ロゴ | 1〜3個 |
| 動画 | 3〜5本（3アスペクト比をカバー） |

## 5. オーディエンスシグナル

### 5-1. オーディエンスシグナルとは

オーディエンスシグナルはターゲティングではなく「ヒント」。Googleのアルゴリズムに最適な見込み顧客の特徴を教える入力情報。P-MAXはシグナルを起点に学習し、シグナル外のユーザーにも自動的に配信を拡大する。

### 5-2. シグナルの種類と優先度

| 優先度 | シグナルタイプ | 説明 | 設定方法 |
|-------|-------------|------|---------|
| 最高 | カスタマーマッチ | 既存顧客のメールアドレスリスト | Customer Matchリストをアップロード |
| 高 | Webサイト訪問者 | リマーケティングリスト | Google Adsのオーディエンスリスト |
| 高 | YouTube視聴者 | チャンネル登録者・動画視聴者 | YouTube連携 |
| 中 | 購買意向の強いセグメント | 特定カテゴリの購入を検討中のユーザー | Google定義のセグメントから選択 |
| 中 | カスタムセグメント | 特定の検索語句やURL訪問に基づくセグメント | KWやURLを指定して作成 |
| 低 | アフィニティセグメント | 興味・関心カテゴリ | Google定義のセグメントから選択 |
| 低 | ライフイベント | 結婚・引越し・就職など | Google定義のセグメントから選択 |
| 低 | デモグラフィック | 年齢・性別・世帯年収・子供の有無 | 基本属性で指定 |

### 5-3. オーディエンスシグナル設計ルール

| ルール | 理由 |
|-------|------|
| カスタマーマッチリストを最優先で登録 | 最も強力なシグナル。AIの学習起点として最適 |
| 1 AGに複数のシグナルタイプを組み合わせる | 多角的なシグナルでAIの学習精度を向上 |
| シグナルは「ヒント」であり「制限」ではないと理解する | Googleはシグナル外にも配信を拡大する |
| 小さすぎるオーディエンスリストを避ける | 1,000人以上が推奨 |
| AGごとに異なるシグナルを設定 | 同じシグナルを複数AGで使うとデータが分散 |

## 6. 検索テーマとURL拡張

### 6-1. 検索テーマ

検索テーマは、P-MAXの検索チャネル配信をガイドするキーワードテーマ。検索キャンペーンの「キーワード」とは異なり、AIが関連検索語句を自動拡張する起点として機能する。

| 項目 | 仕様（調査時点） |
|------|------|
| 登録上限 | 25テーマ / アセットグループ |
| 機能 | AIが予測する配信に加えて、指定テーマの検索語句にも配信を広げる（加算的） |
| マッチタイプ | なし（AIが自動的に関連語句に拡張） |

**設計ルール:**
- 8〜12テーマの登録を推奨
- 自社の主要サービス名・商品カテゴリ・顧客が使う検索語句を含める
- 検索キャンペーンの高パフォーマンスKWを参考にテーマを設計
- 競合ブランド名は検索テーマに入れない（ブランド除外と矛盾するため）
- 検索テーマは「キーワード」ではなく「テーマ」として設計する（完全一致的な発想をしない）

### 6-2. URL拡張（Final URL Expansion）

URL拡張はデフォルトでON。Googleが検索クエリに応じて、指定した最終URLとは異なるLPに自動的にトラフィックを誘導する。

| 設定 | 動作 | 推奨シーン |
|------|------|----------|
| ON（デフォルト） | Googleが最適なLPを自動選択。動的見出し・説明文を生成 | ECサイト（多数の商品ページがある場合） |
| OFF | 指定した最終URLにのみ誘導 | リード獲得（特定LPに誘導したい場合） |
| ON + URL除外ルール | 特定URLを除外しつつ自動選択を活用 | ECだがブログ・キャリアページ等への流出を防ぎたい場合 |

**URL拡張のリスクと対策:**

| リスク | 対策 |
|-------|------|
| ブログ記事・プライバシーポリシーなどCVに無関係なページへの誘導 | URL除外ルールで /blog/、/privacy/、/careers/ 等を除外 |
| ブランドページ等、意図しないページでの広告表示 | URL除外ルールを設定 |
| 意図したLP誘導・メッセージマッチが崩れる（LPテスト設計も無効化される） | リード獲得ではURL拡張OFFを推奨 |

## 7. 除外キーワードとブランド除外

### 7-1. 除外キーワード

2025年1月から、P-MAXにキャンペーンレベルの除外キーワードが自己管理で追加可能になった。共有除外キーワードリストにも対応。

| 項目 | 仕様（調査時点） |
|------|------|
| 上限 | 10,000件 / キャンペーン |
| 共有リスト | 対応（複数P-MAXキャンペーンに一括適用可能） |
| 適用範囲 | 主に検索・ショッピングチャネルに影響（Display・YouTube等には効果なし） |

> 上限数は変更される可能性がある。最新の上限はGoogle Ads管理画面の キャンペーン > 除外キーワード で確認可能。

**除外キーワード戦略:**

| カテゴリ | 除外対象の例 |
|---------|------------|
| 無関係語句 | 求人、採用、年収、口コミ、評判（CVに結びつかない場合） |
| 競合ブランド名 | 意図的に競合KWを狙わない場合 |
| 情報検索語句 | 「とは」「やり方」「比較」（CVに結びつかない場合） |
| ネガティブ語句 | 「最悪」「詐欺」「解約」 |

> **過度な除外は避ける。** 非CV検索語句を片っ端から除外すると、AIの探索範囲が狭まり、新規顧客発見の機会を失う。CVに明確に結びつかないカテゴリを戦略的に除外するのが正しいアプローチ。

### 7-2. ブランド除外

ブランド除外は、特定のブランド名を含む検索語句での広告表示を防ぐ機能。

| 設定 | 効果 | 推奨シーン |
|------|------|----------|
| 自社ブランド除外 | P-MAXが自社ブランド検索を奪うのを防止 | 既に検索キャンペーンでブランドKWを運用中 |
| 競合ブランド除外 | 競合名検索での広告表示を防止 | 競合KWを意図的に狙わない方針の場合 |
| ショッピングのみ除外解除 | ブランド除外を検索テキスト広告にだけ適用し、ショッピング広告はブランド検索に表示 | EC。ショッピング広告でブランド指名トラフィックを取りたい場合（2025年〜） |

## 8. 入札戦略

### 8-1. 利用可能な入札戦略

P-MAXで利用可能な入札戦略は4種類。

| 入札戦略 | 最適化対象 | 推奨シーン |
|---------|----------|----------|
| コンバージョン数の最大化 | CV数 | 初期立ち上げ時。CVデータ蓄積フェーズ |
| 目標CPA（tCPA） | CV数（コスト制約付き） | リード獲得。CV値が均一 |
| コンバージョン値の最大化 | CV値（売上） | EC。CVデータ蓄積フェーズ |
| 目標ROAS（tROAS） | CV値（ROAS制約付き） | EC。十分なCVデータがある場合 |

### 8-2. 入札戦略の選び方

```
CV値が商品ごとに異なるか？
├── はい（EC等）
│   ├── 月間CV 50件以上 → tROAS
│   └── 月間CV 50件未満 → コンバージョン値の最大化（tROAS制約なし）
└── いいえ（リード獲得等。CV値が均一）
    ├── 月間CV 30件以上 → tCPA
    └── 月間CV 30件未満 → コンバージョン数の最大化（tCPA制約なし）
```

### 8-3. 入札戦略の設計ルール

| ルール | 理由 |
|-------|------|
| 初期は制約なし（「最大化」系）で開始 | AIの学習を制約しない。データ蓄積を優先 |
| tCPA/tROASの目標値は過去実績ベースで設定 | 理想値ではなく実績値を起点にする |
| tROASは過去30日の実績ROASの80〜90%からスタート | 急な制約は配信量の急減を招く |
| tCPAは過去30日の実績CPAの110〜120%からスタート | 同上 |
| 目標値の変更は10〜20%刻みで段階的に | 大幅な変更は学習リセットを引き起こす |
| 入札変更と予算変更を同時にしない | AIの学習が混乱する。片方ずつ変更して効果を計測 |
| 変更後は2〜4週間の学習期間を確保 | 安定前に再変更すると学習が永続的に不安定になる |

### 8-4. 予算設計

| 項目 | 推奨 | 備考 |
|------|------|------|
| 最低日予算 | 目標CPA × 3倍以上 | Google推奨。データ蓄積速度を確保 |
| 最低ライン | 目標CPA × 1倍 | これ以下は学習が極めて遅い |
| 実務的な最低目安 | 目標CPA × 1〜3倍/日（例: CPA ¥5,000なら¥5,000〜15,000/日） | 業種・CPAにより変動 |
| 予算増額 | 7〜14日ごとに最大20%ずつ | 急な増額は学習リセットを引き起こす |
| 予算減額 | 7〜14日ごとに最大15%ずつ | 急な減額は配信が不安定になる |

## 9. コンバージョン計測の最適化

### 9-1. P-MAXにおけるCV計測の重要性

P-MAXはCVデータを唯一の学習シグナルとして使う。CVの定義・計測精度がそのままキャンペーンの品質に直結する。

### 9-2. CV計測の設計ルール

| ルール | 理由 |
|-------|------|
| 最適化の対象とするCVアクションは、実際のビジネス成果に近いものに限定 | PDFダウンロードやニュースレター登録など低意図アクションを含めると、AIがそれらを量産する |
| Enhanced Conversions（拡張コンバージョン）を必ず実装 | 計測精度が向上することが多い（Google社の報告では5〜10%程度。EC/リード、同意モード設定等により変動）。Cookieレス環境への対応 |
| CV値を正確に設定（ECは動的CV値、リード獲得はリードスコアベース推奨） | CV値の精度がtROASの精度に直結 |
| アトリビューションモデルはデータドリブンを推奨 | Google推奨。ラストクリック以外のタッチポイントも評価 |

### 9-3. クリックベース vs ビュースルーCV

P-MAXはデフォルトでビュースルーコンバージョン（VTC）を含めて最適化する。これにより、ROASが実態より高く見える問題が広く報告されている。

| 指標 | 説明 | 注意点 |
|------|------|-------|
| クリックスルーCV | 広告をクリックしてCVした件数 | 実際の広告効果を反映 |
| ビュースルーCV | 広告を見たがクリックせずにCVした件数 | 因果関係が不明確。過大評価のリスク |

**対策（上級者向け）:**
- クリックベースのみのCVアクションを作成し、P-MAXのCV目標に設定する
- 切替直後は見かけ上のROASが大幅に低下するが、数週間で実質的なROAS改善が起こり得るとの報告がある（出典: ALM Corp 2026年レポート。低下幅・改善幅は業種・アカウント条件・CV量により大きく変動）
- この手法は上級者向け。まずはデフォルト設定で開始し、VTCの割合を監視してから判断する

**自分のアカウントで検証する手順:**
1. Google Adsの「セグメント > コンバージョン > コンバージョンアクション」でVTCの割合を確認
2. VTCが全CVの50%以上を占めている場合、クリックベースCVアクションの作成を検討
3. 切替後は4週間以上の観察期間を設け、クリックスルーCVのCPA/ROASで評価する
4. 切替前後のCPA・ROAS・CV数を記録し、実質的な改善効果を判断する

### 9-4. オフラインコンバージョンインポート

リード獲得ビジネスでは、フォーム送信だけでなく「商談化」「成約」などの後工程データをGoogle Adsにインポートすることで、P-MAXの最適化精度が大幅に向上する。

| ステップ | 内容 |
|---------|------|
| 1. GCLID取得 | フォーム送信時にGCLIDをCRMに保存 |
| 2. 後工程のCV定義 | 商談化・成約をCVアクションとして定義 |
| 3. 定期インポート | CRMからGoogle AdsにCVデータを定期的にアップロード |
| 4. P-MAXの最適化目標に設定 | インポートしたCVを最適化対象に含める |

## 10. 商品フィード連携（EC向け）

### 10-1. 商品フィードの役割

ECサイトの場合、P-MAXはGoogle Merchant Center（GMC）の商品フィードを使ってショッピング広告・動的リマーケティングを配信する。フィードの品質がパフォーマンスを大きく左右する。

> **商品フィードの詳細な設計・最適化方法は [shopping-ads.md](shopping-ads.md) を参照。**

### 10-2. リスティンググループ（商品のフィルタリング）

リスティンググループは、商品フィードのどの商品をどのアセットグループに割り当てるかを制御する。

**フィルタリングに使える属性:**

| 属性 | 用途 | 例 |
|------|------|-----|
| カテゴリ（Googleの商品カテゴリ） | 大分類での分割 | アパレル > メンズ |
| ブランド | ブランド別の管理 | Nike / Adidas |
| 商品ID | 個別商品の包含・除外 | 特定SKUの除外 |
| 状態 | 新品・中古の分割 | 新品のみ |
| カスタムラベル（0〜4） | 自由な分類 | 利益率、季節性、売上ランクなど |

### 10-3. カスタムラベル活用パターン

| ラベル | 用途 | 値の例 |
|-------|------|-------|
| custom_label_0 | 利益率ティア | high_margin / mid_margin / low_margin |
| custom_label_1 | 売上パフォーマンス | best_seller / mid_performer / long_tail |
| custom_label_2 | 季節性 | spring / summer / autumn / winter / year_round |
| custom_label_3 | プロモーション | on_sale / clearance / new_arrival |
| custom_label_4 | 在庫状況 | in_stock / low_stock / pre_order |

**カスタムラベルを使ったキャンペーン分割例:**

```
P-MAX Campaign A（高利益率・高パフォーマンス）
 ├── tROAS: 400%
 ├── AG1: best_seller × high_margin
 └── AG2: mid_performer × high_margin

P-MAX Campaign B（中利益率・その他）
 ├── tROAS: 250%
 ├── AG1: best_seller × mid_margin
 └── AG2: 残りの商品
```

## 11. フィードオンリー vs フルアセット

### 11-1. フィードオンリーキャンペーンとは

テキスト・画像・動画アセットを一切入稿せず、商品フィードのデータだけで配信する方式。実質的にショッピング広告と動的リマーケティングに特化する。

### 11-2. 比較

| 項目 | フィードオンリー | フルアセット |
|------|---------------|------------|
| 配信面 | ショッピング・動的リマーケティング中心 | 全7チャネル（検索・ショッピング・YouTube・GDN・Discover・Gmail・マップ） |
| アセット制作 | 不要（フィードのみ） | テキスト・画像・動画の制作が必要 |
| 立ち上げ速度 | 速い | アセット制作に時間がかかる |
| CPA/ROAS | 短期的にはCPAが低くなる傾向がある（高意図ユーザーに集中するため） | 長期的にはリーチ拡大でCV量が増加する可能性 |
| リーチ | 限定的（購買意向の高いユーザー中心） | 広い（認知〜検討〜購入の全ファネル） |
| ブランド認知 | ほぼなし | YouTube・GDN配信によるブランド接触 |

### 11-3. どちらを選ぶか

| シーン | 推奨方式 | 理由 |
|-------|---------|------|
| 即座のROAS重視 | フィードオンリー | 無駄な配信面を排除し、高意図ユーザーに集中 |
| アセット制作リソースがない | フィードオンリー | フィードだけで開始可能 |
| 新規顧客開拓・ブランド認知も目的 | フルアセット | 全チャネルへのリーチで新しい顧客を発見 |
| 長期的な成長戦略 | フルアセット | 認知→検討→購入のフルファネルをカバー |
| 既にDisplay/YouTubeを別キャンペーンで運用 | フィードオンリー | P-MAXとの重複・カニバリを防止 |

## 12. 新規顧客獲得モード

### 12-1. 概要

P-MAXには新規顧客を優先的に獲得するための専用モードがある。

| モード | 動作 | 推奨シーン |
|-------|------|----------|
| 新規顧客の入札値引き上げモード（New Customer Value Mode） | 新規顧客のCVに追加価値を設定し、新規顧客への入札を強化。既存顧客にも配信される | 新規獲得を強化しつつ、既存顧客からのCV機会も維持 |
| 新規顧客のみモード（New Customer Only Mode） | 新規顧客のCVのみを最適化対象とする。既存顧客は配信対象外 | 新規獲得に完全特化したい場合 |
| 高価値新規顧客モード（High-Value New Customer Mode）| カスタマーマッチリストで高価値既存顧客を定義し、類似する高LTV新規顧客への入札をさらに強化 | LTVに大きな差がある顧客基盤を持つビジネス |

### 12-2. 設定要件

| 要件 | 詳細 |
|------|------|
| カスタマーマッチリスト | 既存顧客のメールアドレスリストのアップロードが必要 |
| コンバージョンタグ | 新規/既存顧客を識別するパラメータの実装が必要 |
| 高価値モード | 高価値顧客セグメントのカスタマーマッチリストが追加で必要 |

### 12-3. 設計上の注意

- 新規顧客の「追加価値」は過大に設定しない。実際の新規顧客のLTVに基づいて算出する
- 「新規顧客のみモード」は配信量が大幅に減少する可能性がある。十分なトラフィックが見込める場合のみ使用
- カスタマーマッチリストは定期的に更新する（月1回以上推奨）

## 14. 他キャンペーンとの共存戦略

### 14-1. P-MAXと検索キャンペーンの関係

P-MAXは検索チャネルにも配信されるため、既存の検索キャンペーンとのカニバリゼーション（食い合い）が発生する。

**オークション優先ルール:**

| 条件 | 優先されるキャンペーン |
|------|-------------------|
| 検索KWが完全一致（Exact Match）で一致 | **検索キャンペーン** |
| 検索KWがフレーズ一致・部分一致で一致 | Ad Rankが高い方（多くの場合P-MAX） |
| 検索KWが検索キャンペーンに存在しない | **P-MAX** |

**カニバリ防止策:**

| 対策 | 効果 |
|------|------|
| 検索キャンペーンのブランドKWを完全一致で登録 | ブランド指名は検索キャンペーンが優先される |
| P-MAXにブランド除外を設定 | P-MAXがブランド検索を奪うのを防止 |
| P-MAXの検索語句インサイトを定期確認 | P-MAXがどの検索語句で配信されているかを把握 |
| 高パフォーマンス語句を検索キャンペーンに完全一致で追加 | 検索キャンペーンに優先権を確保 |
| 検索キャンペーンのCPC推移を監視 | P-MAX導入後にCPCが上昇していないか確認 |

### 14-2. P-MAXとショッピングキャンペーンの関係

| 状況 | 推奨 |
|------|------|
| Standard Shoppingを運用中 | P-MAXがオークションで優先される。両方運用する場合は商品セグメントの重複に注意 |
| P-MAXのみで運用 | 一般的に推奨。1キャンペーンで管理可能 |
| 並行運用 | Standard Shoppingで特定商品（高利益率など）を細かく管理し、残りをP-MAXに任せるパターン |

### 14-3. P-MAXと他キャンペーンタイプの関係

| キャンペーンタイプ | カニバリリスク | 推奨方針 |
|----------------|-------------|---------|
| 検索（Search） | 高 | 完全一致KWで優先権を確保。P-MAXにブランド除外 |
| ショッピング（Standard Shopping） | 高 | 商品セグメントの重複を排除 |
| ディスプレイ（Display） | 中 | 目的が異なれば共存可能。CV最適化目的の場合は要注意 |
| 動画（YouTube） | 低〜中 | 認知目的の動画キャンペーンとは共存可能 |
| Demand Gen | 高 | 両方がCV最適化の場合、Demand Genが抑制されるリスク |
| アプリ | なし | 配信チャネルが完全に異なる |

> **各キャンペーンタイプの詳細なベストプラクティスは、各リファレンス（[search-ads.md](search-ads.md)、[display-ads.md](display-ads.md)、[shopping-ads.md](shopping-ads.md)、[video-campaigns.md](video-campaigns.md)、[app-campaigns.md](app-campaigns.md)）を参照。**

## 19. 学習期間と運用サイクル

### 19-1. 学習期間

| フェーズ | 期間 | 内容 |
|---------|------|------|
| 初期学習 | 2〜4週間 | AIが入札・ターゲティング・クリエイティブの最適化を学習。パフォーマンスが不安定 |
| 安定化 | 4〜6週間 | 学習が進み、パフォーマンスが安定し始める |
| 成熟 | 6週間以降 | 安定したパフォーマンス。この段階で最適化・拡張を検討 |

**学習期間のリセットが発生する変更:**
- 入札戦略の変更
- 予算の20%以上の増減
- CVアクションの追加・変更・削除
- アセットグループの大幅な変更（アセットの50%以上の差替え）
- ターゲット地域の大幅な変更

### 19-2. 運用サイクル

| 頻度 | アクション |
|------|----------|
| 毎日 | 予算消化率の確認。異常な支出がないかチェック |
| 週次 | チャネルレポート確認 / アセットパフォーマンス確認 / 検索語句レポート確認 / 支出集中度チェック（上位20%の商品/AGに80%以上の支出が集中していないか） |
| 隔週 | 「Low」評価アセットの差替え / 除外KWの追加 |
| 月次 | オーディエンスインサイトの分析 / 入札目標の見直し / 予算配分の見直し / カスタマーマッチリストの更新 |
| 四半期 | 全アセットのリフレッシュ / キャンペーン構造の見直し / 競合動向と照合 |

### 19-3. 最適化の優先順位

パフォーマンスが期待を下回る場合の改善順序。

| 優先度 | チェック項目 | 対策 |
|-------|------------|------|
| 1 | CV計測は正確か | Enhanced Conversions実装 / CV定義の見直し |
| 2 | 予算は十分か | 目標CPA × 3倍以上の日予算を確保 |
| 3 | アセット品質は十分か | 「Low」アセットの差替え / アセット数の充足 |
| 4 | オーディエンスシグナルは適切か | カスタマーマッチリストの追加 / シグナルの見直し |
| 5 | 除外KW・ブランド除外は適切か | 検索語句レポートで無関係語句を除外 |
| 6 | 入札目標は現実的か | 過去実績に基づいた目標値に調整 |
| 7 | URL拡張の設定は適切か | 不要ページへの流出がないか確認 |
| 8 | カニバリは発生していないか | 検索キャンペーンのCPC・IS推移を確認 |

## 20. よくある失敗パターンと回避策

### 20-1. 設計段階の失敗

| 失敗パターン | なぜ失敗か | 回避策 |
|------------|----------|-------|
| CVデータが不足した状態で開始 | AIが学習できず、無駄な配信が増える | 月間30CV以上を検索キャンペーン等で蓄積してから導入 |
| キャンペーンを過剰に分割 | 各キャンペーンのCVデータが分散 | 1キャンペーンあたり月間30CV以上を確保できる構造にする |
| アセットグループを過剰に分割 | データ分散でAIの学習が遅延 | 初期は1〜3 AGで開始。データに基づいて段階的に増やす |
| 検索キャンペーンなしでP-MAXのみ開始 | ブランドKWの制御ができず、カニバリ判断もできない | 検索キャンペーンを先に構築してからP-MAXを追加 |

### 20-2. アセット関連の失敗

| 失敗パターン | なぜ失敗か | 回避策 |
|------------|----------|-------|
| 汎用的なストック画像のみ使用 | CVRが低い。ブランドの差別化ができない | 自社商品・サービスの実写画像を使用 |
| 動画アセット未登録 | Googleが画像から自動生成した低品質動画が配信される | 最低でも横長+縦長の2本を用意 |
| テキストアセットが少ない | AIのテスト可能な組み合わせが限定される | 見出し11個以上、説明文4個以上を登録 |
| 画像にテキストを過度に入れる | GDN・Discover配信でポリシー違反・配信制限のリスク | テキストは画像面積の20%以下に |

### 20-3. 運用段階の失敗

| 失敗パターン | なぜ失敗か | 回避策 |
|------------|----------|-------|
| 学習期間中に頻繁に変更 | 学習がリセットされ、永続的に不安定 | 変更後は最低2週間は触らない |
| URL拡張ONのまま放置 | ブログ・採用ページ等にトラフィックが流出 | URL除外ルールを設定。リード獲得はOFF推奨 |
| ブランド除外なしで運用 | P-MAXがブランド検索のCVを奪い、見かけ上のROASが高いが実質的な増分は少ない | ブランド除外を設定し、検索キャンペーンとの棲み分けを明確に |
| 非CV検索語句を全て除外 | AIの探索範囲が狭まり、新規顧客の発見機会を失う | 戦略的に「カテゴリ」で除外する。個別の非CV語句をすべて除外しない |
| ROASの数値だけで評価 | VTCを含むROASは過大評価の可能性 | クリックスルーCVのみのROASも併せて評価 |
| 予算を一気に増額 | 学習リセットが発生 | 7〜14日ごとに最大20%ずつ増額 |
| tCPA/tROASの目標値を理想値に設定 | 配信量が激減し、学習が停止 | 過去実績値をベースに設定 |
| P-MAXを「設定して放置」 | 環境変化に対応できずパフォーマンスが劣化 | 週次のヘルスチェックを習慣化 |

### 20-4. P-MAXのリターゲティングバイアス

P-MAXはデフォルトで既存顧客・サイト訪問者など「CVしやすいユーザー」を優先的にターゲティングする傾向がある。これにより、ROASは高く見えるが実質的な新規獲得が少ないという問題が報告されている。

**対策:**
- 新規顧客獲得モードを有効化（[12章](#12-新規顧客獲得モード)参照）
- カスタマーマッチリストで既存顧客を識別させる
- チャネルレポートでDisplay/YouTubeの割合を確認（リタゲ偏重ならDisplay比率が高い傾向）
- 新規 vs 既存のCV内訳を定期的に分析

## 付録A: 症状別トラブルシューティング

よくある症状から原因を特定し、打ち手を判断するための逆引き表。

| 症状 | 主な原因 | 打ち手 |
|------|---------|-------|
| CVはあるがROASが悪化している | VTCが多く実質ROASが見かけより低い / tROASが緩すぎる / 低利益率商品にCVが集中 | VTCの割合を確認 / tROASを実績ベースで段階的に引き上げ / カスタムラベルで利益率別に分割 |
| 支出が一部の商品/AGに集中する | AIが一部のCVしやすい商品に最適化している | リスティンググループで商品を分割 / 低支出AGのアセット品質を改善 / キャンペーン分割で予算を独立管理 |
| YouTube/Display比率が高すぎる | フルアセット配信でリタゲ偏重 / 検索チャネルの競合が激しい | フィードオンリーへの切替を検討 / 新規顧客獲得モードを有効化 / チャネルレポートで実際のCV貢献を確認 |
| CPA/CPCが急に上昇した | 学習リセットが発生 / 競合環境の変化 / 季節要因 | 直近の変更を確認（入札・予算・アセット） / 2〜3週間待って学習を安定させる / 検索語句レポートで新しい高CPCクエリを確認 |
| 検索キャンペーンの成果が落ちた | P-MAXとのカニバリゼーション | P-MAXの検索語句レポートで重複クエリを確認 / ブランド除外の設定 / 重要KWを検索キャンペーンに完全一致で追加 |
| CVが極端に少ない・学習が進まない | 予算不足 / CVアクション定義が厳しすぎる / アセット品質が低い | 日予算を目標CPA × 3倍以上に引き上げ / マイクロCVの追加を検討 / アセットを充足 |
| 「広告の強度」が「低い」のまま | アセット数不足 / アセット種類の欠落 | 見出し11個以上・説明文4個以上・画像各5枚以上・動画1本以上を確保 |
