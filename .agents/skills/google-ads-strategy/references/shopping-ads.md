# Googleショッピング広告

---

## 用語定義

本マニュアルで使用する略語・用語の定義。

| 略語 | 正式名称 | 定義 |
|------|---------|------|
| GMC | Google Merchant Center | 商品データをGoogleに送信・管理するプラットフォーム |
| PLA | Product Listing Ads | 商品画像・価格・店舗名を表示するショッピング広告 |
| GTIN | Global Trade Item Number | JANコード/UPC/EAN等の国際的な商品識別コード |
| MPN | Manufacturer Part Number | メーカー型番 |
| SKU | Stock Keeping Unit | 在庫管理単位 |
| ROAS | Return on Ad Spend | 広告費用対効果（CV値合計 ÷ 費用） |
| CPA | Cost Per Acquisition | 1件のCV獲得にかかった広告費 |
| CTR | Click Through Rate | 表示回数に対するクリック数の割合 |
| CPC | Cost Per Click | 1クリックあたりの費用 |
| CVR | Conversion Rate | クリック数に対するCV数の割合 |
| PMax | Performance Max | Google全配信面に自動配信するキャンペーンタイプ |
| RLSA | Remarketing Lists for Search Ads | リマーケティングリストを使った入札調整（ショッピングでは「オーディエンス（観察）」として適用） |

### 2-1. 初期設定チェックリスト

| 5 | 返品ポリシー | 返品条件と返品URL（対応国で必須化が進行中） |
| 6 | Google Ads連携 | Google Adsアカウントとリンク |
| 7 | プログラム登録 | 「ショッピング広告」「無料リスティング」を有効化 |

### 2-2. Merchant Center Next（新UI）の主な変更点

2025〜2026年にかけてのアップデート:

- **Product Studio**: 商品画像のAI編集機能（背景削除、画像アニメーション化、解像度アップスケール）

- **Merchant API v1**: Content API for Shoppingの後継として正式リリース（v1betaは2026年2月終了）（[API更新情報](https://developers.google.com/merchant/api/latest-updates)）

※ Merchant Centerの機能名・提供範囲・期限は変更される場合がある。実装前に公式ヘルプで最新情報を確認すること。

## 3. 商品フィードの設計と最適化

商品フィードはショッピング広告の**品質の土台**。フィードの品質がCTR・CVR・CPCの全てに直結する。

### 3-1. 必須属性

全商品で必ず設定が必要な属性。

| 属性 | 説明 | 注意点 |
|------|------|--------|
| `id` | 商品の一意識別子 | SKU等を使用。変更不可 |
| `title` | 商品タイトル | 最大150文字。詳細は第4章 |
| `description` | 商品説明文 | 最大5,000文字。特徴・素材・用途を記載 |
| `link` | 商品詳細ページURL | ランディングページと完全一致必須 |
| `image_link` | メイン商品画像URL | 詳細は第5章 |
| `availability` | 在庫状況 | `in_stock` / `out_of_stock` / `preorder` のいずれか |
| `price` | 価格 | 通貨コード付き（例: `2980 JPY`）。LP上の価格と完全一致 |
| `condition` | 商品の状態 | `new` / `refurbished` / `used` |

### 3-2. 条件付き必須属性

商品カテゴリや状況に応じて必要になる属性。

| 属性 | 必須条件 | 説明 |
|------|---------|------|
| `brand` | 新品商品 | ブランド名（GTIN登録済み商品は特に重要） |
| `gtin` | MPNがない場合 | JANコード/UPC/EAN。広告の表示優先度に影響 |
| `mpn` | GTINがない場合 | メーカー型番 |
| `google_product_category` | 推奨（全商品） | Googleの商品カテゴリ分類 |
| `product_type` | 推奨 | 自社独自の商品カテゴリ |
| `age_group` | アパレル | `newborn` / `infant` / `toddler` / `kids` / `adult` |
| `color` | アパレル | 商品の色 |
| `gender` | アパレル | `male` / `female` / `unisex` |
| `size` | アパレル | サイズ表記 |
| `item_group_id` | バリエーション商品 | 色違い・サイズ違い等のグループID |
| `shipping` | 配送情報 | GMC設定で一括指定も可能 |

### 3-3. カスタムラベル（custom_label_0〜4）

カスタムラベルは最大5つ設定でき、**キャンペーン構造・入札・予算配分の最重要ツール**。

| ラベル | 用途例 | 値の例 |
|--------|--------|--------|
| `custom_label_0` | 利益率 | `high_margin` / `mid_margin` / `low_margin` |
| `custom_label_1` | 季節性 | `spring` / `summer` / `autumn` / `winter` / `evergreen` |
| `custom_label_2` | 販売パフォーマンス | `bestseller` / `new_arrival` / `clearance` |
| `custom_label_3` | 価格帯 | `under_3000` / `3000_10000` / `over_10000` |
| `custom_label_4` | プロモーション | `sale` / `bundle` / `regular` |

**活用のポイント:**
- 利益率ラベルにより、**利益ベースの入札制御**が可能（ROASだけでなく実際の粗利を反映）
- 季節性ラベルにより、シーズン前の予算増・シーズン後の予算減を自動化
- 売上パフォーマンスラベルにより、ベストセラーに予算を集中

### 3-4. 商品説明文（description）のベストプラクティス

| ルール | 説明 |
|--------|------|
| 最初の160〜500文字に重要情報 | 切り詰め表示への対応。ブランド・商品種別・主要特徴を先頭に |
| LPと一致した表現 | フィードとLPで異なる表現を使わない |
| 特徴と利点を併記 | 素材・サイズ・用途・解決する課題を具体的に |
| プロモーション文言禁止 | 「送料無料」「セール中」等の宣伝文句はdescriptionに入れない |
| キーワードスタッフィング禁止 | 不自然なキーワード詰め込みはポリシー違反 |

### 3-5. フィードの更新とアップロード

| 方式 | 特徴 | 推奨シーン |
|------|------|-----------|
| URL指定（自動取得） | GoogleがURLから定期的に自動取得 | 頻繁に更新がある大規模カタログ |
| ファイルアップロード | 手動でXML/CSV/TSVファイルをアップロード | 更新頻度が低い小規模カタログ |
| FTP/SFTP | サーバー経由で自動送信 | 1日に複数回の更新が必要な場合 |
| Content API / Merchant API | プログラムで直接送信 | リアルタイム更新が必要な場合 |

**更新頻度のルール:**
- **最低30日に1回**のフィード再送信が必要（超えると商品データが期限切れとなり無効化される）。ただし、Merchant Centerに直接登録した商品やGoogleの自動アイテム更新が有効な場合は挙動が異なる
- **推奨は毎日更新**（価格・在庫の不一致は不承認の最多原因。不一致を防ぐにはフィード更新頻度を上げるのが最も確実）
- スケジュール取得の場合、更新間隔は**最短30分**

### 3-6. 補助フィード（Supplemental Feed）

メインフィードを変更せずに、データの追加・上書きが可能。

| 用途 | 説明 |
|------|------|
| 不承認の修正 | メインフィードに手を入れずに属性値を修正 |
| カスタムラベル追加 | パフォーマンスデータに基づくラベルを追加 |
| 季節属性の付与 | シーズンごとの属性を一括で上書き |
| タイトル・説明文のテスト | A/Bテスト用の代替タイトルを設定 |

## 4. 商品タイトルの最適化

商品タイトルは**ショッピング広告で最も重要な最適化ポイント**。Googleが検索クエリと商品をマッチングする主要なシグナルであり、ユーザーのクリック判断にも直結する。

### 4-1. 基本ルール

| ルール | 説明 |
|--------|------|
| 最大150文字 | ただし表示は約70文字で切れる。**重要な情報を先頭70文字以内に** |
| LP上の商品名と一致 | フィードとLPで異なる商品名を使うと不承認リスク |
| プロモーション文言禁止 | 「送料無料」「50%OFF」「セール」等はtitleに含めない |
| 全て大文字禁止 | 「FREE SHIPPING」のような全大文字表記はNG |
| 具体的な属性を含める | ブランド、色、サイズ、素材等を含めると検索マッチ精度が上がる |

### 4-2. カテゴリ別タイトル構成

カテゴリごとに最適なタイトル構成は異なる。

| カテゴリ | 推奨構成 | 例 |
|---------|---------|-----|
| **アパレル** | ブランド + 商品種別 + 性別 + 素材/特徴 + 色 + サイズ | ナイキ ランニングジャケット レディース クライマクール ブルー M |
| **家電・電子機器** | ブランド + 商品名/型番 + スペック + 特徴 | ソニー WH-1000XM5 ワイヤレスヘッドホン ノイズキャンセリング ブラック |
| **家具・インテリア** | 商品種別 + 特徴 + 素材 + サイズ + 色 | L字型ソファ モダン ベルベット 3人掛け ネイビー |
| **食品・飲料** | ブランド + 商品名 + フレーバー/味 + 容量 + 入数 | 伊藤園 お〜いお茶 緑茶 525ml 24本入 |
| **コスメ・美容** | ブランド + 商品名 + タイプ + 効果/特徴 + 容量 | SK-II フェイシャルトリートメントエッセンス 化粧水 保湿 230ml |

### 4-3. タイトル最適化のコツ

- **ブランド認知度が高い場合** → ブランド名を先頭に
- **ブランド認知度が低い場合** → 商品種別（検索されやすい一般名詞）を先頭に
- **同じ商品のバリエーション** → 色・サイズ等の差異を必ずタイトルに反映
- **Googleの構造化タイトル属性（structured_title）** → AIによるタイトル最適化に対応する属性。提供状況・対応範囲は変動するため、[公式ヘルプ](https://support.google.com/merchants/answer/6324415)で最新情報を確認

## 5. 商品画像の要件とベストプラクティス

### 5-1. 画像サイズ要件

| 項目 | 一般商品 | アパレル商品 |
|------|---------|-------------|
| 最小サイズ | 100 × 100 px | 250 × 250 px |
| 推奨サイズ | **1,500 × 1,500 px以上** | **1,500 × 1,500 px以上** |
| 最大サイズ | 64メガピクセル | 64メガピクセル |
| 最大ファイルサイズ | 16 MB | 16 MB |

### 5-2. 画像の要件

| ルール | 説明 |
|--------|------|
| 商品が画像面積の75〜90%を占める | 余白が多すぎても少なすぎてもNG |
| 白背景推奨 | メイン画像は白背景が基本。ライフスタイル画像はadditional_image_linkで |
| 追加テキスト・透かし・装飾の禁止 | 画像上にウォーターマーク、テキスト、装飾を後から重ねない（不承認リスク）。ただし**商品自体やパッケージに印刷されたロゴ・文字**は通常問題にならない |
| プロモーションテキスト禁止 | 「SALE」「送料無料」等のテキストオーバーレイ不可 |
| ボーダー・フレーム禁止 | 画像を枠線で囲まない |
| プレースホルダー画像禁止 | 「No Image Available」等の仮画像は不承認 |

### 5-3. 追加画像（additional_image_link）

- 最大10枚まで設定可能
- ライフスタイル画像（使用シーン）、アングル違い、ディテール、パッケージ画像等に活用
- メイン画像では見えない情報を補完する役割

### 5-4. 画像のベストプラクティス

- **高解像度で鮮明な画像**を使用（ぼやけ・ノイズは印象を下げる）
- **複数アングルの画像**を用意（正面、側面、背面、使用シーン）
- **画像URLを安定的に維持**（タイムスタンプ付きURLは避ける。URLが変わるとGoogleの再クロールが必要になる）
- **Product Studio**を活用（背景除去、解像度向上などのAI編集が無料で利用可能）

## 6. キャンペーン構造の設計

### 6-1. 命名規則

一貫した命名規則で管理効率を上げる。

**推奨フォーマット:**

```
[地域] Shopping [ブランド有無] [商品カテゴリ] [年]
```

**例:**
- `JP Shopping Non-Brand 家具 2026`
- `JP Shopping Brand ソファ 2026`
- `JP Shopping Remarketing 全商品 2026`

### 6-2. キャンペーン分離の考え方

| 分離軸 | 理由 |
|--------|------|
| ブランド検索 vs 非ブランド検索 | ブランド検索はCVRが高く、混在すると非ブランドの実力値が見えない |
| 利益率別 | 高利益率商品に予算を集中し、低利益率商品は控えめに |
| ベストセラー vs その他 | 実績のある商品を独立させ、積極的に入札 |
| リマーケティング vs 新規 | オーディエンス（観察）を使い、過去の訪問者には高い入札、新規には低い入札 |
| 季節商品 vs 通年商品 | シーズンに応じた予算の増減を容易にする |

**ブランド/非ブランドの分離方法（ショッピング特有）:**

ショッピング広告ではキーワード入稿ができないため、ブランド/非ブランドの分離は**キャンペーン優先度 + 除外キーワード**の組み合わせで実現する。完全な制御ではなく「配信の誘導」である点に注意。

```
[Non-Brand キャンペーン]（優先度: 高）
  ├── 自社ブランド名を除外キーワードに登録
  ├── → ブランド名を含まない検索クエリに優先配信される
  └── 入札: 非ブランドは一般的にCVRが低いため、控えめに設定

[Brand キャンペーン]（優先度: 低）
  ├── 除外キーワードなし（ブランド語を含む検索をここで拾う）
  ├── → Non-Brandが自社ブランド語を除外しているため、ブランド検索が流入
  └── 入札: ブランド検索はCVRが高いため、高めに設定可能
```

**注意:**
- Non-Brand側に「ブランド語」を除外する方式が一般的（逆方向の除外は誤除外のリスクが高い）
- 完全分離は不可能。検索語句レポートで漏れを定期的にチェックする
- 競合ブランド名の除外は、法務/ポリシーの観点からも慎重に判断する

### 6-3. 推奨キャンペーン構成テンプレート

EC事業者向けの基本構成（3キャンペーン構成）:

```
[Standard Shopping] ベストセラー
  ├── 商品グループ: カスタムラベル = bestseller
  ├── 入札: 手動CPC（高め）→ 目標ROAS へ移行
  ├── 予算配分: 全体の50%
  └── 優先度: 高

[Standard Shopping] 一般商品
  ├── 商品グループ: ベストセラー以外の全商品
  ├── 入札: 手動CPC（標準）→ コンバージョン数最大化へ移行
  ├── 予算配分: 全体の35%
  └── 優先度: 中

[Standard Shopping] テスト / 新商品
  ├── 商品グループ: カスタムラベル = new_arrival
  ├── 入札: 手動CPC（低め）
  ├── 予算配分: 全体の15%
  └── 優先度: 低
```

### 6-4. 予算の目安

**日予算の考え方:**

1日あたり十分なクリック数を確保しないと、AIの学習やパフォーマンス評価に必要なデータが集まらない。目安として**1日10クリック以上**を確保できる予算を設定する。

```
推定CPC × 10（最低クリック数目安） = 最低日予算
```

例: 推定CPC 30円 → 最低日予算 300円/日（月約9,000円）

※ この計算式は「最低限のデータ蓄積に必要なクリック数」に基づく目安であり、CV最適化には1日あたりの予算をさらに上げる必要がある。

**スケール段階別の予算配分:**

| 月間予算規模 | 推奨構成 |
|------------|---------|
| 〜30万円 | Standard Shopping 1キャンペーンで開始 |
| 30〜100万円 | ベストセラー + 一般の2キャンペーン分離 |
| 100〜300万円 | 3キャンペーン構成 + PMax追加検討 |
| 300万円〜 | 3キャンペーン + PMax + ブランド防衛キャンペーン |

## 7. ショッピング広告の入札設計

### 7-1. 手動CPCの設定方法

**上限CPC算出式:**

```
上限CPC = 商品価格 × 粗利率 × コンバージョン率
```

例: 商品価格 10,000円、粗利率 40%、CVR 2%
→ 上限CPC = 10,000 × 0.40 × 0.02 = **80円**（損益分岐点）

**初期入札は損益分岐点の50〜75%で開始**し、パフォーマンスを見て調整。

### 7-2. 手動から自動入札への移行

| 段階 | 条件 | アクション |
|------|------|-----------|
| 1. 手動CPC開始 | — | 商品グループ別に手動入札。データ蓄積を優先 |
| 2. データ蓄積 | 30件以上のCV | 入札パフォーマンスの傾向を把握 |
| 3. 自動入札テスト | 安定したCV実績 | 目標ROAS or 目標CPAを下書き設定で事前テスト |
| 4. 自動入札移行 | テスト結果が良好 | 完全移行。目標値は過去実績ベースで設定 |

**重要: 新規キャンペーンでいきなり目標ROASを設定しない。** CVデータ不足でAIの学習が不安定になり、配信量が極端に絞られるか、非効率に拡大する。

### 7-3. 入札調整の考え方

**利益率ベースの入札:**

| 利益率 | 入札姿勢 | 例 |
|--------|---------|-----|
| 高（60%以上） | 積極的 | 上限CPCの75〜100% |
| 中（30〜59%） | 標準 | 上限CPCの50〜75% |
| 低（30%未満） | 保守的 | 上限CPCの25〜50% |

**オーディエンス（観察）による入札調整:**

ショッピングキャンペーンでは、オーディエンスを「観察」モードで追加し、入札調整を適用できる。なお、スマート入札を使用している場合、入札調整は自動的にシグナルとして取り込まれるため、手動での調整は不要。

| オーディエンス | 入札調整の目安（手動CPC時） |
|---------------|--------------------------|
| カート放棄者 | +50% |
| 商品ページ閲覧者 | +25% |
| サイト全訪問者 | +10% |

※ 類似セグメント（Similar Audiences）は2023年8月に廃止済み。代替として、スマート入札のシグナル最適化やカスタマーマッチの活用を推奨。

### 7-4. キャンペーン優先度（Priority）

Standard Shoppingでは、**同一商品が複数キャンペーンに存在する場合**、優先度で入札するキャンペーンが決まる。

| 優先度 | 入札の仕組み |
|--------|------------|
| **高（High）** | 最優先で入札される。他の優先度のキャンペーンより先に入札 |
| **中（Medium）** | 高優先度キャンペーンの予算切れ時、または高優先度に含まれない商品で入札 |
| **低（Low）** | 高・中が予算切れの場合に入札。デフォルト設定 |

**同一優先度の場合 → 入札額が高い方が使用される。**

## 8. 商品グループと除外キーワード

### 8-1. 商品グループの設計

ショッピング広告ではキーワード指定ではなく、**商品グループ**で入札を管理する。

**セグメント軸（使用可能な属性）:**

| 属性 | 説明 | 活用例 |
|------|------|--------|
| カテゴリ | Googleの商品カテゴリ | 大カテゴリ → 中カテゴリの階層で分割 |
| ブランド | ブランド名 | 自社ブランド vs 仕入れブランド |
| 商品タイプ | 自社定義のカテゴリ | 自由度の高い分類が可能 |
| カスタムラベル | custom_label_0〜4 | 利益率・季節・パフォーマンス等 |
| 商品ID | 個別SKU | ベストセラーの個別入札に |
| 状態 | new / refurbished / used | 状態別の入札調整 |

**推奨の階層構成:**

```
全商品
├── カテゴリ: 家具
│   ├── カスタムラベル: bestseller → 入札: 高
│   ├── カスタムラベル: regular → 入札: 中
│   └── カスタムラベル: clearance → 入札: 低
├── カテゴリ: 家電
│   ├── カスタムラベル: bestseller → 入札: 高
│   └── ...
└── その他すべて → 入札: 低（キャッチオール）
```

**「その他すべて」の入札を必ず設定する。** 未分類商品がデフォルト入札で無駄に配信されるのを防ぐ。

### 8-2. 除外キーワード

ショッピング広告はキーワードで表示を制御できないため、**除外キーワードが唯一の検索クエリ制御手段**。

**除外キーワードの設定レベル:**

| レベル | 範囲 |
|--------|------|
| キャンペーンレベル | キャンペーン内の全商品グループに適用 |
| 広告グループレベル | 特定の広告グループにのみ適用 |

**マッチタイプ:**

| マッチタイプ | 記法 | 説明 |
|------------|------|------|
| 部分一致 | `キーワード` | デフォルト。語順不問で含まれる検索に適用 |
| フレーズ一致 | `"キーワード"` | 指定した語順を含む検索に適用 |
| 完全一致 | `[キーワード]` | 完全に一致する検索のみに適用 |

**初期除外キーワードの例:**

| カテゴリ | 除外キーワード例 |
|---------|----------------|
| 購入意図なし | 無料、DIY、手作り、作り方、修理、レビュー、比較 |
| 競合・他社 | （該当する場合）競合社名・競合ブランド名 |
| 関係のない商品 | 取り扱い外のカテゴリ・商品種別 |
| 中古・不要 | 中古、ジャンク、買取、売りたい |

### 8-3. 検索語句レポートの活用

- **週次**で検索語句レポートを確認
- 無関係なクエリ、CVに繋がらないクエリを除外キーワードに追加
- 高パフォーマンスのクエリを発見 → 商品タイトルへの反映を検討

## 9. ポリシーと審査

### 9-1. 禁止されている商品

以下の商品はショッピング広告で配信不可。

| カテゴリ | 例 |
|---------|-----|
| 偽造品 | ブランド品の模倣品・コピー商品 |
| 危険な商品 | 武器、弾薬、爆発物、花火、薬物、ドラッグ関連器具 |
| 不正行為を助長する商品 | ハッキングツール、学術不正サービス |
| 不適切なコンテンツ | ヘイト・暴力を助長するもの |

### 9-2. 制限付きの商品

広告は可能だが、追加の条件や制約がある。

| カテゴリ | 制限内容 |
|---------|---------|
| アルコール | 各国の法律に準拠。未成年ターゲティング禁止 |
| アダルト商品 | 成人向けフラグが必要。一部国では配信不可 |
| 医薬品・健康関連 | 各国の規制に準拠。処方薬は原則不可 |
| HFSS食品・飲料 | 高脂肪・高糖・高塩分の食品は未成年へのターゲティング禁止 |

### 9-3. 不承認の主な原因と対処

| 不承認理由 | 原因 | 対処法 |
|-----------|------|--------|
| 価格の不一致 | フィードの価格とLPの価格が異なる | フィード更新頻度を上げる。構造化データ（schema.org）を実装 |
| 在庫状況の不一致 | フィードでin_stockだがLPでは品切れ | 在庫連動の自動フィード更新を導入 |
| 画像の不備 | サイズ不足、テキストオーバーレイ、プレースホルダー | 第5章の要件に準拠して差し替え |
| GTIN/MPNの不備 | 識別子が不正確または欠落 | GTINバリデーターで検証。カスタム品はGTIN不要と申告 |
| LPが404エラー | リンク切れ | URL監視を導入。フィード送信前にリンクチェック |
| 配送情報の不備 | 配送料・配送日数が未設定 | GMCの配送設定を正確に記入 |
| ポリシー違反コンテンツ | 禁止・制限カテゴリの商品 | 該当商品をフィードから除外 |

### 9-4. 審査のエスカレーション

不承認に異議がある場合:
1. GMCの「診断」タブで不承認理由を確認
2. 問題を修正してフィードを再送信
3. 再審査を待つ（通常3〜5営業日）
4. 正当な場合は「再審査をリクエスト」から申請

**アカウント停止を防ぐ:** 不承認が繰り返されるとアカウント停止のリスクがある。エラーや警告は早期に対処し、放置しない。

## 10. 計測と評価

### 10-1. コンバージョン設定

ショッピング広告（EC）ではCV値（売上金額）の正確な計測が特に重要。

| 設定項目 | 推奨 |
|---------|------|
| プライマリCV | 購入完了（purchase） |
| CV値 | **動的な値（実売上金額）** をトランザクションごとに送信 |
| アトリビューション | データドリブンアトリビューション（DDA） |
| CV計測ウィンドウ | クリック後30日（デフォルト）。高額商品は60〜90日も検討 |

### 10-2. 主要KPI

| 指標 | 定義 | 目安 |
|------|------|------|
| ROAS | CV値 ÷ 広告費 | 損益分岐ROASは「1 ÷ 粗利率」で算出（粗利率30%→333%、40%→250%、50%→200%）。目標ROASは損益分岐を上回る水準で設定 |
| CTR | クリック ÷ 表示 | ショッピング広告全体の平均は業界により変動するが、改善余地のベンチマークに使用 |
| CVR | CV ÷ クリック | LP品質・価格競争力に大きく依存 |
| CPC | 費用 ÷ クリック | 商品カテゴリ・競合状況で変動 |
| インプレッションシェア | 表示回数 ÷ 表示可能回数 | 予算・入札の充足度を示す |

### 10-3. 利益ベースの評価

ROASだけでなく、**粗利ベースでの収益性**を評価する。

```
広告利益 = (売上 × 粗利率) − 広告費
```

例:
- 売上 100万円、粗利率 40%、広告費 25万円
- 広告利益 = (100万 × 0.4) − 25万 = **15万円**
- ROAS = 400%（一見良好だが、粗利率が低い商品ではROAS400%でも赤字になりうる）

### 10-4. 計測のよくある落とし穴

| 落とし穴 | 問題 | 対処法 |
|---------|------|--------|
| Google AdsのCVとGA4のCVが一致しない | アトリビューションモデル・CV計測ウィンドウ・カウント方式が異なる | **Google Adsの購入CVを入札最適化の「正」**とし、GA4はユーザー行動分析用に使い分ける |
| プライマリ/セカンダリCVの混乱 | プライマリCVが入札最適化に使われる。不要なCVをプライマリにすると学習が崩れる | **購入完了のみプライマリ**。マイクロCV（カート追加等）はセカンダリに |
| 拡張コンバージョン（Enhanced Conversions）未設定 | Cookie制限でCVの計測漏れが増加傾向 | ファーストパーティデータ（メールアドレス等）をハッシュ化して送信する拡張コンバージョンの導入を推奨 |
| レポートの指標がブレる | 「売上ROAS」と「粗利ROAS」で判断が異なる | レポート標準形を定義: **売上ROAS**（広告管理画面で確認）、**粗利ROAS**（粗利 ÷ 広告費。別途計算）、**広告利益**（粗利 − 広告費）の3指標で評価 |

### 10-5. 商品レベルの分析

商品単位で以下を定期的にチェック:

| 分類 | 条件 | アクション |
|------|------|-----------|
| 勝ち商品 | ROAS目標以上、CV安定 | 入札引き上げ、予算増加 |
| 育成中 | 表示多、CV少 | タイトル・画像・LP最適化 |
| 無駄遣い | 費用高、CV無し | 入札引き下げ or 除外 |
| 未露出 | 表示ほぼゼロ | 入札引き上げ、タイトル改善 |

## 12. Standard Shopping vs Performance Max

### 12-1. 比較表

| 項目 | Standard Shopping | Performance Max |
|------|------------------|-----------------|
| 配信面 | 検索結果、ショッピングタブ、画像検索 | 検索、ショッピング、ディスプレイ、YouTube、Gmail、Discover、Maps |
| キーワード制御 | 除外キーワード設定可 | キャンペーンレベルの除外キーワード対応（2025年1月〜。上限10,000件）。共有除外キーワードリストも対応（2025年8月〜） |
| 検索語句レポート | 詳細な検索語句レポートあり | 「検索語句のインサイト」（概要レベル） |
| 入札制御 | 手動CPC可 | スマート入札のみ |
| 商品グループ制御 | 詳細な商品グループ設定可 | アセットグループとリスティンググループ |
| 必要なアセット | 商品フィードのみ | 商品フィード + テキスト + 画像 + 動画 |
| 推奨CV件数 | 明確な最低件数なし | 30件/月以上推奨 |
| 透明性 | 高い | Standard Shoppingより低い |

### 12-2. 優先ロジックの変更（2024年）

**従来:** PMaxが同一商品でStandard Shoppingより自動的に優先されていた。

**現在:** **Ad Rank（広告ランク）で決定**される。同一商品が両キャンペーンに存在する場合、オークション時の広告ランクが高い方が配信される。（[公式ヘルプ](https://support.google.com/google-ads/answer/13825980)）

この変更により、Standard Shoppingの競争力が回復し、両方を併用するハイブリッド戦略がより有効になった。

※ 変更の正確な適用時期はアカウントにより異なる場合がある。自アカウントでの挙動を検索語句レポートで確認すること。

### 12-3. 使い分けの指針

| シチュエーション | 推奨 |
|----------------|------|
| 立ち上げ初期、CVデータが少ない | Standard Shopping（データ蓄積・学習） |
| 詳細な検索語句分析がしたい | Standard Shopping |
| 手動で入札を細かく制御したい | Standard Shopping |
| 全配信面で最大リーチを狙いたい | Performance Max |
| 十分なCVデータがある（30件/月以上） | Performance Max追加検討 |
| テキスト・画像・動画のアセットが揃っている | Performance Max |

### 12-4. PMaxの制御ポイント

PMaxは自動化が中心だが、以下のレバーで制御可能。

**アセットグループの分け方:**

| 分割軸 | 例 | メリット |
|--------|-----|---------|
| 商品カテゴリ別 | 家具 / 家電 / アパレル | カテゴリに最適なテキスト・画像アセットを設定 |
| 利益率別 | 高利益 / 低利益 | 利益率に応じた目標ROAS設定 |
| 新規獲得 vs リマーケ | 新規顧客獲得目標ON / OFF | 獲得コストの異なるユーザーを分離 |

**リスティンググループの設計:**

Standard Shoppingの商品グループと同様に、PMaxでもリスティンググループで商品の絞り込みが可能。カスタムラベルやカテゴリで分割し、アセットグループごとに異なる商品セットを配信する。

**「検索語句が見えにくい」前提での改善:**

PMaxは詳細な検索語句レポートがないため、以下で改善判断を行う:
- **検索語句のインサイト**: カテゴリ別の検索テーマと表示回数を確認
- **商品別パフォーマンス**: 商品単位のROAS・CV・費用で勝ち負けを判断
- **アセット別パフォーマンス**: テキスト・画像・動画ごとの評価（最良/良好/低）で差し替え判断

### 12-5. ハイブリッド運用

**推奨アプローチ:**

1. **Phase 1（1〜4週目）:** Standard Shoppingのみで開始。ベースラインデータを蓄積
2. **Phase 2（2ヶ月目〜）:** Standard Shoppingが安定し利益が出たら、PMaxを追加
3. **継続運用:** Standard Shoppingで実績のある商品を最適化しつつ、PMaxで新規顧客獲得を拡大

- Standard Shopping = **既知の勝ちパターンの最適化（Exploitation）**
- Performance Max = **新規発見・リーチ拡大（Exploration）**

## 13. 無料リスティング（Free Listings）

### 13-1. 概要

2020年から開始された無料掲載枠。商品フィードをGMCに送信するだけで、有料広告を出さなくてもGoogleの各面に商品が表示される可能性がある。

**表示場所:**
- Googleショッピングタブ
- Google検索（一部）
- Google画像検索
- Google Lens

### 13-2. 有料広告との違い

| 項目 | 有料ショッピング広告 | 無料リスティング |
|------|-------------------|----------------|
| 費用 | クリック課金（CPC） | 無料 |
| 表示位置 | ショッピングカルーセル（上部）の最上位 | ショッピングタブの下部、有料広告の後ろ |
| 制御性 | 入札・予算で制御可能 | Googleのアルゴリズム次第 |
| トラフィック規模 | 大（カルーセルがクリックの大半を獲得） | 有料に比べ限定的 |

### 13-3. Merchant Centerの活用機能

ショッピング広告のパフォーマンスを引き上げるMerchant Centerの追加機能。

**プロモーション（Merchant Promotions）:**

商品に「特別セール」「送料無料」等のプロモーションバッジを表示でき、CTR向上に寄与する。

| 設定項目 | 説明 |
|---------|------|
| プロモーションフィード | GMCでプロモーション情報を登録。`promotion_id`で商品フィードと紐付け |
| 対象範囲 | 全商品 or 特定商品（`promotion_id`を商品フィードにマッピング） |
| 条件設定 | 金額割引、割合割引、送料無料、BOGO等 |
| ターゲティング | 新規顧客限定、地域別のプロモーション条件が設定可能 |

**商品レビュー・評価（Product Ratings）:**

商品に星評価を表示でき、クリック判断の後押しになる。

- Google カスタマーレビュー、または第三者レビューサービスとの連携で取得
- 最低要件: 商品ごとに一定数のレビューが必要（国により異なる）
- レビューフィード（XML）をGMCに送信して連携

**自動アイテム更新（Automatic Item Updates）:**

- Googleがサイトをクロールし、フィードと異なる価格・在庫を自動修正
- 有効にすると不承認リスクが低減するが、意図しない修正の可能性もある
- **推奨**: 有効にしつつ、フィード自体の更新も毎日行う（自動更新はあくまで補助）

**価格・在庫の整合性監視:**

- LPに構造化データ（schema.org `Product`マークアップ: `price`、`availability`）を実装すると、Googleの自動検証精度が向上
- 404エラーや価格差分の検知を外部ツールや自前スクリプトで定期監視すると、不承認の予防に効果的

### 13-4. 無料リスティングの最適化

有料広告を出さなくても、フィード品質を高めれば無料リスティングの表示機会が増える。

- 全ての推奨属性（GTIN、google_product_category、色、サイズ等）を漏れなく記入
- 商品タイトル・説明文を最適化（第4章参照）
- 高品質な画像を使用（第5章参照）
- フィードの更新頻度を上げる（毎日推奨）
- 商品レビュー・評価を連携

## 14. よくある失敗パターンと回避策

| # | 失敗パターン | 原因 | 回避策 |
|---|------------|------|--------|
| 1 | 新規キャンペーンで最初から目標ROASを設定 | CVデータが不足しAIが正しく学習できない | 手動CPCで開始し、30件以上のCV蓄積後に移行 |
| 2 | 商品タイトルが短く一般的 | 検索マッチの精度が低く、表示機会を逃す | カテゴリ別のタイトル構成に従い、属性を含めた詳細タイトルに |
| 3 | フィード更新が月1回以下 | 価格・在庫の不一致で不承認が増加 | 毎日のフィード更新を自動化 |
| 4 | 除外キーワードを設定していない | 無関係な検索クエリに広告費が流出 | 初期除外リスト設定 + 週次の検索語句レポート確認 |
| 5 | 「その他すべて」の入札を放置 | 未分類の低パフォーマンス商品に予算が流出 | 低入札またはフィードから除外 |
| 6 | カスタムラベルを使っていない | 利益率・季節性に基づく入札制御ができない | 最低限 利益率 + 販売パフォーマンスの2軸でラベル設定 |
| 7 | 全商品を1つのキャンペーンで管理 | 高利益商品と低利益商品が同じ入札になる | 利益率・パフォーマンスでキャンペーン分離 |
| 8 | 画像が低品質・小サイズ | CTRが低下し、広告の競争力が落ちる | 1,500×1,500px以上の高品質画像を用意 |
| 9 | LPが商品ページでない | カテゴリページやトップページに遷移すると離脱率が上がる | フィードのlinkは必ず個別商品ページのURLに |
| 10 | Standard ShoppingかPMaxの二者択一 | 片方の利点を活かせない | ハイブリッド運用でExploitation + Explorationの両立 |
| 11 | プロモーション機能を活用していない | CTR向上の機会損失（プロモーションバッジ非表示） | GMCのプロモーション設定を活用（CTR向上に寄与） |
| 12 | 「設定して放置」 | 市場変化・競合変化に対応できず、パフォーマンスが劣化 | 日次・週次・月次のチェックサイクルを必ず回す |
