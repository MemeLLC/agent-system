# Google 動画キャンペーン（YouTube広告）

---

## 用語定義

| 略語 | 正式名称 | 定義 |
|------|---------|------|
| CPV | Cost Per View | 動画1再生あたりの費用。スキッパブル広告で使用 |
| CPM | Cost Per Mille | 1,000インプレッションあたりの費用 |
| tCPV | Target CPV | 目標視聴単価。設定した単価を目安にGoogleが入札を自動調整 |
| tCPM | Target CPM | 目標インプレッション単価。設定した単価平均になるよう自動調整 |
| VTR | View-Through Rate | 視聴率。インプレッションに対する視聴完了の割合 |
| CTR | Click-Through Rate | クリック率。インプレッションに対するクリック数の割合 |
| CTA | Call To Action | 行動喚起。広告上でユーザーにクリック・登録等を促すボタンやテキスト |
| CTV | Connected TV | インターネット接続されたテレビ。スマートTV、ストリーミングデバイスなど |
| VACa | Video Action Campaign | 動画アクションキャンペーン。2025年7月にDemand Genへ自動移行済 |
| Demand Gen | Demand Generation | YouTube・Discover・Gmailに配信するCV獲得向けキャンペーン。ディスプレイキャンペーン（GDN）とは別枠 |
| ABCD | Attract, Brand, Connect, Direct | Google推奨の動画クリエイティブフレームワーク |
| Brand Lift | ブランドリフト | 広告接触による認知・想起・好意度の変化を測定する調査 |

## 目的別キャンペーン選択早見表

| 目的 | キャンペーンサブタイプ | 主な広告フォーマット | 入札戦略 | 課金方式 |
|------|---------------------|-------------------|---------|---------|
| **認知最大化（効率重視）** | Video Reach - Efficient Reach | バンパー + スキッパブル + インフィード + Shorts | tCPM | インプレッション |
| **認知最大化（完全視聴）** | Video Reach - Non-Skippable | ノンスキッパブル + バンパー | tCPM | インプレッション |
| **接触頻度の管理** | Video Reach - Target Frequency | バンパー + スキッパブル + ノンスキッパブル | tCPM | インプレッション |
| **視聴・エンゲージメント** | Video Views | スキッパブル + インフィード + Shorts | tCPV | 視聴 |
| **ストーリーテリング** | Ad Sequence | スキッパブル + ノンスキッパブル + バンパーの組み合わせ | tCPM / tCPV | 混合 |
| **コンバージョン獲得** | Demand Gen | スキッパブル + インフィード + Shorts + 画像 | tCPA / 最大化CV / tROAS | クリック / インプレッション（※最適化目標はCV） |
| **音声リーチ** | Audio Reach | 音声広告（15秒） | tCPM | インプレッション |

> **注**: 2025年4月以降、Video Action Campaign（VAC）の新規作成は不可。コンバージョン目的の動画広告はDemand Genキャンペーンを使用する。

## 1. 広告フォーマット一覧（YouTube固有）

### 1-1. スキッパブル インストリーム広告

動画キャンペーンの基本フォーマット。

| 項目 | 仕様 |
|------|------|
| 表示場所 | 動画の前・中・後（プレロール/ミッドロール/ポストロール） |
| 長さ | 制限なし（推奨: 15秒〜3分。認知目的は15〜20秒、検討目的は60秒〜3分） |
| スキップ | 5秒後にスキップ可能 |
| 課金 | CPV（30秒視聴 or 動画終了 or クリックのいずれか早い方） |
| CTA | オーバーレイ、コンパニオンバナー、サイトリンク |
| デバイス | デスクトップ、モバイル、タブレット、CTV |

### 1-2. ノンスキッパブル インストリーム広告

メッセージの完全視聴を保証したい場合に使用。

| 項目 | 仕様 |
|------|------|
| 表示場所 | 動画の前・中・後 |
| 長さ | 15〜30秒（地域により異なる。CTV在庫では最大30秒） |

| 課金 | CPM（インプレッション課金） |
| デバイス | デスクトップ、モバイル、タブレット、CTV |

> **注意**: ノンスキッパブル広告ではTrueView「視聴」がカウントされないため、視聴ベースのリマーケティングリスト構築には不向き。インプレッションベースのオーディエンスリストは作成可能だが、視聴意思を持ったユーザーの選別精度は下がる点に留意。

### 1-3. バンパー広告

短いメッセージで高頻度のリーチを実現するフォーマット。

| 項目 | 仕様 |

| 長さ | 6秒以下 |
| スキップ | 不可 |
| 課金 | CPM |
| 推奨用途 | ブランド想起の強化、スキッパブル広告の補完、Ad Sequenceの一部 |

### 1-4. インフィード動画広告

### 1-5. YouTube Shorts広告

## 2. キャンペーンの目的とサブタイプ

### 2-1. 目的（Objective）→ サブタイプの対応

Google Adsでは、キャンペーン作成時にまず目的を選択し、その目的に応じて利用可能なサブタイプが決まる。

| キャンペーン目的 | 利用可能なサブタイプ |
|---------------|------------------|
| **Awareness and consideration** ※ | Video Reach（Efficient Reach / Non-Skippable / Target Frequency）、Video Views、Ad Sequence、Audio Reach |
| **Sales / Leads / Website traffic** | Demand Gen（旧 Drive Conversions） |
| **目的を設定しない** | 上記すべてのサブタイプから選択可能 |

> ※ 新規キャンペーンでは「YouTube reach, views, and engagements」と表示される場合がある。UIの名称はアップデートにより変更される可能性がある。

### 2-2. サブタイプ詳細

#### Video Views（動画視聴キャンペーン）

- **目的**: 視聴数とエンゲージメントの最大化
- **入札**: tCPV（Target Cost Per View）
- **視聴カウント**: 30秒視聴 or 動画終了（Shortsは10秒）
- **利用可能フォーマット**: スキッパブル + インフィード + Shorts
- **推奨**: 動画キャンペーンの標準的な開始点。迷ったらまずこれ

#### Video Reach - Efficient Reach

- **目的**: 予算内での最大リーチ（コスト効率重視）
- **入札**: tCPM
- **利用可能フォーマット**: バンパー + スキッパブル + インフィード + Shorts
- **特徴**: Google AIが配信面を自動最適化してリーチを最大化

#### Video Reach - Non-Skippable Reach

- **目的**: メッセージの完全視聴保証
- **入札**: tCPM
- **利用可能フォーマット**: バンパー + ノンスキッパブル（15秒）
- **注意**: TrueView視聴がカウントされないため、視聴ベースのリマーケティングリストには不向き（インプレッションベースは可能）

#### Video Reach - Target Frequency

- **目的**: 同一ユーザーへの接触頻度のコントロール
- **入札**: tCPM
- **利用可能フォーマット**: バンパー + スキッパブル + ノンスキッパブル
- **推奨**: 大規模ブランドキャンペーン向け。中小規模では予算効率が悪くなりやすい

#### Ad Sequence（広告シーケンス）

- **目的**: 複数の動画を指定した順序で配信し、ストーリーを伝える
- **入札**: tCPM / tCPV（フォーマットによる）
- **利用可能フォーマット**: スキッパブル + ノンスキッパブル + バンパーの組み合わせ
- **構成例**:
  - 6秒バンパー（ティーザー）→ 15秒スキッパブル（詳細）→ 6秒バンパー（リマインド）
  - 30秒スキッパブル（ストーリー前半）→ 30秒スキッパブル（ストーリー後半）

#### Audio Reach

- **目的**: 音声リスニング中のユーザーへのリーチ
- **入札**: tCPM
- **フォーマット**: 15秒ノンスキッパブル音声広告
- **推奨**: YouTube Musicリスナー、バックグラウンド再生ユーザーへのリーチ

## 3. 入札戦略と予算設計（動画キャンペーン固有）

### 3-1. 入札戦略の選択

| 入札戦略 | 課金方式 | 対応サブタイプ | 目安の単価水準 |
|---------|---------|-------------|-------------|
| **Target CPV** | 視聴ごと | Video Views | $0.02〜$0.10（グローバル平均 約$0.03。業界・地域・時期で大きく変動） |
| **Target CPM** | 1,000 imp | Video Reach、Ad Sequence、Audio | $5〜$10（グローバル平均 約$9。業界・地域・時期で大きく変動） |
| **Maximize Conversions** | CVごと | Demand Gen | 業界・商材による |
| **Target CPA** | CVごと | Demand Gen | 目標CPA設定 |
| **Target ROAS** | 売上比 | Demand Gen | 目標ROAS設定 |
| **Maximize Conversion Value** | 売上最大化 | Demand Gen | 予算上限内で売上最大化 |

> **注**: Maximum CPVは2025年にTarget CPVへ移行。新規キャンペーンではTarget CPVを使用する。既存のMaximum CPVキャンペーンはそのまま運用可能。

### 3-2. 予算設計の基本

**認知目的（CPM課金）**

| 項目 | 推奨値 |
|------|-------|
| 日予算の下限 | tCPM × 想定日間imp数 / 1,000 |
| 最低期間 | 2週間以上（学習期間を確保） |
| 検証予算の目安 | 月10万円〜（小規模テスト） |

**コンバージョン目的（Demand Gen）**

| 項目 | 推奨値 |
|------|-------|
| tCPA使用時の日予算 | 目標CPA × 15倍以上 |
| 最大化CV使用時の日予算 | 想定CPAの × 10倍以上 |
| CV蓄積目標 | 月15件以上（Smart Biddingの安定稼働目安） |

### 3-3. 予算変更のルール

- 1回の変更幅: **最大20%**
- 変更間隔: **最低7〜14日**
- 入札と予算の同時変更を避ける（片方ずつ変更して効果を分離）
- 学習期間中（開始後7日間目安）は変更を控える

### 3-4. 学習期間（Learning Period）

| トリガー | 内容 |
|---------|------|
| キャンペーン新規作成 | 約7日間の学習期間 |
| 予算を20%以上変更 | 学習リセットの可能性 |
| 入札戦略の変更 | 学習リセット |
| 広告の大幅な追加・変更 | 学習リセットの可能性 |
| ターゲティングの大幅変更 | 学習リセットの可能性 |

## 4. ターゲティング設計（YouTube固有）

### 4-1. オーディエンスターゲティング

| ターゲティング種別 | 概要 | 推奨用途 |
|-----------------|------|---------|
| **アフィニティ** | 趣味・関心・習慣に基づくセグメント | 認知拡大。幅広いリーチ |
| **カスタムセグメント** | 自分で指定したキーワード・URL・アプリに基づく | 検討促進。競合サイト訪問者や特定検索ユーザーにリーチ |
| **購買意向（In-Market）** | 購入・契約を検討中のユーザー | CV獲得。購買意欲の高い層にリーチ |
| **ライフイベント** | 就職・結婚・引越しなどの節目 | 高単価商材の訴求 |
| **詳しいユーザー属性** | 世帯年収・学歴・住宅所有状況など | ターゲット層の絞り込み |
| **リマーケティング** | サイト訪問者・動画視聴者・アプリユーザー | CVR向上。既接触ユーザーへの再アプローチ |
| **カスタマーマッチ** | CRMデータ（メールアドレス等）のアップロード | 既存顧客向け施策、Lookalike Segments（Demand Gen）のシードリスト |
| **Lookalike Segments** | 既存顧客に似た特性を持つユーザー（Demand Genのみ利用可能） | 新規獲得の拡大。利用できない場合はOptimized Targetingで代替 |

### 4-2. コンテンツターゲティング

| ターゲティング種別 | 概要 | 注意点 |
|-----------------|------|-------|
| **トピック** | 特定トピックのコンテンツに配信 | リーチは広いがCV効率は下がりやすい |
| **プレースメント** | 特定のYouTubeチャンネル・動画を指定 | 細かい制御が可能だがスケールしにくい |
| **キーワード** | 動画のメタデータ・コンテンツに基づく | ディスプレイのキーワードとは異なる |

> **重要**: Demand Gen（CV目的）キャンペーンでは、コンテンツターゲティング（キーワード・トピック・プレースメント）を追加すると**配信が大幅に制限**される。CV目的ではオーディエンスターゲティング + 最適化されたターゲティング（Optimized Targeting）の組み合わせを推奨。

### 4-3. デバイス・地域ターゲティング

- **デバイス**: モバイル、デスクトップ、タブレット、CTV（コネクテッドTV）から選択可能
- **CTV指定**: CTVのみの配信も可能（テレビ画面向けクリエイティブとの組み合わせに有効）
- **地域**: 国・都道府県・市区町村・半径指定

### 4-4. 最適化されたターゲティング（Optimized Targeting）

- 手動設定したオーディエンスの外にも、CVの可能性が高いユーザーに自動拡張
- Demand Genでは**ON推奨**（手動オーディエンスだけでは配信量が不足しやすい）
- 認知キャンペーンでは、意図しない拡張を避けるためOFFも検討

## 5. 動画クリエイティブの入稿仕様

### 5-1. 動画ファイルの基本仕様

| 項目 | 仕様 |
|------|------|
| アップロード先 | YouTube（事前にYouTubeに動画をアップロードし、そのURLを広告に使用） |
| ファイル形式 | MP4（推奨）、MOV、AVI、WMV、FLV、WebM、MPEG、ProRes、HEVC等 |
| 最大ファイルサイズ | 256GB |
| フレームレート | 30fps推奨 |
| 音声コーデック | AAC-LC推奨 |
| 映像コーデック | H.264推奨 |

### 5-2. アスペクト比と解像度

| 向き | アスペクト比 | 推奨解像度 | 最低解像度 | 用途 |
|------|-----------|----------|----------|------|
| 横型 | 16:9 | 1920 × 1080 | 1280 × 720 | インストリーム、CTV、デスクトップ |
| 縦型 | 9:16 | 1080 × 1920 | 720 × 1280 | Shorts、モバイルフルスクリーン |
| スクエア | 1:1 | 1080 × 1080 | 480 × 480 | インフィード、一部配信面 |
| その他横型 | 4:3 | 1440 × 1080 | 960 × 720 | レガシー対応 |

> **推奨**: 横型（16:9）+ 縦型（9:16）の**2種類をセットで用意**する。これにより全配信面を最適にカバーできる。縦型動画の追加でモバイルでのエンゲージメントが向上し、CV率が20%以上改善した事例もある（Google発表値。業界・商材により変動）。

**最低制作セット:**

| 優先度 | アスペクト比 | 用途 | 備考 |
|-------|-----------|------|------|
| **必須** | 16:9（横型） | インストリーム、CTV、デスクトップ | 最も配信面が広い基本フォーマット |
| **必須** | 9:16（縦型） | Shorts、モバイルフルスクリーン | セーフゾーンを考慮したレイアウト。横型のトリミングではなく縦型ネイティブで制作 |
| 推奨 | 1:1（スクエア） | インフィード、一部配信面 | 配信面の取りこぼし対策。予算に余裕がある場合 |

### 5-3. フォーマット別の長さ要件

| フォーマット | 最小 | 推奨 | 最大 |
|------------|------|------|------|
| スキッパブル インストリーム | 制限なし | 15秒〜3分 | 制限なし |
| ノンスキッパブル インストリーム | 15秒 | 15秒 | 30秒（CTV在庫） |
| バンパー | — | 6秒 | 6秒 |
| Shorts | 5秒以上 | 10〜60秒 | 60秒 |
| インフィード | 5秒以上 | 60秒〜3分 | 制限なし |
| 音声広告 | — | 15秒 | 15秒 |

### 5-4. テキストアセット（広告に付随するテキスト）

| 要素 | 文字数上限 | 備考 |
|------|----------|------|
| 見出し | 15文字（半角30文字） | CTAボタン付近に表示 |
| 長い見出し | 45文字（半角90文字） | インフィード等の一部配信面 |
| 説明文 | 35文字 × 2行（半角70文字） | インフィード広告用 |
| CTA（行動喚起フレーズ） | 10文字（半角） | 「詳しく見る」「今すぐ購入」など |
| 最終ページURL | — | ランディングページのURL |
| 表示URL | ドメイン + パス（15文字 × 2） | 広告に表示されるURL |

## 6. ABCDフレームワーク（クリエイティブ原則）

GoogleがIpsos・Nielsen Neuro・Kantarとの共同研究に基づいて開発したフレームワーク。ABCDに従った広告は、短期売上で最大30%、長期ブランド貢献で17%のリフトが報告されている（Google / Ipsos共同調査。効果はクリエイティブ品質・業界・キャンペーン設計により大きく変動する）。

### 6-1. A — Attract（注意を引く）

最初の5秒が勝負。スキッパブル広告では5秒でスキップ判断が行われる。

**実践テクニック:**

| テクニック | 説明 |
|----------|------|
| **タイトフレーミング** | 被写体を大きく映し、一目で何の話か伝える |
| **人物の登場** | 冒頭から人の顔・表情を見せることで注意を引きつける |
| **意外性のあるビジュアル** | 予想外の映像で「何だろう？」と思わせる |
| **テンポの速いカット** | 最初の5秒に2〜3カット以上。静止は禁物 |
| **問いかけ** | 「〜で困っていませんか？」など視聴者に語りかける |
| **ダブルフック** | 冒頭フック（0〜5秒）+ セカンドフック（5〜10秒）で離脱を2段階防止 |

### 6-2. B — Brand（ブランドを見せる）

**実践テクニック:**

| テクニック | 説明 |
|----------|------|
| **冒頭5秒以内に表示** | ロゴ・ブランド名・商品を最初の5秒で見せる |
| **音声での言及** | ナレーションやBGMでブランド名を耳に残す |
| **繰り返し露出** | 冒頭・中盤・終盤の3回ブランドを表示する |
| **商品の実使用シーン** | 実際に使っている場面を見せてブランドと体験を結びつける |

### 6-3. C — Connect（共感をつくる）

**実践テクニック:**

| テクニック | 説明 |
|----------|------|
| **ユーモア** | 面白さは広告受容性を最も高める要素（Google調査） |
| **ストーリーテリング** | 緊張→解決のストーリーアークで感情を動かす |
| **ターゲットの課題提示** | 視聴者が「自分のことだ」と感じる悩み・課題を提示 |
| **実際のユーザーの声** | レビュー・体験談で信頼と共感を構築 |
| **音楽の活用** | BGMは感情を増幅する。Shorts広告ではサウンド使用でCVが20%以上向上した事例が報告されている（Google発表。効果は商材・クリエイティブにより変動） |

### 6-4. D — Direct（行動を促す）

**実践テクニック:**

| テクニック | 説明 |
|----------|------|
| **明確なCTA** | 「今すぐ申し込む」「詳しくはこちら」など具体的な動詞 |
| **テキストカード/アニメーション** | CTAを画面上にテキスト表示 + 音声で二重に伝える |
| **緊急性の演出** | 「期間限定」「残りわずか」で行動を加速 |
| **特典の提示** | 「初回50%OFF」「無料トライアル」など具体的なオファー |
| **URLの表示** | 遷移先URLを画面に表示して安心感を与える |

## 7. フォーマット別クリエイティブ設計

### 7-1. スキッパブル インストリーム（15秒〜3分）

**最も汎用的なフォーマット。ABCDフレームワークをフルに活用できる。**

| 秒数 | 内容 |
|------|------|
| 0〜5秒 | フック + ブランド表示（ここで視聴継続を決定） |
| 5〜15秒 | 課題提示 → ソリューション提示 |
| 15〜25秒 | 証拠・ベネフィットの深掘り |
| 最後3〜5秒 | CTA + ブランド再表示 |

**制作のポイント:**
- 5秒でスキップされても、ブランド名と核心メッセージが伝わる設計にする
- 長さの目安: 認知目的なら15〜30秒、検討目的なら30秒〜2分
- 字幕（キャプション）を必ず追加。ミュート再生や聴覚障害への対応

### 7-2. ノンスキッパブル（15〜30秒）

**完全視聴が保証されるため、ストーリーを完結させやすい。**

| 秒数 | 内容 |
|------|------|
| 0〜3秒 | 注意を引くオープニング（フックの重要度はスキッパブルより低い） |
| 3〜12秒 | 課題 → ソリューション → ベネフィット |
| 12〜15秒 | CTA + ブランド締め |

**制作のポイント:**
- CTV視聴者が増加しているため、映画的な品質感が効果的
- 15秒で1つのメッセージに絞る。詰め込みすぎない
- テレビCMの流用は可能だが、冒頭にフックを追加する編集を推奨

### 7-3. バンパー（6秒）

**1メッセージ・1ビジュアル。引き算の美学。**

| 秒数 | 内容 |
|------|------|
| 0〜2秒 | ビジュアルインパクト |
| 2〜4秒 | 核心メッセージ（1文） |
| 4〜6秒 | ブランド + CTA |

**制作のポイント:**
- 伝えるメッセージは1つだけ
- テキストオーバーレイを活用（音声なしでも伝わる設計）
- 長尺スキッパブル広告のリマインド版として使うと効果的
- Ad Sequenceのパーツとして活用

### 7-4. インフィード動画広告

**サムネイルが命。ユーザーの自発的クリックを勝ち取る必要がある。**

**サムネイル設計:**
- 自動生成の3候補 or カスタムサムネイルから選択
- テキストオーバーレイ入りのサムネイルがCTR向上に有効
- サムネイルの人物画像はCTRを高める傾向

**動画設計:**
- 長尺コンテンツ（2〜5分）も有効。視聴者が能動的にクリックしているため
- チュートリアル・解説・レビュー形式が相性良い
- 冒頭で「この動画で学べること」を提示

## 8. YouTube Shorts広告の設計

### 8-1. Shorts広告の特徴

- 縦型フルスクリーン（9:16）のモバイルファーストな体験
- オーガニックのShorts投稿の間に表示される
- デスクトップ・タブレット・CTVにも配信されるが、モバイル最適化が基本
- チャンネル名とチャンネル説明文（2行 × 35文字）が表示される

### 8-2. Shorts広告の推奨仕様

| 項目 | 仕様 |
|------|------|
| アスペクト比 | 9:16（縦型必須） |
| 解像度 | 1080 × 1920（推奨） / 720 × 1280（最低） |
| 長さ | Demand Gen / アプリ / PMax: 10〜30秒 / Video Views: 30〜60秒 |
| サウンド | 強く推奨。音楽・ナレーション・両方のいずれか。無音の場合は字幕・テキストオーバーレイで補完必須 |

### 8-3. Shorts向けクリエイティブのベストプラクティス

| ポイント | 説明 |
|---------|------|
| **ソーシャルファースト** | 広告っぽさを抑え、オーガニック投稿に馴染む質感 |
| **縦型ネイティブ** | 横型動画のトリミングではなく、縦型で企画・撮影する |
| **冒頭で引き込む** | 最初の2〜3秒でスワイプを止めるフック |
| **テンポよく** | 1つのカットは2〜3秒以内。ジャンプカット多用 |
| **サウンド活用** | BGMまたはナレーション使用でCVが20%以上向上した事例が報告されている（Google発表。効果は商材により変動） |
| **テキストオーバーレイ** | 画面内にテキストを配置。セーフゾーンに注意 |
| **UGC風** | ユーザー生成コンテンツ風の自然な映像が高パフォーマンスの傾向 |

### 8-4. Shortsのセーフゾーン

縦型動画のUI要素（いいね・コメント・共有ボタン等）に重要な情報が隠れないよう注意。

- 上部: チャンネル名やステータスバーを避ける
- 下部: 説明文やCTAボタンエリアを避ける
- 右側: いいね・コメント・共有ボタンエリアを避ける
- **安全領域**: 画面中央の約80%を使用する

## 9. CTV（コネクテッドTV）への最適化

### 9-1. CTV視聴の特徴

- 米国ではYouTubeのTV画面視聴シェアがNetflix・Hulu・Prime Videoを超えている（2025年時点の米国データ。地域により状況は異なる）
- CTVでの広告は視聴完了率が最大96%に達するとの報告がある（Google発表。広告フォーマット・業界により変動）
- CTV視聴の約26%はグループ視聴（1回の広告表示で複数人にリーチ）
- CTVでのYouTube広告はリニアTV広告比で広告想起が約10%高いとの調査結果がある

### 9-2. CTV向けクリエイティブのポイント

| ポイント | 説明 |
|---------|------|
| **映画的品質** | テレビ画面の大画面に耐える高解像度・高品質な映像 |
| **ストーリー重視** | ノンスキッパブル広告と相性が良い。じっくり見せる構成 |
| **ペーシング** | テレビ視聴のペースに合わせ、やや落ち着いたテンポ |
| **ブランド訴求** | 画面が大きいため、ロゴ・商品の視認性が高い |
| **タレント活用** | 出演者がブランド名を口頭で言及すると効果的 |
| **QRコード** | CTVにはクリック導線がないため、QRコードで遷移を促す手法も有効 |

### 9-3. CTV特有の設定

- Google Adsのデバイスターゲティングで「テレビ画面」を選択可能
- CTVではバンパー・ノンスキッパブルの完全視聴が期待できる
- CTV在庫ではノンスキッパブル広告が最大30秒まで配信可能

## 10. Demand Genキャンペーン（動画アクションの後継）

### 10-1. VAC → Demand Gen の移行経緯

| 時期 | 内容 |
|------|------|
| 2025年4月 | Video Action Campaign（VAC）の新規作成が不可に |
| 2025年7月 | 既存VACがDemand Genへ自動移行 |

### 10-2. Demand Genの特徴

| 項目 | VAC（旧） | Demand Gen（現） |
|------|---------|---------------|
| 配信面 | YouTube | YouTube + Discover + Gmail（※ディスプレイキャンペーン（GDN）とは別枠。配信面は今後拡張される可能性あり） |
| クリエイティブ | 動画のみ | 動画 + 画像（動画+画像併用で同CPA比20%のCV増。Google発表値） |
| チャンネル制御 | 限定的 | YouTube Shorts単独配信など詳細制御可能 |
| Lookalike Segments | — | シードリストに類似した新規ユーザーにリーチ。利用可否はアカウント・地域で異なる場合あり |
| ROAS実績 | — | VACと比較してROASが平均58%向上との報告（Nielsen分析。実績はアカウント・業界により大きく変動） |

### 10-3. Demand Genの設定ポイント

- **入札**: tCPA or Maximize Conversions から開始。データ蓄積後にtROASへ移行
- **予算**: tCPA × 15倍を日予算に設定
- **クリエイティブ**: 動画 + 画像の両方をアップロード（片方だけだと配信面が限定される）
- **ターゲティング**: カスタムセグメント + カスタマーマッチ + Lookalike Segments（利用可能な場合） + Optimized Targeting ON
- **コンテンツターゲティング**: 追加しない（配信制限の原因になる）
- **配信面**: 最初は全配信面ONにして、データを見ながら絞り込む
- **CV設計**: 軽量CVイベント（カート追加、サイト訪問等）の活用で学習を加速
- **運用期間**: 最低4週間は継続運用する（最大15%の効率改善が見込める）

### 10-4. Demand Gen運用の変更ルール（学習を壊さないために）

Demand Genは学習期間への感度が高い。以下のルールで変更事故を防止する。

| ルール | 説明 |
|-------|------|
| **同時変更禁止** | 目標（tCPA等）・予算・主要アセットを同時に変更しない。1つずつ変更し、7〜14日間の効果確認を挟む |
| **変更ログの記録** | いつ・何を・どの値からどの値に変更したかを記録。パフォーマンス変動の原因特定に必須 |
| **大幅な目標変更はキャンペーン複製** | 目標CPAを30%以上変更する場合は、既存キャンペーンを残したまま新キャンペーンで段階移行する |
| **アセット追加のタイミング** | 学習期間中（開始後7日間目安）はアセットの追加・削除を控える |
| **配信面変更の慎重さ** | チャンネル制御（YouTube / Discover / Gmail）の変更は学習リセットのリスクあり。変更は予算・目標が安定してから |

## 11. 計測と評価指標（動画固有）

### 11-1. 主要指標一覧

| 指標 | 定義 | 主な用途 |
|------|------|---------|
| **TrueView再生数** | 30秒視聴 or 動画終了 or クリック（2025年10月に「Views」から名称変更） | Video Viewsの主要KPI |
| **視聴率（VTR）** | TrueView再生数 ÷ インプレッション数 | クリエイティブの訴求力を評価 |
| **平均視聴率** | グローバル平均 約31.9%（2024〜2025年データ。業界・地域・フォーマットにより大きく変動） | ベンチマーク参考値 |
| **CTR** | クリック数 ÷ インプレッション数 | 行動喚起の強さを評価 |
| **平均CTR** | グローバル平均 約0.65%（2024〜2025年データ。業界・地域・フォーマットにより大きく変動） | ベンチマーク参考値 |
| **視聴完了率** | 動画を最後まで視聴した割合 | 動画の内容維持力を評価 |
| **CPV** | 費用 ÷ TrueView再生数 | 視聴コスト効率 |
| **CPM** | 費用 ÷ インプレッション数 × 1,000 | リーチコスト効率 |
| **CV数** | コンバージョン数 | Demand Genの主要KPI |
| **CPA** | 費用 ÷ CV数 | CVコスト効率 |
| **ビュースルーCV** | 広告を視聴したが直接クリックせず、後にCVした数 | 動画広告の間接効果 |
| **クロスデバイスCV** | あるデバイスで視聴 → 別デバイスでCV | マルチデバイス効果 |

### 11-2. Brand Lift調査

動画広告の認知・検討へのインパクトを測定するGoogleの公式調査ツール。

| 項目 | 詳細 |
|------|------|
| 測定可能指標 | 広告想起、ブランド認知、ブランド検討、好意度、購入意向 |
| 必要回答数 | 1指標あたり約4,100回答 |
| 設定方法 | Google Ads > 計測 > リフト測定 > Brand Lift |
| 費用 | 追加費用なし（広告配信費に含まれる） |
| 推奨予算 | 指標検出に十分なインプレッション数が必要（Google担当者に確認推奨） |

### 11-3. 目的別の評価指標セット（見るべき指標 / 見ないべき指標）

| キャンペーン目的 | 主要KPI（これで判断） | 補助KPI（参考にする） | 見ないべき指標（判断を誤る原因） |
|---------------|--------|--------|--------|
| 認知 | リーチ数、CPM、Brand Lift | フリクエンシー、広告想起率 | CPA、CVR（認知キャンペーンでCV効率を評価すると過小評価になる） |
| 検討 | VTR、視聴完了率、CPV | エンゲージメント率、チャンネル登録数、GA4サイト来訪 | CPA単体（動画は検討段階の貢献が大きいため、直接CVでは測れない） |
| CV獲得 | CV数、CPA、ROAS | ビュースルーCV、クロスデバイスCV、CPC | VTR（CV最適化キャンペーンでは視聴率よりCVの質・量で判断） |

> **ビュースルーCVの扱い**: ビュースルーCVは主要KPIとは別枠で追跡する。直接CVと合算せず、「動画の間接貢献」として独立で評価すること。合算するとCPAが不当に低く見え、スケーリング判断を誤るリスクがある。

## 12. ブランドセーフティと配信面管理

### 12-1. インベントリタイプ（配信在庫の品質レベル）

| タイプ | 内容 | 推奨対象 |
|-------|------|---------|
| **拡張（Expanded）** | 収益化基準を満たす全動画に配信。過激な言葉やゲームの暴力表現を含む可能性あり | 最大リーチ重視 |
| **標準（Standard）** | 大部分のブランドに適した範囲。人気音楽動画、ドキュメンタリーなど | **推奨（デフォルト）** |
| **制限（Limited）** | 不適切な言語・性的示唆を厳しく除外。一部人気コンテンツも除外される | ブランドガイドラインが厳格な場合 |

> **注意（調査時点: 2026-02-23）**: インベントリタイプはアカウントレベル（広告主レベル）で設定され、キャンペーン個別の変更は不可になっている。設定変更はUIが変わる可能性があるため、確認先: Google Ads > アカウント設定 > ブランドの適合性（Brand suitability）。

### 12-2. コンテンツ除外カテゴリ

以下のカテゴリを広告配信先から除外可能:

- 悲劇・紛争
- デリケートな社会問題
- 過激・衝撃的な内容
- 性的な暗示
- 冒涜的な内容

### 12-3. プレースメント除外

| 除外対象 | 設定レベル | 上限 |
|---------|----------|------|
| 特定のYouTubeチャンネル | アカウント / キャンペーン | 合計65,000件/アカウント |
| 特定の動画 | アカウント / キャンペーン | 同上 |
| 特定のウェブサイト・アプリ | アカウント / キャンペーン | 同上 |
| 1回の適用上限 | — | 20,000件 |

- アカウントレベルプレースメント除外が導入されており（調査時点）、PMax・Demand Gen・YouTube・Displayの全キャンペーンに一括適用可能。確認先: Google Ads > アカウント設定 > プレースメントの除外リスト
- サードパーティ除外リスト（DoubleVerify、IAS、Zefr）も利用可能

### 12-4. フリクエンシーキャップ

| 設定レベル | 内容 |
|----------|------|
| キャンペーン | 日/週/月単位でインプレッション数の上限を設定 |
| 推奨設定 | 認知: 週3〜5回 / 検討: 週2〜3回 / リマーケティング: 週5〜7回 |

## 13. 動画クリエイティブ運用

### 13-1. クリエイティブのリフレッシュサイクル

| 指標 | 推奨 |
|------|------|
| リフレッシュ間隔 | 2〜4週間ごと（高成長アカウントの平均は18.2日） |
| バリエーション数 | 最低3〜5本（高予算帯では10本以上が理想） |
| テスト方法 | Google Ads動画実験機能（A/Bテスト: 2アーム、カスタム: 最大10アーム） |
| テスト優先順位 | 1. フック（冒頭5秒） → 2. オファー内容 → 3. ビジュアル → 4. CTA |

**リフレッシュ間隔とパフォーマンスの関係（業界データ。Adzoola 2024-2025年分析。対象は主に欧米市場。日本市場では傾向が異なる可能性あり）:**

| リフレッシュ間隔 | 月間成長率（目安） |
|---------------|---------------|
| 14日未満 | +1.67% |
| 14〜30日 | +0.42% |
| 30日超 | -0.95% |

### 13-2. A/Bテストのルール

- 1回のテストで変更する変数は**1つだけ**
- テスト期間: **最低2週間**
- 統計的有意性が出るまで途中で判断しない
- テスト対象例: フック、CTA文言、BGM有無、動画の長さ、サムネイル

## 14. よくある失敗パターンと回避策（動画固有）

### 14-1. クリエイティブの失敗

| 失敗パターン | 問題 | 回避策 |
|------------|------|-------|
| TV CMをそのまま流用 | 冒頭にフックがなく5秒でスキップされる | 冒頭5秒にフックを追加編集。ABCDフレームワークに沿って再構成 |
| 横型動画のみ | Shorts・モバイルフルスクリーンで上下に黒帯。エンゲージメント低下 | 縦型（9:16）を別途制作してセットで入稿 |
| メッセージを詰め込む | 情報過多で何も記憶に残らない | 1動画1メッセージ。特にバンパーは厳守 |
| CTA不在 | 視聴後のアクションが不明 | 動画終盤 + テキストオーバーレイ + 音声でCTAを明示 |
| サムネイル未最適化 | インフィード広告のCTRが低い | カスタムサムネイルを作成。人物 + テキストの組み合わせが有効 |
| 字幕なし | ミュート環境で内容が伝わらない | 全動画に字幕（キャプション）を追加 |

### 14-2. キャンペーン設計の失敗

| 失敗パターン | 問題 | 回避策 |
|------------|------|-------|
| Demand Genにコンテンツターゲティング追加 | 配信量が大幅に制限される | CV目的ではオーディエンスターゲティングのみ使用 |
| 予算を急激に増額 | 学習リセットでパフォーマンス悪化 | 週20%以内の段階的増額 |
| 学習期間中に変更 | 最適化が進まない | 最低7日間は変更を控える |
| CV目的にVideo Viewsを使用 | CV最適化がかからない | CV目的にはDemand Genを使用する |
| 全フォーマットを1キャンペーンに混在 | 目的別の最適化ができない | 認知・検討・CVで別キャンペーンに分離 |

### 14-3. ターゲティングの失敗

| 失敗パターン | 問題 | 回避策 |
|------------|------|-------|
| ターゲットが狭すぎる | 配信量不足で学習が進まない | 認知キャンペーンは広め、CV目的はOptimized Targeting ON |
| フリクエンシーキャップ未設定 | 同一ユーザーに過剰に配信され、広告疲れ | 目的に応じたキャップを設定（認知: 週3〜5回） |
| プレースメントレポート未確認 | 低品質な配信先で予算を浪費 | 週次でプレースメントレポートを確認し、不適切な配信先を除外 |
| リマーケティングリスト未構築 | 動画視聴者を再アプローチできない | 動画視聴者リストを作成し、リマーケティングに活用 |

### 14-4. 計測の失敗

| 失敗パターン | 問題 | 回避策 |
|------------|------|-------|
| ビュースルーCVを無視 | 動画広告の間接効果を過小評価 | レポートにビュースルーCV列を追加 |
| ラストクリックのみで評価 | 動画は検索やリマーケの手前で貢献するため、過小評価される | データドリブンアトリビューション（DDA）を採用 |
| Brand Lift未実施 | 認知目的の効果を測定できない | 予算が十分な場合はBrand Lift調査を設定 |
