# Google検索広告（リスティング広告）

---

## 用語定義

| 略語 | 正式名称 | 定義 |
|------|---------|------|
| KW | キーワード | 広告表示のトリガーとなる語句 |
| CV | コンバージョン | 広告の目標アクション（購入、問い合わせ等）の完了 |
| CV値 | コンバージョン値 | 1件のCVに紐づく金額（売上、リードスコア等） |
| CVR | コンバージョン率 | クリック数に対するCV数の割合（CV数 ÷ クリック数） |
| CPA | コンバージョン単価 | 1件のCV獲得にかかった広告費（費用 ÷ CV数） |
| ROAS | 広告費用対効果 | 広告費に対する売上の倍率（CV値合計 ÷ 費用） |
| CTR | クリック率 | 表示回数に対するクリック数の割合（クリック数 ÷ 表示回数） |
| CPC | クリック単価 | 1クリックあたりの費用 |
| IS | インプレッションシェア | 表示機会に対する実際の表示割合 |
| RSA | レスポンシブ検索広告 | 複数の見出し・説明文からAIが最適な組み合わせを選ぶ広告 |
| DSA | 動的検索広告 | サイトコンテンツに基づいてGoogleが自動で広告を生成する仕組み |
| LP | ランディングページ | 広告クリック後にユーザーが最初に到達するページ |
| DDA | データドリブンアトリビューション | 機械学習でCV貢献度を各接点に配分するモデル |
| LTV | 顧客生涯価値 | 1顧客が生涯にもたらす売上・利益の合計 |

## CV件数別 意思決定早見表

| 月間CV件数 | 入札戦略 | マッチタイプ | CV設計 |
|-----------|---------|-------------|--------|
| **0〜15件** | 手動CPC or クリック数最大化 | 完全一致・フレーズ一致 | マイクロCV（フォーム開始等）をプライマリに追加して学習データを補完 |
| **15〜30件** | コンバージョン数最大化（目標CPA未設定） | 完全一致・フレーズ一致 | プライマリCVのみで運用可能。マイクロCVはセカンダリに |
| **30〜50件** | 目標CPA（実績ベースで設定） | フレーズ一致中心、部分一致テスト開始 | プライマリCVのみ。オフラインCV導入検討（B2B） |
| **50〜100件** | 目標CPA or 目標ROAS | 部分一致 + スマート入札が主力 | CV値ルールの活用開始。DDAの利用条件確認 |
| **100件以上** | 目標ROAS / ポートフォリオ入札 | 部分一致中心 | 目標ROAS本格運用。CV値の精緻化 |

## 1. アカウント・キャンペーン構造の設計

### キャンペーンの分け方

基本原則: **目的・予算・ターゲットが異なるものは別キャンペーンにする。**

| 分割軸 | 説明 | 理由 |
|--------|------|------|
| ブランドKW vs 一般KW | 自社名・商品名と一般KWを完全に分離 | ブランドKWはCVRが高く、混在すると一般KWの実力値が見えなくなる |
| 商材・サービス別 | 異なるサービスカテゴリごとにキャンペーン作成 | 予算配分の最適化、入札戦略の個別調整が可能 |
| 競合KW | 競合社名・競合商品名は独立キャンペーン | リスク管理と効果測定を分けるため |
| 地域別 | ターゲットエリアごとに分ける（必要な場合のみ） | 地域別の予算管理、入札調整が可能 |

> **注意**: 非ブランドキャンペーンには、ブランドKWを除外キーワードとして必ず登録する。

### 広告グループの設計: STAG（テーマ別広告グループ）

2025-2026年の推奨は **STAG（Single Theme Ad Group）**。

旧来のSKAG（1広告グループ1キーワード）は以下の理由で推奨されなくなっている。

- Googleの「類似バリエーション」拡張により、完全一致でも厳密な一致ではなくなった
- データが分散しすぎて、スマート入札の学習に必要なCVデータが不足する
- 管理の複雑さに対してリターンが見合わない

**STAGの設計原則:**

| 項目 | 推奨 |
|------|------|
| KW数 | 1広告グループあたり3〜20個（共通テーマ・共通意図でグルーピング） |
| 広告数 | 1広告グループあたり3〜5個のRSA |
| グループ数 | 1キャンペーンあたり7〜10広告グループ |
| 関連性 | 広告文とLPが各グループのテーマに高い関連性を持つこと |

**ハイブリッドアプローチ**: 高価値・ニッチなKWにはSKAGを維持しつつ、それ以外はSTAGで運用する方法も有効。

### 日本で知られるアカウント設計手法

#### Hagakure（葉隠）構造

アカウント構造をシンプルにして機械学習を機能しやすくする基本思想。広告グループを細分化しすぎず、1つの広告グループにデータを集約する。これにより表示・クリック・CVデータが蓄積し、GoogleのAIが最適化しやすくなる。

#### GORIN（五輪）

Hagakureの上に効率的な情報配信を実現する考え方。適切なユーザー・適切なタイミング・適切な情報の配信を重視。

#### MUGEN（無限）

HagakureとGORINを前提に、機械学習による最適化を維持しながら新規ユーザーへのリーチを拡大する運用指針。

| 柱 | 内容 |
|----|------|
| スマート自動入札 | KPI最適化を自動化 |
| 動的検索広告（DSA） | 手動設定していないKWもカバーし、リーチを拡大 |
| RSA | 広告品質を向上 |

> **注意**: HagakureやGORINの基盤が不十分な状態でMUGENを導入すると、費用増加・CPA高騰のリスクがある。

### 2025-2026年のGoogle推奨構造

1. **構造はシンプルに、データは集約** — AIは超細分化された構造を必要としない。必要なのは十分なCVデータ
2. **部分一致 + スマート入札** が月間50件以上のCVがあるアカウントの標準アプローチ
3. **AI Max for Search** — 見出し・説明文の自動生成、KWリスト外の検索語句マッチングを自動化する新機能
4. **矛盾する目標を同一キャンペーンに混在させない** — シグナルの明確さを確保する

## 2. キーワード戦略

### キーワードリサーチのツール

| ツール | 特徴 | 費用 |
|--------|------|------|
| Google キーワードプランナー | 公式。月間検索数、競合性、入札単価の参考値 | 無料 |
| ラッコキーワード | サジェストKWの網羅的取得。日本市場向け | 無料/有料 |
| SEMrush | 包括的なKWリサーチ、競合分析 | 有料（中価格帯） |
| Ahrefs | バックリンク分析とKWリサーチ | 有料（高価格帯） |
| SpyFu | PPC競合分析特化。競合の出稿KW・広告コピー・予算を調査 | 有料（低価格帯） |
| Google Search Console | 実際にサイトに流入しているクエリデータ | 無料 |
| Google検索のオートコンプリート | リアルタイムの検索トレンドを把握 | 無料 |

### リサーチのプロセス

```
1. ビジネスの主要テーマ・サービスをリストアップ
2. キーワードプランナーで種KWから候補を展開
3. 競合ツール（SpyFu/SEMrush）で競合が入札しているKWを調査
4. サジェストツール（ラッコキーワード等）でロングテールを発掘
5. 検索意図別にグルーピング（情報検索 / 比較検討 / 購入意思）
6. 検索ボリューム・競合性・推定CPCでフィルタリング・優先順位付け
```

### マッチタイプの使い分け戦略

#### 段階的アプローチ

```
Phase 1（データ不足時）: 完全一致で開始し、データを蓄積
    ↓
Phase 2（ボリュームが必要な場合）: フレーズ一致を追加
    ↓
Phase 3（月50件以上のCV）: 部分一致 + スマート入札に拡大
    ↓
全段階を通じて: 除外KWリストを積極的に構築
```

#### 2025-2026年のトレンド

- スマート入札利用広告主の多くが部分一致をメインのマッチタイプとして使用する傾向（出典: Google公式ブログ、Search Scientists 2025年調査）
- 完全一致でも「厳密な一致」ではなく、全マッチタイプでGoogleのAIによる意味解釈が介入
- 少数のコアKWを適切なオーディエンスターゲティングと組み合わせるホリスティックなアプローチが主流

#### 部分一致 + スマート入札の成功条件

- 正確なCVトラッキングが設定されていること
- 月間30〜50件以上のCVがあること
- 包括的な除外KWリストの併用
- 定期的な検索語句レポートの確認

### キーワードの整理・グルーピング

| 分類軸 | 例 |
|--------|-----|
| **検索意図別** | 情報収集（「とは」「方法」）/ 比較検討（「比較」「おすすめ」）/ 購入意思（「価格」「申し込み」） |
| **テーマ別** | 同じ製品・サービス・問題に関するKWを1グループに |
| **ファネル段階別** | 認知（TOFU）/ 検討（MOFU）/ 決定（BOFU） |
| **CV価値別** | 高LTV顧客を獲得しやすいKWを優先 |

### ロングテールKWの活用

ロングテールKW（4語以上の具体的なフレーズ）はCPCが低く、CVRが高い傾向がある。

| ショートテール | ロングテール |
|---------------|-------------|
| 「CRMソフト」 | 「中小企業向け BtoB CRMソフト 比較」 |
| 「ランニングシューズ」 | 「マラソン練習用 ランニングシューズ おすすめ」 |

- 各広告グループを1つの明確なテーマに集中させる
- 広告文とLPをロングテールKWに最適化する
- 部分一致 + スマート入札でロングテールを自動発見させることも有効

### ブランドKWの扱い方

**基本方針: 必ず独立キャンペーンで運用する。**

| 理由 | 内容 |
|------|------|
| データの明確化 | ブランドKWはCVRが高いため、非ブランドの実力値が見えなくなる |
| 予算管理 | ブランドと非ブランドの予算を独立管理 |
| 入札戦略 | ブランドKWは低CPCで済むため、別の入札戦略が最適 |
| 競合防衛 | 競合がブランド名に入札している場合は防衛の優先度が上がる |

### 除外キーワードの運用

#### 確認頻度

| アカウント規模 | 推奨頻度 |
|----------------|----------|
| 高予算アカウント | 毎日 |
| 中規模アカウント | 週次 |
| 全アカウント | 月次の定期レビュー |

#### 設定レベル

| レベル | 用途 | 例 |
|--------|------|-----|
| アカウントレベル | 絶対に関連しないワード | 「無料」「求人」「方法」 |
| キャンペーンレベル | キャンペーン固有のノイズ除去 | 他キャンペーンのテーマ |
| 広告グループレベル | カニバリゼーション防止 | 広告グループ間の重複排除 |

#### マッチタイプ別の使い分け

- **部分一致の除外**: 明らかに無関係な用語（「求人」「給与」「無料」等）
- **フレーズ一致の除外**: 他の文脈では関連する可能性がある用語
- **完全一致の除外**: 非常に具体的な除外

> 除外KWは通常KWと異なり、類義語への自動拡張が行われない。ただし2024年6月以降、スペルミスはブロック対象に含まれるようになった。

#### 上限（2025年時点）

- 共有除外KWリスト: 最大20リスト、各5,000KWまで
- アカウントレベルの除外: 1,000KWまで

> 汎用的な除外KWの初期セットは [付録A](#付録a-除外キーワード初期セット) を参照。

## 3. 入札戦略の詳細運用

> **注意**: 拡張CPC（eCPC）は2025年3月末に廃止済み。検索・ディスプレイ・ホテルキャンペーンが対象。既存のeCPCキャンペーンは手動CPCに自動移行された。

### 目標CPA vs 目標ROAS

| 項目 | 目標CPA | 目標ROAS |
|------|---------|----------|
| 使う場面 | 全CVの価値がほぼ同じ（リード獲得、SaaS等） | CV値にばらつきがある（EC等） |
| 最適化目標 | ボリューム（適正コスト内） | 売上・収益 |
| 最低データ | 30日間で15件以上（推奨30件以上） | 30日間で50件以上（推奨100件以上） |

### 目標値の設定と調整

**初期設定:**
- 過去30日間の**実績値**をベースにする（理想値ではない）
- 目標CPAの場合: 過去30日の平均CPAとほぼ同じ値で開始
- 目標ROASの場合: 過去30日の「CV値/費用」に近い値で開始
- 目標値を高く設定しすぎると、ボリュームが極端に制限される

**調整のコツ:**

| ルール | 内容 |
|--------|------|
| 20%ルール | 一度に20%以上変更しない（急激な変更は学習をリセット） |
| 段階的引き締め | 目標CPA: 5〜10%ずつ引き下げ / 目標ROAS: 5〜10%ずつ引き上げ |
| 判断の時間軸 | 週単位ではなく2〜4週間単位で判断 |
| CVの遅延 | コンバージョンサイクルが長い業種は遅延を考慮 |

### 学習期間の管理

- **期間**: 通常7〜30日（大半は約2週間）（出典: [Google Ads ヘルプ - 学習期間](https://support.google.com/google-ads/answer/13020501)）
- **必要データ**: 約50件のCVイベント、または3コンバージョンサイクル

**学習期間がリセットされるトリガー:**
- 入札戦略の切り替え
- 目標CPA/ROASの大幅変更（20%以上の変更）
- 日予算の大幅な増減
- CVアクションの変更
- ターゲティング・クリエイティブ・オーディエンスの大幅変更

**管理のベストプラクティス:**
- 予算・入札・設定の変更は1週間あたり20%以内
- 最低2〜3週間はパフォーマンス判断を保留
- 移行後のキャンペーン変更を最小限に

### ポートフォリオ入札戦略

複数キャンペーン・広告グループ・KWを横断して1つの入札戦略を適用する仕組み。

- パフォーマンスが高いキャンペーンに自動的にリソース集中
- 類似目標のキャンペーン間でデータを共有し、予測精度を向上
- **同じ目標を持つキャンペーンをグループ化**すること（矛盾する目標を混在させない）
- データが少ない個別キャンペーンを集約して学習データ量を補う効果もある

### 季節変動への対応

Google Adsの**季節性調整（Seasonality Adjustments）**機能を活用する。

- プロモーション・セール等の予定イベントに対して、CVRの予想変動をスマート入札に事前通知
- **最適な期間: 1〜7日間**の短期イベント向け（14日以上は非推奨）
- 終了後に逆調整は不要（自動で元に戻る）
- 出典: [Google Ads ヘルプ - 季節性調整](https://support.google.com/google-ads/answer/10369906)

**一般的な対応:**

| 時期 | 対応 |
|------|------|
| 繁忙期前 | 予算を段階的に増加（一度に20%以内） |
| 繁忙期中 | 季節性調整を活用 |
| 閑散期 | 予算を削減し、高意図KWに集中 |
| 年間 | 過去年の実績データを参考に事前計画 |

## 4. レスポンシブ検索広告（RSA）の運用詳細

### 説明文の書き方

| ポイント | 内容 |
|---------|------|
| 構造 | 1文目で価値提案、2文目で裏付け or CTA |
| CTA | 明確な行動喚起を含める（「今すぐ相談」「無料で試す」） |
| 差別化 | 競合にはない強みを具体的に |
| 数字 | 可能な限り具体的な数値を含める |

### ピン留め機能

- Googleの推奨は「最小限のピン留め」
- やむを得ない場合は**同一ポジションに2〜3個のピン留め**で多様性を維持
- ピン留めはAd Strengthを下げる傾向があるが、Ad Strengthとパフォーマンスは必ずしも相関しない

### Ad Strength（広告の有効性）

- 「低い」→「非常に高い」への改善で平均12%のCV増加（出典: [Google Ads ヘルプ - RSA](https://support.google.com/google-ads/answer/7684791)。全業種・全地域の検索キャンペーン平均。個別の効果は業界・アカウント状況により異なる）
- ただし最終判断はCPA/ROAS等の実績指標で行う
- 「良い」以上を目指す（「優良」は必須ではない）

### A/Bテスト

- **広告バリエーション機能**が最も手軽
- 1つの変数のみ変更する
- 最低2〜4週間、統計的有意性を確保
- 評価指標は**CPI（CTR x CVR）**が最重要

> テスト設計の詳細テンプレートは [付録B](#付録b-abテスト設計テンプレート) を参照。

### 広告カスタマイザー

| 機能 | 内容 |
|------|------|
| DKI（動的キーワード挿入） | 検索語句に応じて見出し・説明文内のテキストを自動差し替え |
| カウントダウン | セール終了までの残り時間を自動表示 |
| 地域挿入 | ユーザーの所在地を自動挿入 |
| IF関数 | デバイス・オーディエンスに応じて表示を出し分け |

### 動的検索広告（DSA）との併用

- RSAで主力KWをカバーし、DSAでKWギャップを発見する戦略が有効
- 同一キャンペーン内で共存可能（ただし同一広告グループは不可）
- DSAで見つかった高パフォーマンスKWをRSAの広告グループに移行

## 5. 広告コピーのフレームワーク

### 主要フレームワーク

**AIDA（注意→興味→欲求→行動）**
- 見出し1: 注意（Attention）を引く
- 見出し2: 興味（Interest）を持たせる
- 説明文: 欲求（Desire）を喚起し、行動（Action）を促す

**PAS（問題→煽り→解決）**
- 見出し1: 問題（Problem）を提示
- 見出し2: 煽り（Agitation）で痛みを強調
- 説明文: 解決策（Solution）を提示

**4P（約束→描写→証拠→後押し）**
- Promise: 核となるベネフィットを約束
- Picture: 具体的な未来像を描く
- Proof: 実績・データで証明
- Push: 行動を後押し

### 業界別の効果的な訴求パターン

| 業界 | 効果的な訴求 |
|------|-------------|
| BtoB | コスト削減率、ROI、導入実績社数、無料トライアル |
| BtoC | 価格、口コミ評価、手軽さ、送料無料 |
| EC | 送料無料、即日配送、品揃え数、返品保証 |
| 士業・専門職 | 専門性（資格・経験年数）、相談実績、初回無料相談 |

## 7. ランディングページの最適化

### 広告とLPの一貫性

品質スコアの3要素（推定CTR、広告の関連性、LP利便性）が「平均以上」の広告は、「平均以下」の広告と比較してCPCが大幅に低下し、CVRが大幅に向上する傾向がある。具体的な改善幅は業界・アカウント・KWの競合状況によって異なるが、品質スコア改善の費用対効果は非常に高い。

### LP設計の基本原則

| 原則 | 内容 |
|------|------|
| 広告との一致 | 広告で訴求した内容がLPのファーストビューに存在すること |
| 1ページ1目的 | LPごとにCVゴールを1つに絞る |
| 明確なCTA | ファーストビューにCTAボタン、スクロール後もフローティングCTA |
| 信頼性 | 会社情報、プライバシーポリシー、お客様の声、実績数字 |
| KW対応 | 広告グループごとに対応するLPを用意（汎用トップページではなく専用LP） |

### ページ速度

- ページ速度の遅延はモバイルCVRに大きく影響する（Googleの調査で、表示速度が1秒→5秒になると直帰率が90%増加と報告）
- 目標: LCP 2.5秒以内、全体読み込み2秒以内
- 改善方法: 画像圧縮、不要なJS削減、CDN活用、遅延読み込み

### モバイル最適化

- レスポンシブデザイン
- タップしやすいボタンサイズ（最低48x48px）
- 電話番号のタップ発信対応
- フォームの入力しやすさ（オートコンプリート対応）

### フォーム最適化

- フォームのフィールド数を減らすとCVRが向上する傾向（HubSpotの調査でフィールド3個以下のフォームが最も高いCVR）
- マルチステップフォームが効果的（個人情報は最後のステップに配置）
- 進捗バーの表示
- エラーメッセージはリアルタイムで表示

### 品質スコアとLPの関係

2025年以降、GoogleがLPのユーザー体験と透明性により大きな重みを置く新しい予測モデルを導入。以下が評価に影響する。

- コンテンツの関連性（広告KWとLPの内容の一致）
- ナビゲーションのしやすさ
- ページ速度（特にモバイル）
- 透明性（会社情報、問い合わせ先、プライバシーポリシー）

## 8. コンバージョン計測の運用詳細

### B2B / EC別 CV設計テンプレート

#### B2B（リード獲得型）

| 項目 | 設定 |
|------|------|
| プライマリCV | 問い合わせフォーム送信、電話発信 |
| セカンダリCV | 資料DL、メルマガ登録、料金ページ閲覧 |
| CV値の決め方 | リードスコア × 商談化率 × 平均受注単価（例: スコアA=5万円、B=2万円、C=5千円） |
| 入札戦略 | 目標CPA（月30CV以上で導入） |
| オフラインCV | **必須**。CRMから商談化・成約データをフィードバックし、「質の高いリード」を学習させる |
| マイクロCV活用 | 月間問い合わせ30件未満の場合、「フォーム開始」をプライマリに追加 |

#### EC（購買型）

| 項目 | 設定 |
|------|------|
| プライマリCV | 購入完了 |
| セカンダリCV | カート追加、チェックアウト開始、会員登録 |
| CV値の決め方 | 実際の注文金額（動的CV値）。定額商品の場合は平均注文単価 |
| 入札戦略 | 目標ROAS（月50CV以上で導入） |
| CV値ルール活用 | 「リピート顧客からのCVはCV値を1.5倍」等、LTVを反映した調整 |
| マイクロCV活用 | 月間購入30件未満の場合、「カート追加」をプライマリに追加 |

### CVアクション変更時の手順

CVアクション（プライマリCVの追加・削除・変更）はスマート入札の学習をリセットするため、慎重に行う。

```
1. 変更前のパフォーマンスを記録（CPA、ROAS、CV数の直近30日平均）
2. 変更を実施
3. 学習期間（2〜4週間）は大幅な入札・予算変更を避ける
4. 変更前後の比較は「変更日から学習完了まで」を除外して行う
5. 学習完了後、必要に応じて目標CPA/ROASを再設定
```

### コンバージョン値ルール

オーディエンス・地域・デバイス条件でCV値を自動調整する機能。

例:
- 「東京からのリードはCV値を1.5倍」（商圏内のリードは成約率が高い）
- 「モバイルからのCVはCV値を0.8倍」（モバイル経由は成約率が低い）
- 「リマーケティングリストのユーザーからのCVはCV値を1.3倍」

### コンバージョンウィンドウ

| 設定 | 推奨値 | 補足 |
|------|--------|------|
| クリックスルーCV | 30〜90日 | BtoB（検討期間長い）は90日、EC（即決型）は30日 |
| ビュースルーCV | 1日（デフォルト） | ディスプレイ/動画向け。検索のみなら影響小 |
| エンゲージビューCV | 10秒視聴後のCV | YouTube動画広告向け |

## 9. 検索キャンペーンの設定チェック

実務で事故りやすい、成果に直結する設定項目を整理する。

### 地域設定の落とし穴

Google Adsの地域ターゲティングには2つのオプションがある。

| オプション | 内容 | 推奨 |
|-----------|------|------|
| **所在地（Presence）** | ターゲット地域に物理的にいるユーザーのみに配信 | **ほとんどのケースで推奨** |
| **所在地または関心（Presence or Interest）** | ターゲット地域にいるユーザー + その地域に関心を示しているユーザーに配信 | デフォルト設定。旅行・観光・不動産など地域関心が購買意図に直結する場合のみ |

> **重要**: デフォルトは「所在地または関心」になっている。この設定のままだと、ターゲット地域外のユーザーにも配信されて予算が漏れる。**ほとんどのローカルビジネスは「所在地」に変更すべき。**

**設定場所**: キャンペーン設定 → 地域 → 地域のオプション → ターゲット

**除外地域の設定**:
- 除外も同様に「所在地」で設定する
- 半径ターゲティングの場合、半径外のインプレッションが発生していないか定期確認

### 検索パートナーの扱い

「Google検索パートナー」にチェックが入っていると、Google以外のサイト（Google以外の検索エンジン、Googleマップ等）にも広告が配信される。

| 状況 | 推奨 |
|------|------|
| 初期段階 / 予算が限られている | **OFF推奨**。まずはGoogle検索のみでデータを蓄積 |
| CVデータが十分にあり、リーチを拡大したい | ONにしてテスト。パフォーマンスをセグメント別に確認 |
| 検索パートナーのCPAが許容範囲外 | OFFに戻す |

**確認方法**: レポートのセグメント → ネットワーク で「Google検索パートナー」のパフォーマンスを個別確認。

### スマート入札下でのデバイス・時間帯調整

スマート入札（目標CPA、目標ROAS等）を使用している場合、**手動のデバイス入札調整・時間帯入札調整は実質的に無視される**（GoogleのAIがリアルタイムで独自に調整するため）。

| 入札戦略 | デバイス調整 | 時間帯調整 |
|---------|-------------|-----------|
| 手動CPC | **有効** | **有効** |
| クリック数最大化 | **有効** | **有効** |
| 目標CPA / 目標ROAS | **無視される**（-100%のみ例外: デバイス完全除外） | **無視される** |
| コンバージョン数/値最大化 | **無視される**（-100%のみ例外） | **無視される** |

**対処法**:
- スマート入札下でデバイスを完全に除外したい場合は -100% を設定（例: タブレットを除外）
- 時間帯による配信制御が必要なら、広告スケジュールで「配信しない時間帯」を設定する（入札調整ではなく配信ON/OFFで制御）
- デバイス別にLP最適化を行い、品質スコア経由で間接的にパフォーマンスを改善する

### 検索語句の品質評価（特にB2B）

除外KWの追加だけでなく、獲得したリードの**質**を評価する仕組みが重要。

**評価軸:**

| 軸 | 確認方法 |
|----|---------|
| 意図の質 | 検索語句が情報収集か購入意思か |
| 商談品質 | CRMデータとの突合で、検索語句別の商談化率を確認 |
| 成約率 | オフラインCVデータで検索語句別の成約率を確認 |
| LTV | 検索語句別の顧客LTVを追跡（長期的な取り組み） |

**実装方法:**
1. Google AdsのGCLID（クリックID）をフォームの隠しフィールドに記録
2. CRMで商談化・成約時にGCLIDと紐づけて記録
3. Google Adsにオフラインコンバージョンとしてアップロード
4. 検索語句レポートで「質の高いリードを生むKW」を特定

### ブランド防衛の基準

競合がブランド名に入札している場合の対応方針。

**防衛すべきか判断する基準:**

| 条件 | 判断 |
|------|------|
| 競合が自社ブランド名で出稿している | 防衛キャンペーンを作成 |
| オーガニック1位を独占しており、競合出稿もない | 防衛不要（予算を非ブランドに回す） |
| ブランド名の検索ボリュームが大きい | 防衛の費用対効果が高い |

**防衛キャンペーンの設定:**

| 項目 | 推奨 |
|------|------|
| 入札戦略 | 目標インプレッションシェア（最上部90〜95%） |
| マッチタイプ | 完全一致 + フレーズ一致 |
| 日予算 | 通常は少額で済む（ブランドKWはCPCが低い） |
| 監視 | オークション分析で競合の参入・撤退を定期確認 |

### ディスプレイネットワーク拡張の確認

検索キャンペーン作成時に「ディスプレイネットワークを含める」オプションが存在する場合がある。

> **推奨: 検索キャンペーンではOFFにする。** ディスプレイ配信が必要なら専用のディスプレイキャンペーンを作成する。検索とディスプレイが混在するとデータが汚染され、スマート入札の学習に悪影響を与える。

## 11. 分析と改善

### KPIベンチマーク参考値

以下は業界横断の平均値であり、**自社の過去データとの比較を最優先**すること。業界・国・規模・商材によって大きく異なるため、あくまで目安として使用する。

| KPI | 参考値 | 条件・出典 |
|-----|--------|-----------|
| CTR | 約6〜7% | Google検索広告、全業種平均、米国中心のデータ。出典: WordStream 2025 Benchmarks |
| CVR | 約7〜8% | 同上。業界差が大きい（法律系15%超 vs アパレル2%前後） |
| CPA | $20〜30（中央値） | 同上。2025年は前年比で上昇傾向 |
| ROAS | 3〜4倍 | EC中心のデータ。リード獲得型には直接適用しにくい |

> **注意**: これらのベンチマークは主に米国市場のデータに基づく。日本市場では業界別のCPCやCVRが異なるため、Google キーワードプランナーの推定値や自社の過去実績を基準にすること。

### 検索語句レポートの活用

| ステップ | 内容 |
|---------|------|
| 1. 除外KW特定 | 無関係なクエリを除外KWに追加 |
| 2. 新規KW発見 | 高パフォーマンスのクエリをKWとして追加 |
| 3. マッチタイプ最適化 | 意図しない拡張が起きていないか確認 |
| 4. 広告文/LP改善 | 検索語句からユーザーの意図を読み取り、広告文やLPを改善 |

> AI Max有効時は、検索語句レポートに「AI Max」マッチタイプが表示される。AIが拡張したクエリの品質を確認し、不要なものは除外する。

### オークション分析（Auction Insights）の活用

| 指標 | 見方 |
|------|------|
| インプレッションシェア（IS） | 自社の表示機会に対する実際の表示割合 |
| 重複率 | 同じオークションに参加している競合の割合 |
| 上位掲載率 | 自社が競合より上位に表示された割合 |
| ページ上部表示率 | 検索結果のページ上部に表示された割合 |
| ページ最上部表示率 | 検索結果の最上部に表示された割合 |
| 優位表示シェア | 競合の広告が自社より上位に表示された割合 |

**活用方法:**
- 新しい競合の参入を検知
- 特定の競合との入札競争の状況を把握
- ISの低下原因を特定（IS損失率(予算) → 予算不足、IS損失率(ランク) → 品質スコア/入札額不足）

### セグメント分析

| セグメント | 分析ポイント |
|-----------|-------------|
| デバイス | モバイル/PC/タブレットのCVR差 → 入札調整（手動CPC時）またはLP改善（スマート入札時） |
| 時間帯 | CVが集中する時間帯 → スケジュール設定（手動CPC時）または参考情報（スマート入札時） |
| 曜日 | 曜日別のパフォーマンス差 → 同上 |
| 地域 | 都道府県別のCVR → 地域入札調整 or 地域別キャンペーン分離 |
| オーディエンス | リマーケティングリストのCVR → モニタリングモードで追加し、データに基づいて入札調整 |

### アトリビューション

2025年以降、Google Adsで選択できるアトリビューションモデルは**DDA（データドリブン）とラストクリックの2モデルのみ**。線形、減衰、ポジションベースモデルは廃止済み。

| モデル | 利用条件 | 特徴 |
|--------|---------|------|
| **DDA（推奨）** | 過去30日間で3,000インタラクション + 300CV | 機械学習でCV貢献度を各接点に配分。ファネル上部の接触も評価される |
| ラストクリック | 条件なし | 最後にクリックされた広告に100%のCVクレジットを付与 |

- 条件を満たさない場合はラストクリックを使用
- 出典: [Google Ads ヘルプ - データドリブンアトリビューション](https://support.google.com/google-ads/answer/6394265)

## 12. 予算管理と最適化

### 予算設定のポイント

| 項目 | 推奨 |
|------|------|
| 日予算 | 目標CPAの10〜15倍以上 |
| 月予算 | Google Adsは日予算の2倍まで使用可能（月間で平均化） |
| 予算不足の兆候 | IS損失率（予算）が20%以上なら増額を検討 |

### 損益分岐点の計算

```
【基本】
限界CPA = 売価 - 原価
目標CPA = 限界CPA - 確保したい利益

【LTV考慮版（サブスク・リピート商材向け）】
限界CPA = 顧客LTV × 粗利率
目標CPA = 限界CPA × (1 - 利益マージン)

【B2Bリード獲得型】
限界CPA = 平均受注単価 × 粗利率 × 商談化率 × 成約率
目標CPA = 限界CPA × (1 - 利益マージン)
```

### 予算制限への対処

| 対処法 | 内容 |
|--------|------|
| KW絞り込み | 低パフォーマンスKWを停止し、高パフォーマンスKWに集中 |
| 時間帯調整 | CVが少ない時間帯の配信をOFF（手動CPC時は入札引き下げ） |
| 地域調整 | パフォーマンスの低い地域の入札を引き下げ |
| デバイス調整 | CVRの低いデバイスの入札を引き下げ（手動CPC時） |
| 予算増額 | ROASが十分であれば予算増額を検討 |

### クリエイティブリフレッシュ頻度

| 素材タイプ | 推奨頻度 |
|-----------|----------|
| RSA広告文 | パフォーマンスが低下したら随時 |
| 画像アセット | 4〜6週間ごと |
| サイトリンク等 | 月次で確認、低パフォーマンスを入れ替え |

### 2026年新機能: キャンペーン合計予算

特定期間の合計予算を設定し、GoogleのAIが日別配分を自動最適化する機能。検索・P-MAX・ショッピングで利用可能。繁忙期や特定プロモーション期間での予算管理に有効。

## 13. よくある失敗パターンと回避策

### 初心者の失敗トップ10

| # | 失敗 | 回避策 |
|---|------|--------|
| 1 | CVトラッキング未設定 or 不正確 | 運用開始前にGTMプレビューで全タグの発火確認。拡張CVも設定 |
| 2 | 除外KW未設定 | 初日から汎用除外リストを適用（[付録A](#付録a-除外キーワード初期セット)参照）。週次で検索語句レポート確認 |
| 3 | 全部分一致でスタート | 初期は完全一致・フレーズ一致から開始し、段階的に拡張 |
| 4 | 1広告グループにKW詰め込みすぎ | テーマを統一した3〜20KWのSTAGで設計 |
| 5 | 広告が1パターンのみ | 1広告グループに3〜5個のRSA、異なるメッセージ |
| 6 | LPがトップページ | 広告グループのテーマに対応した専用LPを用意 |
| 7 | Googleの自動推奨を鵜呑み | 推奨は参考にしつつ、自社のデータと目標に基づいて判断 |
| 8 | CVデータ不足でスマート入札 | 月30CV以上蓄積してから移行。不足時はマイクロCVで補完 |
| 9 | 月1回しか確認しない | 最低週次で検索語句・パフォーマンスを確認 |
| 10 | モバイル非対応のLP | レスポンシブデザイン、ページ速度、フォーム最適化を必ず実施 |

### スマート入札の6つの落とし穴

| # | 落とし穴 | 回避策 |
|---|---------|--------|
| 1 | CVデータ不足での導入（月30CV未満） | 手動入札 or クリック最大化でデータ蓄積を優先 |
| 2 | 非現実的な目標値の設定 | 過去の実績値ベースで開始し、段階的に引き締め |
| 3 | 学習期間中の頻繁な変更 | 2〜3週間は大幅な変更を避ける |
| 4 | 予算と目標CPAの不整合 | 日予算は目標CPAの10〜15倍以上を確保 |
| 5 | CVデータの汚染 | 正確なCVトラッキング。マイクロCVの混入に注意 |
| 6 | Googleの自動提案の鵜呑み | 推奨スコアの鵜呑みは危険。自社データで判断 |

## 14. AI Max for Search Campaigns

2025年5月に全広告主にベータ提供開始。既存検索キャンペーンにAI機能を追加するレイヤー。（出典: [Google Blog - AI Max for Search](https://blog.google/products/ads-commerce/google-ai-max-for-search-campaigns/)）

### 3つのコア機能

| 機能 | 内容 |
|------|------|
| KWマッチング拡張 | ブロードマッチ + キーワードレス技術で検索意図に基づくマッチング |
| 広告文の自動生成 | 見出し・説明文をAIが生成し、既存アセットと組み合わせ |
| LP自動選択 | 検索意図に最も適したLPをサイト内から自動選択 |

### パフォーマンス実績

- 一般アカウントでCV/CV値 **14%向上**（出典: Google公式発表。全業種・全地域の平均値）
- 完全一致・フレーズ一致中心アカウントで **27%向上**（同上）
- 類似CPA/ROAS維持
- ただし独立テストでは一部の広告主のみが良好な結果を報告（効果は個別のアカウント状況により大きく異なる）

### テキストガイドライン（2026年Q1〜）

除外ワードと自然言語指示でAI生成コピーを制御可能。例: 「価格に言及しない」「競合名を使わない」等。

### 導入推奨フロー

```
1. まず1キャンペーンでA/Bテスト（4週間以上）
2. 結果を分析（CV数、CPA、検索語句の質）
3. 効果が確認できたら段階的に展開
4. ブランドKWキャンペーンはAI Max OFFを推奨
```

### 運用ガードレール

AI Maxは適切に制御しないとリスクがあるため、以下のガードレールを設定する。

#### AI Max OFF推奨のケース

| ケース | 理由 |
|--------|------|
| ブランドKWキャンペーン | AIが非ブランドクエリに拡張し、ブランドKWの低CPC・高CVRのメリットが薄れる |
| 法規制が厳しい業界（医療、金融、法律等） | AI生成の広告文が薬機法・景表法・金商法等に抵触するリスク |
| 競合名への出稿を厳密に避けたい場合 | AI Maxが競合関連のクエリに拡張する可能性 |
| CVデータが月30件未満のキャンペーン | AIの学習データが不十分で最適化が不安定 |

#### 広告文生成の制御

テキストガイドラインで以下を設定することを推奨。

| 設定 | 理由 |
|------|------|
| 価格に関する虚偽表示の禁止 | 実際と異なる価格をAIが生成するリスク |
| 競合名の使用禁止 | 商標侵害のリスク |
| 誇大表現の禁止 | 「業界最安」「日本一」等の根拠のない表現 |
| 薬機法・景表法に抵触する表現の禁止 | 「治る」「必ず効果がある」等（医療・健康商材） |
| ブランドトーンの指定 | 自社のブランドガイドラインに沿ったトーン |

#### LP自動選択の制御

AI MaxのLP自動選択機能が意図しないページに誘導しないよう、除外URLを設定する。

**除外すべきURL例:**
- サンキューページ（CV完了ページ）
- ログインページ
- 採用ページ
- 管理画面・マイページ
- 古い・廃止予定のLP
- プライバシーポリシー・利用規約ページ

### 検索語句レポート

AI Maxが有効な場合、「AI Max」という新しいマッチタイプが検索語句レポートに表示され、AIが拡張したクエリを確認・除外可能。**導入後の初期2週間は毎日検索語句レポートを確認**し、意図しない拡張がないかチェックすること。

## 付録B: A/Bテスト設計テンプレート

テストを実施する前に以下を明確にし、記録する。

### テスト設計シート

| 項目 | 記入 |
|------|------|
| **テスト名** | 例: 「見出しのベネフィット訴求 vs 価格訴求」 |
| **仮説** | 例: 「具体的な価格を見出しに入れると、ベネフィット訴求よりCVRが向上する」 |
| **変更する変数**（1つのみ） | 例: RSAの見出し2〜4個（価格訴求に変更） |
| **固定する変数** | KW、LP、入札戦略、予算、オーディエンス、その他の広告文 |
| **評価指標（主）** | 例: CPA |
| **評価指標（副）** | 例: CTR、CVR |
| **合格基準** | 例: CPAが10%以上改善、かつ統計的有意（p < 0.05） |
| **停止基準** | 例: CPAが30%以上悪化した場合は即停止 |
| **テスト期間** | 最低2〜4週間（またはCV 100件以上/バリエーション） |
| **テスト方法** | 広告バリエーション機能 or キャンペーンエクスペリメント |

### サンプル不足時の対処

| 状況 | 対処法 |
|------|--------|
| CV数が足りない | テスト期間を延長（4〜6週間） |
| それでも不足 | マイクロCV（フォーム開始等）を評価指標に使用 |
| KWの検索ボリュームが少ない | テスト対象の広告グループを統合して実施 |
| 統計的有意性が出ない | 変更のインパクトが小さい可能性。より大胆な変更でテスト |
