# Google Demand Gen

---

## 1. 広告フォーマット

### 1-1. シングル画像広告（マルチアセット広告）

複数の画像・見出し・説明文・CTAを登録し、Google AIが配信面・ユーザーごとに最適な組み合わせを選択して配信する。

- 横長・正方形・縦長の3アスペクト比をすべて登録すると配信面が最大化
- 各アセットは独立して意味が通る内容にする（AIが任意の組み合わせを生成するため）

### 1-2. 動画広告（動画レスポンシブ広告）

YouTubeにアップロード済みの動画を使用。横長・正方形・縦長を組み合わせると、In-Stream/In-Feed/Shortsの全面に配信可能。

- 最初の5秒でブランドと価値提案を伝える
- 無音でも伝わるよう字幕・テキストオーバーレイを活用
- 推奨: 16:9（横長）+ 1:1（正方形）+ 9:16（縦長）の3比率セット

### 1-3. カルーセル画像広告

2〜10枚のカード型画像をスワイプ形式で表示。各カードに個別の見出し・リンク先URLを設定可能。

- 1つのカルーセル内で全カードのアスペクト比は統一が必要
- 対応アスペクト比: 1.91:1（横長）/ 4:5（縦長）/ 1:1（正方形）
- ストーリー性のある展開（課題→解決→CTA）や複数商品の並列紹介に有効

### 1-4. 商品フィード付き広告（EC向け）

Merchant Centerの商品フィードと画像/動画を組み合わせて配信。3つのサブタイプがある:

| サブタイプ | 内容 |
|-----------|------|
| **画像+商品** | 画像クリエイティブ + 商品フィード |
| **動画+商品** | 動画クリエイティブ + 商品フィード |
| **商品のみ** | クリエイティブなし。商品フィードのみで配信 |

## 2. 入稿仕様

### テキストアセット

| 項目 | 仕様 | 数量 |
|------|------|------|
| **見出し** | 最大40文字（日本語実質20文字） | 最大5本 |
| **長い見出し** | 最大90文字（日本語実質45文字） | 最大5本 |
| **説明文** | 最大90文字（日本語実質45文字） | 最大5本 |
| **ビジネス名** | 最大25文字（日本語実質12〜13文字） | 1つ（必須） |
| **CTA** | 選択式 or カスタム（10文字以内） | 1つ |
| **最終URL** | 必須 | 1つ |

**日本語の文字数カウント:** 日本語・中国語・韓国語は1文字=2文字としてカウント。半角英数字は1文字=1文字。

### 画像アセット

| アスペクト比 | 最小サイズ | 推奨サイズ | ファイルサイズ |
|-------------|----------|----------|-------------|
| **横長 1.91:1（必須）** | 600×314px | 1200×628px | 5MB以下 |
| **正方形 1:1（必須）** | 300×300px | 1200×1200px | 5MB以下 |
| **縦長 4:5（任意）** | 480×600px | 960×1200px | 5MB以下 |
| **ロゴ 1:1** | 144×144px | 1200×1200px | 150KB以下 |

- ファイル形式: JPG / PNG / 静止GIF
- 推奨登録数: 15〜20枚（全アスペクト比合計）
- 縦長は任意だがYouTube Shorts/Discover面の拡大に推奨

### 動画アセット

| アスペクト比 | 推奨解像度 | 用途 |
|-------------|----------|------|
| **横長 16:9** | 1920×1080px | YouTube In-Stream、標準 |
| **正方形 1:1** | 1080×1080px | Discover、Gmail |
| **縦長 9:16** | 1080×1920px | YouTube Shorts |

| 項目 | 仕様 |
|------|------|
| ファイル形式 | MP4（H.264）or MPG（MPEG-2/4） |
| 最大ファイルサイズ | 256GB or 12時間（短い方） |
| 最小解像度 | 720p |
| 最小長さ | 5秒（YouTubeは6秒以上） |
| Shorts推奨 | 10〜20秒（最大1分） |
| 登録数 | 1〜5本 |

### カルーセル広告

| 項目 | 仕様 |
|------|------|
| カード枚数 | 2〜10枚 |
| カード見出し | 最大40文字 |
| カード説明文 | 最大40文字 |
| カード画像 | 全カード同一アスペクト比 |
| 対応アスペクト比 | 1.91:1 / 4:5 / 1:1 |
| CTA | 必須 |

### CTA選択肢

**カルーセル/画像広告:** 今すぐ申し込む / 予約する / お問い合わせ / ダウンロード / 詳細 / サイトを見る / 今すぐ購入 / 登録する / 見積もりを取得 / 登録 / もっと見る

**動画広告:** 上記に加えて、購入する / 寄付する / 今すぐ注文 / 今すぐプレイ / 今すぐ開始 / 視聴する

**推奨:** 「自動」設定。Google AIがユーザーごとに最適なCTAを選択する。

### 推奨アセット数のまとめ

| アセット | 最低限 | 推奨 | 上限 |
|---------|-------|------|------|
| 横長画像（1.91:1） | 1 | 5〜10 | 20（全画像合計） |
| 正方形画像（1:1） | 1 | 5〜10 | 20（全画像合計） |
| 縦長画像（4:5） | 0 | 3〜5 | 20（全画像合計） |
| ロゴ（1:1） | 1 | 1〜2 | — |
| 見出し | 1 | **5** | 5 |
| 長い見出し | 1 | **5** | 5 |
| 説明文 | 1 | **5** | 5 |
| 動画 | 0 | 2〜3 | 5 |

## 3. 配信面（プレースメント）

### 3-1. 配信面の詳細

| 配信面 | 課金モデル | 形式 | 特徴 |
|--------|----------|------|------|
| **YouTube In-Stream** | CPM | 動画 | プレロール/ミッドロール。スキッパブル |
| **YouTube In-Feed** | CPC | 画像/動画 | ホームフィード、検索結果、関連動画 |
| **YouTube Shorts** | CPM | 縦型画像/動画 | Shorts間に挿入。9:16推奨 |
| **Discover** | CPC | 画像/カルーセル | Googleアプリのフィード。モバイルメイン |
| **Gmail** | CPC | 画像 | プロモーション/ソーシャルタブ。展開型 |
| **GDN**（2025年4月〜） | CPM/CPC | 画像 | 200万以上のサイト・アプリ |
| **Google Video Partners** | CPM | 動画 | 第三者サイトの動画枠 |
| **Google Maps**（2025年後半〜） | — | プロモーテッドピン | ローカル検索時の表示 |

### 3-2. 配信面の個別選択（2025年3月〜）

2025年3月から全Demand Genキャンペーンで配信面の個別選択が可能になった。

- YouTube Shortsのみのキャンペーンが可能
- 配信面ごとに予算配分を分離可能
- YouTube / Discover / Gmail / GDN を個別にON/OFF

**活用例:**
- Shorts専用キャンペーンで縦型クリエイティブを最大活用
- GDN配信をOFFにして高品質面（YouTube/Discover）に集中
- 配信面ごとにパフォーマンスを検証してから追加

### 3-3. リーチ規模

- Demand Gen全体で約**29億ユーザー**にリーチ可能
- Demand GenのCV全体の**68%**が、過去30日間にそのブランドの検索広告を見ていないユーザーからの増分

## 4. オーディエンスターゲティング

### 4-1. ファーストパーティデータ

| 種類 | 内容 |
|------|------|
| **カスタマーマッチ** | 自社顧客リスト（メール/電話番号）をアップロード。Lookalike拡張のシードとしても使用 |
| **リマーケティングリスト** | サイト訪問者、アプリユーザー、YouTubeチャンネル視聴者/登録者 |
| **高LTVセグメント** | 高LTV顧客リストをアップロードしてLookalike精度を向上 |

### 4-2. Lookalike（類似）セグメント — Demand Gen専用

**Demand Genだけが利用できるGoogle Adsの類似オーディエンス機能。** ファーストパーティデータをシードとして、類似ユーザーに拡張する。

| リーチ設定 | 対象地域人口比 | 用途 |
|-----------|-------------|------|
| **Narrow（狭い）** | 約2.5% | 高精度。CPAを抑えたい場合 |
| **Balanced（バランス）** | 約5% | 推奨の出発点 |
| **Broad（広い）** | 約10% | リーチ最大化。認知拡大 |

**重要な変更（2026年3月〜）:** Lookalikeセグメントがデフォルトで「サジェスションモード」に移行。リーチスライダーがハードなターゲティング制約ではなく、AIへのシグナルとなる。従来のターゲティング制約動作を維持するにはオプトアウト申請が必要。

### 4-3. カスタムセグメント

- 検索キーワード（上位20〜30語）でカスタムオーディエンスを作成
- ウェブサイト訪問、アプリ使用に基づく定義も可能
- 広告グループレベルで適用

### 4-4. Googleの既成オーディエンス

| 種類 | 内容 |
|------|------|
| **購買意向の強い（In-Market）** | 特定の商品・サービスを積極的に検討中のユーザー |
| **ライフイベント** | 引越し、結婚、卒業など人生の節目 |
| **アフィニティ** | 長期的な趣味・関心・習慣 |
| **詳しいユーザー属性** | 子供の有無、婚姻状況、学歴、住宅所有、雇用状況 |

### 4-5. 最適化されたターゲティング（Optimized Targeting）

手動設定のオーディエンスを超えて、CVの見込みが高いユーザーにリーチを自動拡張する。

| シーン | 推奨設定 |
|--------|---------|
| 新規Prospecting（冷たいオーディエンス） | **ON** — 広めに探索 |
| リマーケティング | **OFF** — 対象を制御 |
| カート放棄者 | **OFF** — 対象を制御 |
| 手動セグメンテーション | **OFF** — 手動設定を尊重 |

**注意:** ONの場合、オーディエンス設定は「ターゲティング制約」ではなく「シグナル（参考情報）」として扱われ、Googleが判断で拡張する。

### 4-6. オーディエンス除外

- 既にCVしたユーザーの除外
- 他キャンペーン（特にP-MAX）との重複防止のためのオーディエンス除外

## 5. クリエイティブ設計のベストプラクティス

### 5-1. 配信面別の戦略

Demand Genは配信面ごとに課金モデルとユーザー行動が異なるため、クリエイティブの狙いを分ける必要がある。

| 配信面 | 課金 | ユーザー状態 | クリエイティブの狙い |
|--------|------|------------|------------------|
| **YouTube（In-Stream/Shorts）** | CPM | 動画視聴中 | 最初の5秒でフック。証拠・実績を早期提示。字幕必須 |
| **Discover/Gmail** | CPC | フィード閲覧中 | 価格フィルター・資格条件を見出しに入れて低品質クリックを排除 |

### 5-2. 画像の設計原則

| 原則 | 説明 |
|------|------|
| ブランドファースト | ロゴ・ブランドカラーを目立たせる。フィード上で認識されやすくする |
| インスピレーショナル | 商品単体よりも「使用シーン」「ベネフィット体験」を伝えるビジュアル |
| テキストオーバーレイは最小限 | 小さな配信面で読めない。画像面積の20%以下 |
| 高品質・高コントラスト | 鮮明でプロフェッショナルな画像。ぼやけた/低解像度は厳禁 |
| 3アスペクト比を網羅 | 横長+正方形+縦長のセットで全配信面に対応 |

### 5-3. 動画の設計原則

| 原則 | 説明 |
|------|------|
| 最初の5秒で勝負 | ブランド+価値提案を最初の5秒で提示 |
| 無音対応 | Discover/Gmail経由のユーザーは音声OFFが多い。字幕・テキストオーバーレイ必須 |
| 複数比率を用意 | 16:9 + 1:1 + 9:16 の3セットが理想 |
| Shorts用は10〜20秒 | 短尺・テンポの良い構成。最大1分 |
| 15秒以上推奨 | リーチ最大化のため15秒以上が推奨 |

### 5-4. テキスト（見出し・説明文）の設計原則

- 各見出しは**独立して**意味が通る内容にする（AIが任意の組み合わせを生成）
- 5本すべて異なる角度で作成（価格、機能、実績、限定、感情など）
- 見出しと説明文は**重複させない**（組み合わせが冗長になる）
- Discover/Gmail面（CPC課金）では、価格や条件を見出しに含めて事前フィルタリングすることで低品質クリックを削減

### 5-5. クリエイティブの更新サイクル

| 種別 | 更新推奨間隔 |
|------|------------|
| 静止画像 | 4〜6週間ごと |
| 動画 | 8〜12週間ごと |

### 5-6. AI クリエイティブ機能（2025年〜）

| 機能 | 内容 |
|------|------|
| **アニメーション画像** | 静止画から動きのあるバリエーションを自動生成 |
| **アダプティブデザイン** | 画像にブランドスタイルのテキストオーバーレイを自動追加 |
| **自動動画生成** | 静止画アセットからAIで動画を自動生成 |
| **動画短縮** | 長尺動画を配信面に適した短尺に自動編集 |
| **動画フリップ** | 横長動画から縦長（9:16）を自動生成 |

## 6. 入札戦略（Demand Gen固有）

### 6-1. 利用可能な入札戦略

| 戦略 | 目的 | 推奨シーン |
|------|------|----------|
| **コンバージョン数の最大化** | 予算内でCV数を最大化 | 新規キャンペーン、CVデータ少ない初期。小予算の場合にも推奨 |
| **目標CPA（tCPA）** | 指定CPAでCV数を最大化 | 50件以上のCVを蓄積後。予算は目標CPAの15倍以上 |
| **コンバージョン値の最大化** | 予算内でCV値を最大化 | CV値（売上等）が設定済みの場合 |
| **目標ROAS（tROAS）** | 指定ROASでCV値を最大化 | 十分なCV値データ + 明確なROAS目標がある場合 |
| **目標CPC（tCPC）**（2025年〜） | 平均CPCを制御 | キャンペーン目標が「クリック」の場合のみ利用可能。認知・トラフィック目的 |

### 6-2. Demand Gen向け入札戦略の選び方フロー

```
キャンペーンの主目的は？
├── トラフィック/認知 → 目標CPC
└── コンバージョン → CVデータはあるか？
    ├── No → コンバージョン数の最大化
    └── Yes → 50件以上のCVがあるか？
        ├── No → コンバージョン数の最大化（データ蓄積）
        └── Yes → CV値（売上額等）を計測しているか？
            ├── No → 目標CPA
            └── Yes → 目標ROAS
```

### 6-3. 入札に関する注意点

| 注意点 | 説明 |
|--------|------|
| **目標CPCの粒度** | キャンペーンレベルと広告グループレベルの両方で設定可能。広告グループレベルが優先 |
| **予算の最低ライン** | 1日あたり$100（約15,000円）以上、または目標CPAの15〜20倍が推奨 |
| **小予算の場合** | 地域を限定すれば1日3,000〜5,000円でも開始可能 |
| **学習期間** | 新キャンペーン開始後3〜4週間は大幅な変更を避ける |

## 7. 商品フィード連携（EC向け）

### 7-1. 前提条件

| 条件 | 詳細 |
|------|------|
| Merchant Centerアカウント | 有効で承認済み（承認に最大3日） |
| Google Adsとのリンク | Merchant CenterとGoogle Adsアカウントを連携 |
| 最低商品数 | **4商品以上**（推奨: 50商品以上） |
| CVゴール | キャンペーンに「購入」CVゴールを設定 |
| 地域一致 | ターゲット地域とMerchant Centerの配信地域設定が一致 |

### 7-2. フィードの主要属性

| 属性 | 仕様 | 備考 |
|------|------|------|
| **title** | 最大150文字 | 広告タイトルとして表示 |
| **short_title** | — | 短いキャッチーなタイトル用 |
| **description** | 最大5,000文字 | 商品説明 |
| **image** | 最小300×300px、最大10MB | 商品画像 |
| **price** | 通貨コード含む | 価格表示 |
| **item_group_id** | — | バリエーション商品の重複画像を防止 |

> **商品フィードの詳細な最適化方法は [shopping-ads.md](shopping-ads.md) を参照。**

### 7-3. サイトリンク

- 最小4本、最大20本
- 表示URLパス: 各最大15文字

### 7-4. 2025年のフィード強化

- 広告内で商品詳細ページに遷移せずに全情報を閲覧可能に
- ローカル在庫表示で実店舗への集客にも対応
- オムニチャネルオークション: オンライン/店舗購入を自動最適化

## 8. キャンペーン構成と運用設計

### 8-1. キャンペーン構成の原則

| 原則 | 説明 |
|------|------|
| **統合型で開始** | 最初は1キャンペーンに統合してAIの学習速度を上げる |
| **広告グループで配信面を分離** | YouTube Shorts / YouTube In-Stream / Discover+Gmail 等を広告グループで分け、配信面別の成果を可視化 |
| **リマケとProspectingは分離** | 暖かいオーディエンスと冷たいオーディエンスを混ぜない |
| **非共有予算** | Demand Genは共有予算プール（共有予算）を使用できない |

### 8-2. 推奨キャンペーン構成テンプレート

```
Demand Gen（3広告グループ構成）
│
├── Ad Group 1: [DG] Prospecting - Lookalike
│   ├── ターゲティング: Lookalike（Balanced）+ カスタムセグメント
│   ├── 最適化されたターゲティング: ON
│   ├── 除外: リマーケティングリスト、CV済みユーザー
│   ├── クリエイティブ: 画像+動画（3アスペクト比）
│   └── 配信面: YouTube + Discover + Gmail
│
├── Ad Group 2: [DG] Prospecting - In-Market
│   ├── ターゲティング: 購買意向セグメント + アフィニティ
│   ├── 最適化されたターゲティング: ON
│   ├── 除外: リマーケティングリスト、CV済みユーザー
│   └── クリエイティブ: 画像+動画（3アスペクト比）
│
└── Ad Group 3: [DG] Remarketing
    ├── ターゲティング: サイト訪問者 / YouTube視聴者 / カスタマーマッチ
    ├── 最適化されたターゲティング: OFF
    ├── 除外: CV済みユーザー
    └── クリエイティブ: リマケ用メッセージ（限定/リマインド）
```

### 8-3. Shorts専用キャンペーン（オプション）

```
Demand Gen（Shorts特化）
├── 配信面: YouTube Shortsのみ（チャネルコントロール使用）
├── クリエイティブ: 縦型9:16の動画/画像に特化
├── 動画長: 10〜20秒
└── 目的: Shorts面の効果を単独検証
```

### 8-4. 予算設計

| 項目 | 推奨 |
|------|------|
| 最低日予算 | 目標CPAの15〜20倍 or 15,000円/日 |
| 小規模テスト | 地域限定で3,000〜5,000円/日も可 |
| 総キャンペーン予算 | 配信期間全体で均等に消化する設定も可能 |

## 9. 計測と評価

### 9-1. 重要指標

| 指標 | 用途 |
|------|------|
| **CPA / ROAS** | 直接的なパフォーマンス評価 |
| **VTC（ビュースルーCV）** | 広告を見たがクリックせず後日CVしたユーザー |
| **プラットフォーム比較可能CV**（2025年〜） | Meta/TikTok/LinkedInとの直接比較が可能な新指標 |
| **帰属ブランド検索数**（2026年1月〜） | Demand Genが引き起こしたブランド検索のボリューム |
| **アシストCV** | 最終CVに至るまでに貢献したDemand Genの接触回数 |

### 9-2. パフォーマンスの正しい読み方

Demand Genは検索広告と異なり、ミッドファネルの需要喚起型キャンペーン。レポート上のCVだけでは効果を過小評価しやすい。

| 評価方法 | 説明 |
|---------|------|
| **ブレンドCPA** | アカウント全体の合算CPAで判断（Demand Gen単体のCPAだけで判断しない） |
| **アシストCV** | Demand Genが初回接触→検索広告でCV、というパスを確認 |
| **ブランド検索の増減** | Demand Gen開始前後でブランド検索ボリュームが増加したか |
| **地域ホールドアウト** | 配信地域 vs 非配信地域のCV率を比較 |
| **期間ON/OFF** | 一定期間停止し、停止前後のCV数変化を確認 |

**注意:** P-MAXがDemand Genの初回接触CVを横取りして最終CVとして計上するケースが多い。アシストCVレポートとブレンドCPAで総合評価すること。

### 9-3. アセットレベルレポート

アセットごとのパフォーマンスラベル:

| ラベル | 意味 | アクション |
|--------|------|----------|
| 最良 | 他のアセットより優れている | 維持。類似パターンを追加テスト |
| 良い | 平均的 | 維持しつつ改善版をテスト |
| 低い | 他より劣る | 新しいアセットに差し替え |
| 学習中 | データ蓄積中 | 結果が出るまで待つ |

## 10. A/Bテスト（エクスペリメント）

### 10-1. テストの種類

| 種類 | 内容 |
|------|------|
| **アセットA/Bテスト** | クリエイティブを単一変数としてテスト（画像/動画/見出し/CTAの比較） |
| **カスタムエクスペリメント** | オーディエンス、入札戦略、フォーマットを含むマルチ変数テスト |

### 10-2. 設定要件

| 項目 | 要件 |
|------|------|
| 必要キャンペーン数 | 2本（新規で未配信のキャンペーン） |
| 変数 | **1つだけ変える**（それ以外の条件は統一） |
| トラフィック分割 | 50/50推奨（80/20も可） |
| 推奨期間 | 3〜4週間以上（30〜60日が理想） |
| 必要CV数 | 最低100件で信頼性のある結果 |
| 信頼度 | 70% / 80% / 95% の3段階で表示 |

### 10-3. テスト対象の例

| テスト対象 | 例 |
|-----------|-----|
| フォーマット | 動画 vs 画像広告 |
| CTA | 「詳細を見る」vs「今すぐ購入」 |
| 動画比率 | 縦長 vs 正方形 |
| クリエイティブ | 画像のみ vs 画像+動画ミックス |
| ターゲティング | Lookalike vs In-Market |
| ターゲティング | デモグラフィック追加 vs 最適化ターゲティング |
| 入札 | コンバージョン最大化 vs 目標CPA |

### 10-4. テスト時の注意

- テスト期間中にキャンペーンを**編集しない**（結果が歪む）
- **1つの変数だけ**を変えて因果関係を特定する
- 結果は「エクスペリメント」タブで確認

## 11. 他キャンペーンとの関係

### 11-1. Demand Gen vs P-MAX

| 項目 | Demand Gen | P-MAX |
|------|-----------|-------|
| **ファネル位置** | 上部〜中部（需要喚起・検討促進） | 全段階（特にCV最大化） |
| **オーディエンス制御** | 高い（ターゲティングパラメータを指定） | 低い（AI全自動） |
| **配信面** | YouTube/Discover/Gmail/GDN/Maps | 全7チャネル（検索含む） |
| **フォーマット** | 画像/動画/カルーセル/商品フィード | テキスト/画像/動画 |
| **Lookalike** | 利用可能（Demand Gen専用） | 利用不可 |
| **レポート透明性** | 高い（配信面別・アセット別） | 改善中だが制限あり |
| **戦略的役割** | 意図を「作る」、教育する、探索する | 意図を「刈り取る」、CV、リマケ |

**共存のポイント:**
- Demand Gen = Prospecting・教育系クリエイティブ
- P-MAX = リマーケティング・オファー系クリエイティブ
- オーディエンス除外で重複・カニバリを防止

### 11-2. Demand Gen vs ディスプレイ

| 項目 | Demand Gen | ディスプレイ |
|------|-----------|------------|
| **配信面** | YouTube+Discover+Gmail+GDN+Maps | GDNのみ |
| **フォーマット** | 画像/動画/カルーセル/商品フィード | バナー/リッチメディア/動画 |
| **Lookalike** | 利用可能 | 利用不可（最適化ターゲティングで代替） |
| **コンテンツターゲティング** | 不可 | 可能（KW/トピック/プレースメント指定） |
| **AI最適化度** | 高い | 中 |

**使い分け:**
- **ディスプレイ**: GDN内の特定プレースメント・トピック・KWでのコンテンツターゲティングが必要な場合
- **Demand Gen**: YouTube/Discoverへの配信、ビジュアルリッチなクリエイティブ、Lookalikeセグメント活用

> 2025年4月にDemand GenにGDN画像在庫が追加され、Demand Genはディスプレイのスーパーセットになりつつある。Googleは将来的にDemand Genをディスプレイの後継に位置づける方向。

## 13. よくある失敗パターン（Demand Gen固有）

| 失敗 | 回避策 |
|------|--------|
| **予算が少なすぎる** | 1日1万〜3万円程度は最低ライン。$10/日では学習が進まない |
| **初手でDemand Genを使う** | 先に検索/ショッピング/P-MAXでCVベースラインを確立すべき。Demand Genは支援型キャンペーン |
| **リマケとProspectingの混在** | 暖かい/冷たいオーディエンスを同一広告グループに入れない。データが混濁し最適化が困難 |
| **最適化ターゲティングの誤用** | リマケ/カート放棄者でONにすると対象が無制限に拡張。冷たいProspectingのみON |
| **レポートの鵜呑み** | Google Adsのレポート値だけでは過少計上。ブレンドCPA・アシストCV・ブランド検索増を確認 |
| **早期の撤退判断** | ミッドファネルのためCV到達が遅い。最低3ヶ月はデータを見る |
| **汎用的なクリエイティブ1パターン** | 見出し5本・説明文5本・画像15枚以上の多様なアセットが必要 |
| **無効トラフィックの見落とし** | 0秒セッション、異常なマイクロCV率をチェック。Google自社面（YouTube/Discover/Gmail）から開始し、GDN追加は品質確認後に |
| **P-MAXとのカニバリ放置** | P-MAXがDemand Genの初回接触CVを最終CVとして横取りする。オーディエンス除外+アシストCVレポートで管理 |
| **配信面のデータ未活用** | YouTube In-Stream/Shorts/Discover/Gmailで成果が異なる。チャネルコントロールで予算配分を最適化 |

## 14. 運用チェックリスト

### 配信開始前チェック

- [ ] コンバージョントラッキングが正しく設定されている
- [ ] 画像アセット: 横長・正方形の両方を登録（推奨: 各5枚以上+縦長3枚以上）
- [ ] ロゴを登録済み
- [ ] 見出し5本・長い見出し5本・説明文5本を登録
- [ ] ビジネス名を設定
- [ ] 動画アセット: 可能なら3比率（16:9/1:1/9:16）を登録
- [ ] 入札戦略を確認（初期: コンバージョン数の最大化）
- [ ] 日予算が目標CPAの15倍以上
- [ ] オーディエンス設定: Prospectingとリマケは広告グループを分離
- [ ] 最適化されたターゲティング: Prospecting=ON / リマケ=OFF
- [ ] オーディエンス除外: CV済みユーザーを除外
- [ ] P-MAXとの重複防止のためのオーディエンス除外を設定
- [ ] 配信面の選択を確認（初期: Google自社面のみ推奨）
- [ ] LPの表示速度がモバイルで3秒以内
- [ ] LPのメッセージが広告クリエイティブと一致

### 週次チェック

- [ ] 配信量（imp/click/cost）の推移を確認
- [ ] CTR/CPC/CPAを確認
- [ ] VTC（ビュースルーCV）を確認
- [ ] 配信面別パフォーマンスを確認
- [ ] 無効トラフィックの兆候（0秒セッション等）を確認

### 月次チェック

- [ ] アセットパフォーマンスレポートを確認
- [ ] 「低い」アセットの差し替え
- [ ] 新しいクリエイティブのテスト追加（静止画は4〜6週間ごと）
- [ ] Lookalikeセグメントのリーチ設定を見直し
- [ ] 入札戦略の移行を検討（CV50件以上でtCPA移行）
- [ ] 予算配分の見直し
- [ ] ブランド検索ボリュームの変化を確認
- [ ] アシストCVレポートでDemand Genの貢献を評価

## 15. 広告ポリシー

Demand Genの広告ポリシーはディスプレイ広告と共通。詳細は [display-ads.md 5章](display-ads.md) を参照。

追加の注意点:

| 項目 | 説明 |
|------|------|
| **Discover面の品質基準** | Discoverフィードはパーソナライズされた高品質コンテンツが求められる。過度な煽り・クリックベイトは不承認率が高い |
| **YouTube Shorts面** | ショート動画として自然に見えるクリエイティブが推奨。広告感が強すぎるとスキップ率が高い |
| **カルーセルの一貫性** | 全カードで一貫したメッセージ・品質が必要。1枚だけ品質が低いと全体が不承認になる可能性 |
