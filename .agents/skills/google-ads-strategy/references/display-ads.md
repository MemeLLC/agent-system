# Google ディスプレイ広告

---

## 1. 広告フォーマット

### 1-1. レスポンシブ ディスプレイ広告（RDA）

**現在のデフォルト形式。** 画像・ロゴ・見出し・説明文をアップロードすると、Google AIが配信面に応じて最適な組み合わせ・サイズに自動調整する。

- GDN上の画像広告の大半がレスポンシブ形式で配信されている
- 単一のアセットセットで全枠に対応でき、運用工数が少ない
- Google AIが配信面・ユーザーに応じた最適組み合わせを学習

### 1-2. アップロード型ディスプレイ広告

ピクセル単位でデザインを完全にコントロールしたい場合に使用。

| 種類 | 特徴 |
|------|------|
| 静止画像広告 | JPG/PNG/GIF。サイズごとに個別作成が必要 |
| アニメーションGIF | 30秒以内、5fps以下。ファイルサイズ150KB以下 |
| HTML5広告 | ZIP形式。インタラクティブな表現が可能 |
| AMPHTML広告 | AMP準拠。高速表示でモバイルに最適 |

### 1-3. 動画広告（ディスプレイ枠）

- レスポンシブ ディスプレイ広告にYouTube動画をアセットとして追加可能
- 最大30秒の動画を最大5本まで設定可能
- 推奨アスペクト比: 16:9、1:1、4:3、9:16
- 動画アセットを含む広告は静止画のみの場合と比較してCTRが向上する傾向

## 2. レスポンシブ ディスプレイ広告（RDA）の入稿仕様

### 画像アセット

| 項目 | 仕様 |
|------|------|
| **横長（必須）** | アスペクト比 1.91:1 / 推奨 1200×628px / 最小 600×314px |
| **スクエア（必須）** | アスペクト比 1:1 / 推奨 1200×1200px / 最小 300×300px |
| **縦長（任意）** | アスペクト比 4:5 / 推奨 1200×1500px / 最小 320×400px |
| ファイル形式 | JPEG または PNG（アニメーションGIF不可） |
| ファイルサイズ | 各5MB以下 |
| 最大登録数 | **15枚** |
| 推奨登録数 | 各アスペクト比 5〜10枚 |

**注意:** Googleは配信面に合わせて画像をトリミング・リサイズすることがある。重要な要素は画像中央に配置すること（セーフゾーン）。

### ロゴ

| 項目 | 仕様 |
|------|------|
| **スクエアロゴ** | アスペクト比 1:1 / 推奨 1200×1200px / 最小 128×128px |
| **横長ロゴ（任意）** | アスペクト比 4:1 / 推奨 1200×300px / 最小 512×128px |
| ファイル形式 | PNG（透過背景推奨）または JPG |
| ファイルサイズ | 各5MB以下 |
| 最大登録数 | **5個** |

### テキストアセット

| 項目 | 仕様 | 備考 |
|------|------|------|
| **短い見出し** | 最大30文字 × 最大5本 | 日本語は1文字=2文字でカウント（実質15文字） |
| **長い見出し** | 最大90文字 × 1本 | 日本語は実質45文字 |
| **説明文** | 最大90文字 × 最大5本 | 日本語は実質45文字 |
| **ビジネス名** | 最大25文字 × 1つ | 日本語は実質12〜13文字 |
| **CTA（行動を促すフレーズ）** | プルダウンから選択 | 「自動」推奨。手動選択も可 |

**日本語の文字数カウント:** 日本語・中国語・韓国語の文字は1文字が2文字としてカウントされる。半角英数字は1文字=1文字。

### 動画アセット（任意）

| 項目 | 仕様 |
|------|------|
| 最大登録数 | 5本 |
| 最大再生時間 | 30秒 |
| 推奨アスペクト比 | 16:9、1:1、4:3、9:16 |
| 登録方法 | YouTubeの動画URLを指定 |

### 推奨アセット数のまとめ

| アセット | 最低限 | 推奨 | 上限 |
|---------|-------|------|------|
| 横長画像（1.91:1） | 1 | 5〜10 | 15（全画像合計） |
| スクエア画像（1:1） | 1 | 5〜10 | 15（全画像合計） |
| 縦長画像（4:5） | 0 | 2〜5 | 15（全画像合計） |
| ロゴ（1:1） | 1 | 2〜3 | 5 |
| ロゴ（4:1） | 0 | 1〜2 | 5（全ロゴ合計） |
| 短い見出し | 1 | **5** | 5 |
| 長い見出し | 1 | 1 | 1 |
| 説明文 | 1 | **5** | 5 |
| 動画 | 0 | 1〜3 | 5 |

**アセットを多く登録する理由:** 複数のアセットバリエーションにより、Google AIがより多くの組み合わせをテストでき、各配信面・ユーザーに最適な広告を表示できる。Google公式では、複数のアセットバリエーションを使用することで同等のCPAでCV数が向上する傾向が示されている（効果は業界・配信規模・アセット品質により変動）。

## 4. クリエイティブ設計のベストプラクティス

### 4-1. 画像の設計原則

#### やるべきこと

| 原則 | 説明 |
|------|------|
| 高品質な画像を使用 | 鮮明でプロフェッショナルな画像。ぼやけた・低解像度の画像は厳禁 |
| 被写体を中央に配置 | Googleによるトリミングでも重要要素が残るよう、中央寄りに |
| シンプルな構図 | 1つの明確なメッセージ。要素の詰め込みすぎは逆効果 |
| 高コントラストの配色 | 背景と前景のコントラストを強く。サイト上で広告が目立つ |
| 商品/サービスを見せる | 実際の製品写真や利用シーンで訴求力を上げる |
| ブランド要素を統一 | 色・フォント・トーンをブランドガイドラインに沿わせる |
| 実写 > イラスト | 一般的に実写写真のほうがCTR・CVRが高い傾向 |

#### 原則避けること

| 項目 | 理由 | 補足 |
|------|------|------|
| 画像にテキストを重ねる | 小さな配信面（320×50等）で読めない。見出しと重複して冗長になる | やむを得ず入れる場合: 最小フォントサイズ16px以上、高コントラスト、画像面積の20%以下に抑える |
| ロゴを画像内に配置 | ロゴは専用のロゴアセットとして登録すべき | RDAではロゴアセットが自動配置されるため、画像内ロゴと二重表示になる |
| コラージュ画像を使う | 小さなサイズで判別しにくい。1枚の画像に1つのメッセージ | — |
| 過度なフィルター加工 | 不自然に見える。自然な見た目を維持 | — |
| ぼやけた/低解像度画像 | 品質の低さがブランドイメージを損なう | — |
| 白背景のみの画像 | 白背景のサイトで広告が溶け込んでしまう | 薄いボーダーや背景色を追加して境界を明確にする |

#### 絶対禁止（ポリシー違反）

| 項目 | 理由 |
|------|------|
| ボタンのような偽UI要素 | ポリシー違反（ミスリーディング）の対象。広告不承認になる |
| 偽のシステム警告・通知 | ウイルス警告やOS通知に見せかけた画像 |

### 4-2. テキスト（見出し・説明文）の設計原則

#### 短い見出し（30文字 / 日本語実質15文字）

- 商品・サービス・ブランドを明確に伝える
- 特典・価格・プロモーションを含めるとCTR向上
- 各見出しは独立して意味が通る内容にする
- 5本すべて異なる角度で作成（価格訴求、機能訴求、感情訴求、限定訴求、実績訴求など）

#### 長い見出し（90文字 / 日本語実質45文字）

- 短い見出しが表示されない配信面で代わりに表示される
- 価値提案を1文で完結させる
- 説明文なしで表示されることもあるため、単体で機能する内容にする

#### 説明文（90文字 / 日本語実質45文字）

- 見出しを補完する追加情報を記載
- 具体的なベネフィットやCTAを含める
- 5本すべて異なる内容で、どの見出しと組み合わされても自然に読めるようにする
- 見出しの言い換えにしない（Googleは見出し+説明文を自動組み合わせする）

### 4-3. CTAの選び方

| CTA | 適する場面 |
|-----|---------|
| 自動（推奨） | Google AIが最適なCTAを選択。迷ったらこれ |
| 詳細 | 情報提供・認知拡大 |
| 今すぐ購入 | EC・直接販売 |
| お申し込み | 会員登録・サブスクリプション |
| お問い合わせ | B2Bリード獲得 |
| 今すぐダウンロード | アプリ・資料ダウンロード |

### 4-4. アップロード型広告のデザインポイント

| ポイント | 説明 |
|---------|------|
| CTAボタンを目立たせる | 太めのフォント + コントラストの強いボタンカラー |
| 視線誘導を意識 | 画像 → 見出し → CTA の流れを自然に設計 |
| 余白を活かす | 詰め込みすぎず、呼吸のあるレイアウト |
| ファイルサイズを150KB以内に | 圧縮ツールで最適化。表示速度に直結 |
| 全サイズで一貫性を保つ | サイズが変わっても同じブランド体験を提供 |
| アニメーションは控えめに | 動きは注目を集めるが、うるさいとスキップされる |

### 4-5. 動画アセットの活用

- 動画を含むディスプレイ広告は、静止画のみと比較してCTRが向上する傾向
- 最初の5秒でブランドと価値提案を伝える
- 無音でも伝わるように設計する（テキストオーバーレイ・字幕活用）
- 複数のアスペクト比（16:9 + 1:1 + 9:16）を用意すると配信面が広がる

## 5. 広告ポリシー・規約

### 5-1. 編集ポリシー

Google Adsは全広告に対し、明確でプロフェッショナルな品質基準を要求する。

#### テキストに関する規則

| カテゴリ | 禁止例 | 正しい例 |
|---------|--------|---------|
| 不要な記号の多用 | `!!!`、`???`、`★★★` | 句読点は1回ずつ |
| ALL CAPS（英語） | `FREE SHIPPING NOW` | `Free Shipping Now` |
| 意味のない記号 | `F₹€€!!`、`f-r-e-e` | 通常の表記 |
| 曖昧すぎるコピー | 「ここで商品を購入」 | 具体的な商品名・ベネフィットを記載 |
| スペースの乱用 | `買 う な ら 今` | `買うなら今` |
| 広告文内の電話番号 | テキスト内に番号記載 | 電話番号アセットを使用 |

#### 画像に関する規則

| カテゴリ | 禁止事項 |
|---------|---------|
| ぼやけた/不鮮明な画像 | 低解像度・ピクセル化した画像 |
| 過度な色反転・フィルター | 不自然な加工 |
| 傾いた/逆さまの画像 | 正しい向きで表示 |
| 判読不能なテキスト | 画像内テキストが小さすぎて読めない |
| ボタン等の偽UI要素 | クリックできるように見せかける画像 |

### 5-2. クリックベイト・ミスリーディング禁止

以下の手法は広告不承認の対象:

| 禁止手法 | 例 |
|---------|-----|
| センセーショナリズム | 「衝撃の事実が判明」「信じられない結果」 |
| クリック誘導文言 | 「ここをクリックして確認」「見ないと損」 |
| ビフォーアフター画像 | 身体の大幅な変化を示す比較画像 |
| 事故・災害画像の利用 | 実際の事故写真を使った恐怖訴求 |
| 感情操作 | 死亡・逮捕・病気などの否定的事象で恐怖・罪悪感を煽る |
| 誇大な身体パーツの拡大 | 明らかに加工されたズームイン画像 |

### 5-3. 制限コンテンツ

以下のカテゴリは広告配信可能だが制限あり:

| カテゴリ | 制限内容 |
|---------|---------|
| アルコール | 地域の法律に準拠。未成年ターゲティング不可 |
| ギャンブル | 事前認定が必要。地域制限あり |
| ヘルスケア・医薬品 | 地域の法律・認定に準拠 |
| 金融サービス | 金利・手数料等の明示が必要な場合あり |
| 政治広告 | 身元確認・地域制限 |

### 5-4. 禁止コンテンツ

| カテゴリ | 内容 |
|---------|------|
| 偽造品 | ブランドの偽造品の宣伝 |
| 危険な商品 | 薬物、武器、爆発物等 |
| 不正行為の助長 | ハッキング、偽造文書等 |
| 不適切なコンテンツ | ヘイト、暴力、成人向け等 |

### 5-5. リンク先の要件

| 要件 | 説明 |
|------|------|
| 表示URLとの一致 | 広告の表示URLとリンク先のドメインが一致すること |
| ページが正常に機能 | 工事中・エラーページは不可 |
| ブラウザ戻るボタン | 無効化されていないこと |
| 有用なコンテンツ | ユーザーにとって価値のある内容であること |
| モバイル対応 | モバイルデバイスで正常に表示されること |

### 5-6. ポリシー違反時の対応

1. **警告**: 初回違反は通常7日前の警告が発行される
2. **広告不承認**: 個別広告の配信停止
3. **アカウント停止**: 繰り返しの重大違反で停止の可能性
4. **異議申し立て**: Google Ads管理画面から申請可能

## 6. ターゲティング戦略

### 6-1. オーディエンスターゲティング

| 種類 | 内容 | 適する目的 |
|------|------|----------|
| **アフィニティ** | 趣味・関心・習慣に基づくセグメント | 認知拡大（上位ファネル） |
| **購買意向の強い（In-Market）** | 特定の商品・サービスを積極的に調べているユーザー | 検討促進（中位ファネル） |
| **カスタムセグメント** | キーワード・URL・アプリで独自定義 | 柔軟なターゲティング |
| **リマーケティング** | サイト訪問済みユーザー | CV促進（下位ファネル） |
| **カスタマーマッチ** | 自社顧客リスト（メール等）をアップロード | 既存顧客へのリーチ・シード |
| **デモグラフィック** | 年齢・性別・世帯年収・子供の有無 | 属性ベースの絞り込み |
| **ライフイベント** | 結婚・引越し・卒業などの人生の節目 | 特定タイミングの訴求 |

### 6-2. コンテンツターゲティング

| 種類 | 内容 |
|------|------|
| **キーワード** | 指定キーワードに関連するコンテンツのページに配信 |
| **トピック** | 指定トピックに関連するページに配信 |
| **プレースメント** | 特定のウェブサイト・アプリ・YouTubeチャンネルを指定 |

### 6-3. 最適化されたターゲティング（Optimized Targeting）

- ディスプレイキャンペーンでは広告グループレベルでデフォルトON
- Google AIが手動設定のターゲティングを超えて、CVの見込みが高いユーザーにリーチを自動拡張
- ファーストパーティデータ（リマーケティングリスト、カスタマーマッチ）をシードとして活用し、類似ユーザーへ拡張する（旧「類似セグメント」の実質的な後継機能）
- 初期段階やCV数が少ない場合は有効。十分なCVデータがあるほど精度が向上
- コントロールを重視する場合はOFFにすることも検討（Prospecting vs リマケでON/OFF比較テスト推奨）

### 6-4. ターゲティング vs オブザベーション

ディスプレイキャンペーンでは、オーディエンスセグメントの追加時に「ターゲティング」か「オブザベーション」を選択できる。

| モード | 動作 | 使い方 |
|--------|------|--------|
| **ターゲティング** | 指定したオーディエンス/コンテンツにのみ配信を制限 | リーチを絞りたい場合 |
| **オブザベーション** | リーチを絞らず配信し、指定条件でのパフォーマンスを監視・レポート | データ収集→入札調整の参考。リーチを減らさずに効果を検証したい場合 |

**使い分けの例:** リマーケティングリストは「ターゲティング」で絞り込み、アフィニティやデモグラフィックは「オブザベーション」で追加してパフォーマンスを確認した上で、後日「ターゲティング」に切り替えるアプローチが有効。

### 6-5. 旧用語→現行UI用語の対応

| 旧用語 | 現行（2026年時点） | 備考 |
|--------|-------------------|------|
| 類似セグメント（Similar Audiences） | **廃止**（2023年8月） | 代替: 最適化されたターゲティング + ファーストパーティデータ |
| 類似オーディエンス | **廃止** | 同上 |
| Lookalike セグメント | **Demand Gen専用**（ディスプレイキャンペーンでは利用不可） | Demand Genではカスタマーマッチ等から類似拡張可能 |
| オーディエンス拡張 | **最適化されたターゲティング**に統合 | 名称・UIが変更 |

### 6-6. ターゲティング設計の注意点

| 注意点 | 説明 |
|--------|------|
| 絞りすぎない | ターゲティングを複数レイヤーで重ねると配信量が極端に減る |
| リマーケティングは別広告グループ | 新規向けとリマーケティングは分離し、メッセージを変える |
| 除外設定も活用 | 不要なオーディエンス・プレースメントの除外でムダを削減 |
| デモグラフィックの「不明」 | 「不明」を除外するとリーチが大幅に減る。慎重に判断 |

## 7. 入札戦略（ディスプレイ固有）

### 7-1. スマート入札（推奨）

| 戦略 | 目的 | 使い方 |
|------|------|--------|
| **コンバージョン数の最大化** | CV数を最大化 | CV計測が完了し、目標CPAが未定の初期フェーズ |
| **目標CPA** | 指定CPAでCV数を最大化 | 十分なCV実績（月30件以上が目安）がある場合 |
| **コンバージョン値の最大化** | CV値（売上等）を最大化 | CV値が設定済みの場合 |
| **目標ROAS** | 指定ROASでCV値を最大化 | 十分なCV値データがある場合 |

### 7-2. ディスプレイ特有の入札戦略

| 戦略 | 目的 | 注意点 |
|------|------|--------|
| **視認範囲のインプレッション単価（vCPM）** | 視認可能な1000回表示あたりの単価 | ブランド認知が目的の場合に有効。ディスプレイ/動画向け |

**注意:** eCPC（拡張CPC）は2025年3月に検索・ディスプレイキャンペーンで廃止された。既存のeCPCキャンペーンは手動CPCとして動作する。

### 7-3. ディスプレイ向け入札戦略の選び方フロー

```
CV計測は完了しているか？
├── No → クリック数の最大化 or 手動CPC or vCPM
└── Yes → 月間CV数は30件以上か？
    ├── No → コンバージョン数の最大化（学習用データを蓄積）
    └── Yes → CV値（売上額等）を計測しているか？
        ├── No → 目標CPA
        └── Yes → 目標ROAS
```

## 8. 計測と評価（Display版）

ディスプレイ広告は検索広告と異なり「クリック以外の貢献」「ビュースルー接触」「誤クリック」が発生しやすい。検索広告と同じCPA基準だけで評価すると、ディスプレイの価値を過小評価するか、逆に無駄な配信を放置する原因になる。

### 8-1. ビュースルーコンバージョン（VTC）

ユーザーが広告を見たがクリックせず、後日直接サイトを訪問してCVした場合にカウントされる。

| 項目 | 説明 |
|------|------|
| 計測条件 | 広告面積の50%以上が1秒以上表示された場合（Active View基準） |
| デフォルト計測ウィンドウ | 1日 |
| 調整可能な範囲 | 1日〜30日 |
| 重複除外 | 同一ユーザーが広告をクリックした場合は、クリックCVが優先されVTCは除外される |

**VTCの運用方針:**

| 方針 | 使い方 |
|------|--------|
| **モニタリングのみ（推奨の出発点）** | VTCは入札最適化に含めず、レポートで参考値として監視。認知施策の間接効果を把握する |
| **入札に含める（CV数が十分な場合）** | ウィンドウを短く設定（1日）した上で、CVアクションの設定で「Primary」に含めて入札最適化に活用。特にリマーケティングで有効 |

**注意:** VTCを過大評価すると、実際にはCVに寄与していないインプレッションにも予算が流れるリスクがある。VTCウィンドウを短くする（1日）ことで過大計上を抑制できる。

### 8-2. Primary / Secondary コンバージョンの使い分け

Google Adsでは、CVアクションを「Primary（メイン）」と「Secondary（サブ）」に分けて管理できる。

| 種別 | 入札最適化 | レポート | ディスプレイでの使い方 |
|------|----------|---------|-------------------|
| **Primary** | 使用される | 「コンバージョン」列に含まれる | 入札戦略で最適化したいCVアクション（購入、フォーム送信等） |
| **Secondary** | 使用されない | 「すべてのコンバージョン」列のみ | モニタリング用（ページ閲覧、スクロール、VTC等） |

**設計例:**

| ビジネス | Primary | Secondary |
|---------|---------|-----------|
| EC | 購入 | カート追加、商品閲覧 |
| B2Bリード | フォーム送信 | 資料DL、電話タップ、VTC |
| 認知目的 | — （CVなしでvCPM入札） | サイト訪問、ページ滞在 |

### 8-3. 目的別KPI

ディスプレイ広告は目的によって見るべき指標が大きく異なる。

| 目的 | 主要KPI | 補助KPI |
|------|--------|--------|
| **認知拡大** | vCPM（視認可能CPM）、リーチ数、フリークエンシー | ブランド検索の増加率、サイトDirect流入の変化 |
| **検討促進** | CTR、LP直帰率、ページ滞在時間 | マイクロCV（資料DL、動画視聴等） |
| **CV獲得** | CPA、CV数、ROAS | VTC、アシストCV、LTV |
| **リマーケティング** | CPA、CV数、ROAS | VTC、フリークエンシー、リスト消化率 |

### 8-4. インクリメンタル評価（簡易版）

ディスプレイ広告が「本当に増分のCVを生んでいるか」を検証する方法。

| 手法 | やり方 | 精度 |
|------|--------|------|
| **地域ホールドアウト** | 対象地域A（配信あり）vs 対象地域B（配信なし）でCV率を比較 | 中〜高 |
| **期間ON/OFF** | 一定期間ディスプレイを停止し、停止前後のCV数・ブランド検索量を比較 | 中 |
| **最適化ターゲティング ON/OFF** | 同条件の広告グループで最適化ターゲティングのON/OFFを比較 | 中 |
| **Prospecting vs リマケ分離比較** | キャンペーンを分離して各々のインクリメンタル貢献を可視化 | 中 |
| **Google Ads コンバージョンリフト** | Google公式のリフト計測機能を利用（利用条件あり） | 高 |

### 8-5. ランディングページの要件（Display固有）

ディスプレイ広告はユーザーが受動的な状態でクリックするため、検索広告以上にLPでの離脱が起きやすい。

| 要件 | 理由 |
|------|------|
| **メッセージマッチ** | 広告のビジュアル・見出しとLPのファーストビューが一致していること。不一致は即離脱 |
| **表示速度** | モバイルで3秒以内に表示。ディスプレイ経由はモバイル比率が高い |
| **明確なCTA** | 何をすべきかがファーストビューで分かること |
| **フォームの簡素さ** | リード獲得の場合、フィールド数は最小限に |

## 9. キャンペーン構成と運用設計

### 9-1. キャンペーン分離の考え方

| 分離軸 | 例 | 理由 |
|--------|-----|------|
| **目的別** | 認知拡大 / リマーケティング / CV獲得 | 入札戦略・KPIが異なる |
| **商材別** | 製品A / 製品B | 予算配分を個別管理 |
| **地域別** | 日本 / 海外 | ターゲティング・メッセージが異なる |

### 9-2. 推奨キャンペーン構成テンプレート

ディスプレイは「Prospecting（新規）とリマーケティングを混ぜない」ことが最適化の基本。以下は3キャンペーン構成の推奨テンプレート。

```
ディスプレイ（3キャンペーン構成）
│
├── Campaign 1: [Display] Prospecting - 商材名
│   ├── 入札: コンバージョン数の最大化 or 目標CPA
│   ├── ターゲティング: アフィニティ / In-Market / カスタムセグメント
│   ├── 最適化されたターゲティング: ON（初期。データ蓄積後にON/OFFテスト）
│   ├── 除外: リマーケティングリスト（既存訪問者・CV済み）
│   └── AG1: テーマA / AG2: テーマB ...
│
├── Campaign 2: [Display] Remarketing - 商材名
│   ├── 入札: 目標CPA or コンバージョン数の最大化
│   ├── ターゲティング: リマーケティングリスト
│   │   ├── AG1: サイト訪問（7日以内）
│   │   ├── AG2: サイト訪問（8〜30日）
│   │   └── AG3: カート放棄 / フォーム未完了 等
│   ├── 最適化されたターゲティング: OFF（リマケは対象を絞る）
│   └── 除外: CV済みユーザー
│
└── Campaign 3: [Display] Awareness - 商材名（認知目的の場合のみ）
    ├── 入札: vCPM or クリック数の最大化
    ├── ターゲティング: 広めのオーディエンス
    ├── フリークエンシーキャップ: 1日3〜5回
    └── KPI: リーチ数、vCPM、ブランド検索増加率
```

**ポイント:**
- Prospectingキャンペーンからリマーケティングリストを除外し、重複を防ぐ
- リマーケティングキャンペーンでは最適化されたターゲティングをOFFにして対象を制御
- アプリ在庫で誤クリックが多い場合は、プレースメント除外で対応（章10参照）

### 9-3. 広告グループ設計

| 原則 | 内容 |
|------|------|
| テーマ統一 | 1広告グループ = 1ターゲティングテーマ |
| リマーケティング分離 | 新規向けとリマケは別広告グループ |
| メッセージの一致 | ターゲットに合わせた広告クリエイティブを設定 |
| 広告数 | 1広告グループあたり3〜4個のレスポンシブ ディスプレイ広告 |

### 9-4. フリークエンシーキャップ

| 項目 | 設定 |
|------|------|
| 設定単位 | キャンペーン / 広告グループ / 広告 |
| 期間 | 日 / 週 / 月 |
| カウント対象 | ビューアブルインプレッションのみ（ディスプレイキャンペーン） |

**推奨値:**
- 認知拡大目的: 1日3〜5回
- リマーケティング: 1日1〜3回
- 配信量が少ない場合はキャップを緩めて検証

### 9-5. 地域・言語・デバイス設定

| 設定 | 注意点 |
|------|--------|
| **地域** | 「所在地 or 関心」のデフォルト設定に注意。関心ベースの配信が不要なら「所在地のみ」に変更 |
| **言語** | ブラウザ言語設定でフィルタ。日本語サイト向けなら「日本語」+「英語」（バイリンガル対策） |
| **デバイス** | スマート入札使用時はデバイス入札調整が実質無効。デバイス別の戦略は構造（キャンペーン分離）で対応 |

## 10. ブランドセーフティとプレースメント管理

### 10-1. インベントリタイプ

| タイプ | 内容 | 推奨シーン |
|--------|------|----------|
| **拡張インベントリ** | 収益化された全コンテンツに配信。強い表現含む | リーチ最大化。ブランド制約が少ない場合 |
| **標準インベントリ** | 大多数のブランドに適するコンテンツに配信 | **デフォルト推奨** |
| **制限インベントリ** | 不適切な言語・性的表現を厳格に排除 | ブランドガイドラインが厳格な企業 |

### 10-2. コンテンツの除外設定

#### センシティブコンテンツの除外

| カテゴリ | 説明 |
|---------|------|
| 悲劇・紛争 | 戦争、自然災害、社会的悲劇に関するコンテンツ |
| デリケートな社会問題 | 人種・宗教・性的指向等に関する議論的コンテンツ |
| 冒涜的表現 | 下品な言葉を含むコンテンツ |
| 性的に示唆的 | 性的に示唆的なコンテンツ |
| センセーショナル・衝撃的 | 過度に刺激的なコンテンツ |

#### コンテンツタイプの除外

| タイプ | 推奨 |
|--------|------|
| ライブストリーミング | 内容が事前確認できないため除外推奨 |
| 埋め込みYouTube動画 | サイト上の埋め込み動画。コンテキスト不明瞭な場合は除外 |
| ビロウ・ザ・フォールド | スクロールしないと見えない位置の広告枠 |
| パークドメイン | 中身のないドメイン。除外推奨 |

### 10-3. プレースメント除外

| レベル | 説明 |
|--------|------|
| アカウントレベル | 全キャンペーンに適用される除外。ブランドセーフティの基盤 |
| キャンペーンレベル | 特定キャンペーンのみに適用 |
| 広告グループレベル | 最も細かい粒度の除外 |

**注意:** 除外はアディティブ（加算）方式。アカウントレベルで除外されたプレースメントは、キャンペーンレベルで解除できない。

#### 除外すべきプレースメント

| カテゴリ | 理由 |
|---------|------|
| モバイルアプリ（不要な場合） | 誤クリック多発。`adsenseformobileapps.com` を除外 |
| ゲームアプリ | 子供向けアプリでの誤クリック。ブランド不一致の場合 |
| パフォーマンスの悪いサイト | 配信レポートで確認し、CTR/CVRが低いサイトを除外 |
| ブランドに不適切なサイト | 業界・ブランドイメージに合わないサイト |

### 10-4. プレースメント精査の手順（週次）

1. **配信先レポートを確認**: キャンペーン → コンテンツ → 配信先 で、配信されたサイト・アプリ一覧を確認
2. **誤クリック兆候を特定**: CTRが異常に高い（5%以上等）のにCVがゼロのプレースメントは誤クリックの可能性
3. **低品質プレースメントを除外**: imp数が多いのにCTR/CVRが著しく低いサイトを除外リストに追加
4. **ブランド適合性を確認**: 配信先のサイト名・URLを目視チェックし、ブランドに不適切なサイトを除外
5. **除外リストを蓄積**: アカウントレベルの除外プレースメントリストとして管理し、新キャンペーンにも適用

### 10-5. モバイルアプリ除外の方法

モバイルアプリは誤タップが多く、ブランド不一致のアプリにも配信されやすい。

| 除外方法 | 手順 |
|---------|------|
| **URLで一括除外** | プレースメント除外に `adsenseformobileapps.com` を追加（GDN全アプリを除外） |
| **アプリカテゴリ除外** | キャンペーン設定 → コンテンツの除外 → アプリカテゴリを選択（ゲーム、エンタメ等を個別除外） |
| **個別アプリ除外** | 配信先レポートでパフォーマンスの悪いアプリIDを特定し、プレースメント除外に追加 |

**推奨:** 全アプリ除外ではなくアプリカテゴリ除外から始め、パフォーマンスを見ながら調整する。全除外はリーチを大幅に制限する。

## 11. 広告の強度（Ad Strength）と最適化

### 11-1. Ad Strengthとは

レスポンシブ ディスプレイ広告の作成時に表示される評価指標。アセットの量・多様性・品質を評価し、「低い」「平均」「良い」「優秀」の4段階で表示される。

### 11-2. 「優秀」を目指すための基準

| 要素 | 推奨 |
|------|------|
| 短い見出し | 5本すべて埋める。角度を変えた多様な内容 |
| 長い見出し | 1本。価値提案を簡潔に |
| 説明文 | 5本すべて埋める。見出しと重複させない |
| 横長画像 | 5枚以上 |
| スクエア画像 | 5枚以上 |
| ロゴ | 最低1つ。1:1 と 4:1 の両方が理想 |

### 11-3. アセットパフォーマンスレポート

十分なデータが蓄積されると、各アセット（見出し・説明文・画像）にパフォーマンスラベルが付与される:

| ラベル | 意味 | アクション |
|--------|------|----------|
| 最良 | 同種の他アセットより優れたパフォーマンス | 維持。類似アセットを追加テスト |
| 良い | 平均的なパフォーマンス | 維持しつつ改善版をテスト |
| 低い | 同種の他アセットよりパフォーマンスが低い | 新しいアセットに差し替え |
| 学習中 | データ蓄積中 | 結果が出るまで待つ |

### 11-4. クリエイティブの更新サイクル

| 期間 | アクション |
|------|----------|
| 配信開始〜2週間 | 学習期間。大きな変更を避ける |
| 2〜4週間後 | アセットパフォーマンスを確認。「低い」アセットを特定 |
| 4週間以降 | 低パフォーマンスアセットを差し替え。新クリエイティブを追加 |
| 継続的 | 2〜4週間ごとにクリエイティブを更新し、広告疲れを防止 |

## 12. 運用チェックリスト

### 配信開始前チェック

- [ ] コンバージョントラッキングが正しく設定されている
- [ ] 画像アセットは横長・スクエアの両方を登録（最低各1枚、推奨各5枚以上）
- [ ] ロゴを登録済み
- [ ] 短い見出し5本・説明文5本を登録
- [ ] 長い見出し1本を登録
- [ ] ビジネス名を設定
- [ ] Ad Strengthが「良い」以上
- [ ] ターゲティング設定が適切（リマケと新規は別広告グループ）
- [ ] 地域設定が「所在地のみ」か「所在地 or 関心」か確認
- [ ] プレースメント除外（モバイルアプリ等）を設定
- [ ] コンテンツ除外（パークドメイン等）を設定
- [ ] フリークエンシーキャップを設定
- [ ] 入札戦略を確認
- [ ] 日予算が適切に設定されている
- [ ] Primary / Secondary コンバージョンの割り当てを確認
- [ ] ビュースルーコンバージョンのウィンドウ設定を確認
- [ ] LPの表示速度がモバイルで3秒以内か確認

### 週次チェック

- [ ] 配信量（imp/click/cost）の推移を確認
- [ ] CTRを確認（極端な低下がないか）
- [ ] CV数・CPA・ROASを確認
- [ ] VTC（ビュースルーCV）の推移を確認

## 13. Demand Gen キャンペーンとの関係（2025年〜）

### 概要

2025年4月以降、Demand GenキャンペーンがGDNの画像在庫にも配信拡大された。これにより、Demand Genキャンペーンは以下の面に配信可能:

- YouTube（インフィード、ショート等）
- Discover
- Gmail
- **Google ディスプレイ ネットワーク（GDN）** ← 2025年追加

### ディスプレイキャンペーン vs Demand Gen

| 項目 | ディスプレイキャンペーン | Demand Gen |
|------|----------------------|------------|
| 配信面 | GDNのみ | YouTube + Discover + Gmail + GDN |
| クリエイティブ | 画像 + テキスト（+ 動画） | 画像 + 動画 + テキスト（リッチ） |
| ターゲティング | オーディエンス + コンテンツ + 最適化ターゲティング | Lookalike + オーディエンス + 最適化ターゲティング |
| 主な用途 | 認知〜CV（幅広い） | 需要創出・検討促進 |
| AI活用度 | 中 | 高（配信面・クリエイティブの最適化が強い） |

### 使い分けの指針

- **ディスプレイキャンペーン**: GDNに特化したコントロールが必要な場合、コンテンツターゲティング（キーワード・トピック・プレースメント指定）を使いたい場合
- **Demand Gen**: YouTube・Discover・Gmailにもまとめてリーチしたい場合、動画クリエイティブを活用したい場合
