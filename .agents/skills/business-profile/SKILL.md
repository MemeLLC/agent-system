# ビジネスプロファイル（Business Profile）

自社・自プロジェクトの情報を包括的に収集し、ドキュメントとしてまとめる。定量的な基本情報だけでなく、事業内容・ターゲット・ビジネスモデル・マーケティング施策・ブランド・チーム・事業フェーズまで、可能な範囲でユーザーと協力しながら網羅する。

## ワークフロー

### Phase 1: ユーザーへのヒアリング

ユーザーに段階的に質問し、既知の情報を収集する。**一度にすべてを聞かず、3〜5項目ずつ段階的に質問すること。** ユーザーが「わからない」と答えた項目は Phase 2 で調査する。

#### Step 1: 基本情報（最初に聞く）

- 会社名（正式名称）と屋号・ブランド名
- サービス・プロジェクト名
- 何をしているビジネスか（一言で）
- ホームページ / サービスサイトのURL

#### Step 2: 会社の詳細

- 代表者名
- 従業員数（概算でOK）
- 設立年
- 所在地
- 資本金・資金調達状況（わかる範囲で）

#### Step 3: 事業内容・ターゲット

- 提供しているサービス・商品の詳細
- ターゲット顧客（B2B / B2C / B2B2C）
- 主な顧客層の特徴（業種、規模、年齢層など）
- 展開地域（日本全国 / 特定地域 / 海外）
- 現在の事業フェーズ（準備中 / ローンチ直後 / 成長期 / 安定期）
- 主な実績・導入事例（わかる範囲で）

#### Step 4: ビジネスモデル・料金

- 収益モデル（サブスク / 従量課金 / 買い切り / 広告 / フリーミアムなど）
- 料金体系・価格帯
- 主な収益源
- 他に展開している事業やサービスがあるか

#### Step 5: マーケティング・デジタルプレゼンス

- LP（ランディングページ）のURL
- SNSアカウント（X、Instagram、Facebook、LinkedIn、YouTube、TikTokなど）
- 現在出稿している広告の有無・プラットフォーム
- その他のマーケティング施策（SEO、コンテンツ、メルマガ、アフィリエイト、PR等）
- 現在の集客チャネルで最も効果的なもの

#### Step 6: ブランド・方向性（余裕があれば）

- ミッション・ビジョン
- ブランドの雰囲気・トーン（堅い / カジュアル / 高級感 / 親しみやすいなど）
- 現在の課題や悩み
- 短期的な目標（3〜6ヶ月）
- 中長期的な方向性

### Phase 2: Web調査による情報補完

ユーザーが把握していない情報、または裏取り・補完が必要な情報を調査する。

#### 会社・サービスの基本情報

```
Step 1: WebSearchで「[会社名] 会社概要」「[会社名] 企業情報」等を検索
Step 2: WebFetchまたはplaywright-browserのscrapeで公式サイトの会社概要ページを取得
```

- 公式サイトの会社概要ページ、IR情報、プレスリリースを確認する
- 企業データベース（マイナビ、Wantedly、Crunchbase等）も補助的に参照する
- 代表者のプロフィール、メディア掲載、登壇情報なども確認する

#### ホームページ・LPの調査

`playwright-browser` スキルを使用して以下を実行する:

1. **ホームページのスクレイピング** — meta情報、見出し構造、主要コンテンツ、訴求ポイントを取得
2. **LPの特定と確認** — ユーザーが把握していないLPがないか調査する
3. **サービスサイトの構造確認** — 料金ページ、機能ページ、導入事例ページ等の有無と内容
4. **スクリーンショットの撮影** — ホームページとすべてのLPをフルページで撮影

```bash
# ホームページ
node $SKILL_DIR/../playwright-browser/scripts/browser.mjs screenshot <HP URL> --output contexts/screenshots/business-profile/hp-01.png --full-page

# LP
node $SKILL_DIR/../playwright-browser/scripts/browser.mjs screenshot <LP URL> --output contexts/screenshots/business-profile/lp-01.png --full-page
```

#### SNS・外部プレゼンスの調査

ユーザーが把握していないSNSアカウントや外部掲載がないか確認する:
- WebSearchで「[会社名] X(Twitter)」「[会社名] Instagram」等を検索
- 見つかったアカウントのフォロワー数、投稿頻度、直近の投稿傾向を確認する
- レビューサイト、メディア掲載、プレスリリースの有無も確認する

#### 広告運用の調査

`playwright-browser` スキルで以下の広告ライブラリにアクセスし、自社の広告出稿状況を調査する。

| ツール | URL | 用途 |
|--------|-----|------|
| Meta広告ライブラリ | `https://www.facebook.com/ads/library/` | Facebook/Instagram広告の検索 |
| Google広告の透明性センター | `https://adstransparency.google.com/?region=JP` | Google広告の検索 |
| TikTok Top Ads | `https://ads.tiktok.com/business/creativecenter/inspiration/topads/pc/ja?period=30&region=JP` | TikTok広告の確認 |

**調査手順:**
1. 会社名・ブランド名・サービス名で各ライブラリを検索する
2. 確認できた広告のクリエイティブ内容、訴求ポイント、遷移先LPを記録する
3. 広告クリエイティブとLPのスクリーンショットを撮影する

```bash
# 広告クリエイティブ
node $SKILL_DIR/../playwright-browser/scripts/browser.mjs screenshot <広告ライブラリURL> --output contexts/screenshots/business-profile/ad-01.png

# 広告のLP
node $SKILL_DIR/../playwright-browser/scripts/browser.mjs screenshot <LP URL> --output contexts/screenshots/business-profile/ad-lp-01.png --full-page
```

### Phase 3: ドキュメント化

収集した全情報をマークダウンファイルとしてまとめる。情報が得られなかった項目は「未確認」と記載し、空欄にしない。

## アウトプットルール

### 包括的かつ事実ベースの記述

- ユーザーから聞いた情報とWeb調査で得た情報を統合してまとめる
- 分析や評価は含めない（それは後続のスキルの役割）。事実のみを記録する
- URLは必ず記載する
- 情報が得られなかった項目は「未確認」と明記する

### ファイル出力

出力先はユーザーの指示に従う（デフォルト: `contexts/` ディレクトリ）。

### ファイル構成

```
contexts/
├── business-profile.md                      # ビジネスプロファイル
└── screenshots/business-profile/            # スクリーンショット
    ├── hp-01.png                            # ホームページ
    ├── lp-01.png                            # ランディングページ
    ├── ad-01.png                            # 広告クリエイティブ
    └── ad-lp-01.png                         # 広告のLP
```
